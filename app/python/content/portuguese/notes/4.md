## Bibliotecas <a id="bibliotecas"></a>

- Geralmente, bibliotecas são trechos de código escritos por você ou por outros que você pode utilizar em seu programa.
- O Python permite que você compartilhe funções ou recursos com outras pessoas como "módulos".
- Se você copiar e colar código de um projeto antigo, é provável que você possa criar um módulo ou biblioteca que possa ser utilizado em seu novo projeto.

## Aleatório <a id="random"></a>

- `random` é uma biblioteca que vem com o Python e pode ser importada para o seu próprio projeto.
- É mais fácil para um programador aproveitar o trabalho dos programadores anteriores.
- Então, como carregar um módulo no seu próprio programa? Você pode usar a palavra `import` no seu programa.
- Dentro do módulo `random`, existe uma função embutida chamada `random.choice(seq)`. `random` é o módulo que você está importando. Dentro desse módulo, há a função `choice`. Essa função recebe uma `seq` ou sequência que é uma lista.
- Na janela do terminal, digite `code generate.py`. No seu editor de texto, codifique da seguinte forma:

      import random

      coin = random.choice(["cara", "coroa"])
      print(coin)

  Note que a lista dentro do `choice` possui colchetes, aspas e uma vírgula. Como você passou dois itens, o Python faz o cálculo e dá uma chance de 50% para cara e coroa. Ao executar o código, você verá que ele realmente funciona bem!

- Podemos melhorar nosso código. O `from` nos permite ser mais específicos sobre o que gostaríamos de importar. Anteriormente, nossa linha de código `import` estava trazendo todo o conteúdo das funções do `random`. No entanto, e se quisermos carregar apenas uma pequena parte de um módulo? Modifique seu código da seguinte forma:

      from random import choice

      coin = choice(["cara", "coroa"])
      print(coin)

  Note que agora podemos importar apenas a função `choice` do `random`. A partir desse ponto, não precisamos mais codificar `random.choice`. Agora, podemos usar apenas `choice`. A função `choice` é carregada explicitamente em nosso programa. Isso economiza recursos do sistema e potencialmente pode fazer com que nosso código seja executado mais rápido!

- Seguindo em frente, considere a função `random.randint(a, b)`. Essa função gera um número aleatório entre `a` e `b`. Modifique seu código da seguinte forma:

      import random

      number = random.randint(1, 10)
      print(number)

  Observe que nosso código gerará aleatoriamente um número entre `1` e `10`.

- Podemos introduzir em nosso código `random.shuffle(x)`, onde ele embaralhará uma lista em uma ordem aleatória.

      import random

      cards = ["valete", "dama", "rei"]
      random.shuffle(cards)
      for card in cards:
          print(card)

  Note que o `random.shuffle` embaralhará as cartas no próprio lugar. Ao contrário de outras funções, ele não retornará um valor. Em vez disso, ele pegará a lista `cards` e as embaralhará dentro dessa lista. Execute seu código várias vezes para ver o código funcionando.

- Agora temos essas três maneiras acima de gerar informações aleatórias.
- Você pode aprender mais na documentação do Python sobre [`random`](https://docs.python.org/3/library/random.html).

## Estatísticas <a id="estatisticas"></a>

- Python vem com uma biblioteca embutida chamada `statistics`. Como podemos usar esse módulo?
- `average` é uma função dessa biblioteca que é bastante útil. Na janela do seu terminal, digite `code average.py`. Na janela do editor de texto, modifique o seu código da seguinte forma:

      import statistics

      print(statistics.mean([100, 90]))

  Observe que importamos uma biblioteca diferente chamada `statistics`. A função `mean` recebe uma lista de valores. Isso imprimirá a média desses valores. Na janela do seu terminal, digite `python average.py`.

- Considere as possibilidades de usar o módulo `statistics` em seus próprios programas.
- Você pode aprender mais na documentação do Python sobre [`statistics`](https://docs.python.org/3/library/statistics.html).

## Argumentos da Linha de Comando <a id="command-line-arguments"></a>

- Até agora, temos fornecido todos os valores dentro do programa que criamos. E se quiséssemos ser capazes de receber entrada da linha de comando? Por exemplo, em vez de digitar `python average.py` no terminal, e se quiséssemos ser capazes de digitar `python average.py 100 90` e obter a média entre `100` e `90`?
- `sys` é um módulo que nos permite receber argumentos na linha de comando.
- `argv` é uma função dentro do módulo `sys` que nos permite descobrir o que o usuário digitou na linha de comando. Observe como você verá `sys.argv` sendo utilizado no código abaixo. Na janela do terminal, digite `code name.py`. No editor de texto, escreva o seguinte código:

      import sys

      print("olá, meu nome é", sys.argv[1])

  Observe que o programa vai analisar o que o usuário digitou na linha de comando. Atualmente, se você digitar `python name.py David` na janela do terminal, você verá `olá, meu nome é David`. Observe que `sys.argv[1]` é onde `David` está sendo armazenado. Por quê? Bem, em lições anteriores, você pode se lembrar que listas começam no elemento `0`. O que você acha que está atualmente armazenado em `sys.argv[0]`? Se você disse `name.py`, você está correto!

- Existe um pequeno problema com nosso programa conforme está. E se o usuário não digitar o nome na linha de comando? Experimente você mesmo. Digite `python name.py` na janela do terminal. O compilador apresentará um erro `list index out of range`. A razão para isso é que não há nada em `sys.argv[1]` porque nada foi digitado! Veja como podemos proteger nosso programa desse tipo de erro:

      import sys

      try:
          print("olá, meu nome é", sys.argv[1])
      except IndexError:
          print("Argumentos insuficientes")

  Observe que o usuário agora receberá uma dica útil sobre como fazer o programa funcionar se ele esquecer de digitar um nome. No entanto, poderíamos ser mais defensivos para garantir que o usuário insira os valores corretos?

- Nosso programa pode ser melhorado da seguinte forma:

      import sys

      if len(sys.argv) < 2:
          print("Argumentos insuficientes")
      elif len(sys.argv) > 2:
          print("Argumentos demais")
      else:
          print("olá, meu nome é", sys.argv[1])

  Observe como, se você testar seu código, você verá como essas exceções são tratadas, fornecendo ao usuário conselhos mais refinados. Mesmo se o usuário digitar argumentos demais ou insuficientes, ele receberá instruções claras sobre como corrigir o problema.

- No momento, nosso código está logicamente correto. No entanto, há algo muito bom em manter nossa verificação de erros separada do restante do código. Como poderíamos separar nosso tratamento de erros? Modifique seu código da seguinte forma:

      import sys

      if len(sys.argv) < 2:
          sys.exit("Argumentos insuficientes")
      elif len(sys.argv) > 2:
          sys.exit("Argumentos demais")

      print("olá, meu nome é", sys.argv[1])

  Observe como estamos usando uma função interna do `sys` chamada `exit` que nos permite sair do programa se um erro for introduzido pelo usuário. Agora, podemos ter a certeza de que o programa nunca executará a última linha de código e acionará um erro. Portanto, `sys.argv` fornece uma maneira pela qual os usuários podem introduzir informações da linha de comando. `sys.exit` fornece um meio pelo qual o programa pode sair se ocorrer um erro.

- Você pode aprender mais na documentação do Python sobre [`sys`](https://docs.python.org/3/library/sys.html).

## `slice` <a id="slice"></a>

- `slice` é um comando que nos permite pegar uma `lista` e indicar ao compilador onde queremos que ele considere o início e o fim da `lista`. Por exemplo, modifique o seu código da seguinte forma:

      import sys

      if len(sys.argv) < 2:
          sys.exit("Muito poucos argumentos")

      for arg in sys.argv:
          print("Olá, meu nome é", arg)

  Observe que se você digitar `python nome.py David Carter Rongxin` na janela do terminal, o compilador irá exibir não apenas a saída desejada com os nomes, mas também `Olá, meu nome é nome.py`. Como poderíamos então fazer com que o compilador ignore o primeiro elemento da lista onde `nome.py` está atualmente armazenado?

- `slice` pode ser usado em nosso código para iniciar a lista em um local diferente! Modifique o seu código da seguinte forma:

      import sys

      if len(sys.argv) < 2:
          sys.exit("Muito poucos argumentos")

      for arg in sys.argv[1:]:
          print("Olá, meu nome é", arg)

  Observe que, ao invés de começar a lista em `0`, usamos colchetes para indicar ao compilador que ele deve começar em `1` e ir até o final usando o argumento `1:`. Ao executar este código, você perceberá que podemos melhorar o nosso código usando uma sintaxe relativamente simples.

## Pacotes <a id="pacotes"></a>

- Uma das razões pelas quais o Python é tão popular é porque existem inúmeras bibliotecas de terceiros poderosas que adicionam funcionalidades. Chamamos essas bibliotecas de terceiros, implementadas como uma pasta, de "pacotes".
- O PyPI é um repositório ou diretório de todos os pacotes de terceiros atualmente disponíveis.
- `cowsay` é um pacote conhecido que permite que uma vaca converse com o usuário.
- O Python possui um gerenciador de pacotes chamado `pip` que permite instalar pacotes rapidamente em seu sistema.
- Na janela do terminal, você pode instalar o pacote `cowsay` digitando `pip install cowsay`. Após um pouco de saída, você agora pode usar este pacote em seu código.
- Na janela do terminal, digite `code say.py`. No editor de texto, insira o seguinte código:

      import cowsay
      import sys

      if len(sys.argv) == 2:
          cowsay.cow("olá, " + sys.argv[1])

  Observe que o programa verifica primeiro se o usuário inseriu pelo menos dois argumentos na linha de comando. Em seguida, a vaca deve falar com o usuário. Digite `python say.py David` e você verá uma vaca dizendo "olá" para David.

- Modifique ainda mais seu código:

      import cowsay
      import sys

      if len(sys.argv) == 2:
          cowsay.trex("olá, " + sys.argv[1])

  Observe que agora um t-rex está dizendo "olá".

- Agora você pode ver como instalar pacotes de terceiros.
- Você pode aprender mais sobre o [`cowsay`](https://pypi.org/project/cowsay/) na entrada do PyPI.
- Você pode encontrar outros pacotes de terceiros em [PyPI](https://pypi.org/).

## APIs <a id="apis"></a>

- APIs ou "interfaces de programa de aplicação" permitem que você se conecte ao código de outros.
- `requests` é um pacote que permite que seu programa se comporte como um navegador da web.
- No seu terminal, digite `pip install requests`. Em seguida, digite `code itunes.py`.
- Acontece que o Apple iTunes possui sua própria API que você pode acessar em seus programas. No seu navegador de internet, você pode visitar [https://itunes.apple.com/search?entity=song&limit=1&term=weezer](https://itunes.apple.com/search?entity=song&limit=1&term=weezer) e um arquivo de texto será baixado. David construiu este URL lendo a documentação da API da Apple. Observe como esta consulta está procurando uma `música`, com um `limite` de um resultado, relacionado ao `termo` chamado `weezer`. Ao observar este arquivo de texto que é baixado, você pode achar o formato semelhante ao que programamos anteriormente em Python.
- O formato no arquivo de texto baixado é chamado de JSON, um formato baseado em texto que é usado para trocar dados baseados em texto entre aplicativos. Literalmente, a Apple está fornecendo um arquivo JSON que poderíamos interpretar em nosso próprio programa Python.
- Na janela do terminal, digite `code itunes.py`. O código é o seguinte:

      import requests
      import sys

      if len(sys.argv) != 2:
          sys.exit()

      response = requests.get("https://itunes.apple.com/search?entity=song&limit=1&term=" + sys.argv[1])
      print(response.json())

  Observe como o valor retornado de `requests.get` será armazenado em `response`. David, tendo lido a documentação da Apple sobre essa API, sabe que o que é retornado é um arquivo JSON. Executando `python itunes.py weezer`, você verá o arquivo JSON retornado pela Apple. No entanto, a resposta JSON é convertida pelo Python em um dicionário. Ao observar a saída, pode ser bem confuso!

- Acontece que o Python possui uma biblioteca JSON incorporada que pode nos ajudar a interpretar os dados recebidos. Modifique seu código como segue:

      import json
      import requests
      import sys

      if len(sys.argv) != 2:
          sys.exit()

      response = requests.get("https://itunes.apple.com/search?entity=song&limit=1&term=" + sys.argv[1])
      print(json.dumps(response.json(), indent=2))

  Observe que `json.dumps` é implementado de forma que utiliza `indent` para tornar a saída mais legível. Executando `python itunes.py weezer`, você verá o mesmo arquivo JSON. No entanto, desta vez, está muito mais legível. Observe agora que você verá um dicionário chamado `results` dentro da saída. Dentro desse dicionário chamado `results`, existem várias chaves presentes. Observe o valor `trackName` na saída. Qual nome de faixa você vê nos seus resultados?

- Como poderíamos simplesmente imprimir apenas esse nome da faixa? Modifique seu código como segue:

      import json
      import requests
      import sys

      if len(sys.argv) != 2:
          sys.exit()

      response = requests.get("https://itunes.apple.com/search?entity=song&limit=50&term=" + sys.argv[1])

      o = response.json()
      for result in o["results"]:
          print(result["trackName"])

  Observe como estamos pegando o resultado de `response.json()` e armazenando-o em `o` (como em letra minúscula). Em seguida, estamos iterando pelos `results` em `o` e imprimindo cada `trackName`. Observe também como aumentamos o número limite de resultados para `50`. Execute seu programa. Veja os resultados.

- Você pode aprender mais sobre o `requests` através da [documentação da biblioteca](https://docs.python-requests.org/).
- Você pode aprender mais sobre JSON na documentação do [JSON do Python](https://docs.python.org/3/library/json.html).

## Criando suas próprias bibliotecas <a id="making-your-own-libraries"></a>

- Como programador em Python, você tem a capacidade de criar sua própria biblioteca!
- Imagine situações em que você deseja reutilizar trechos de código várias vezes ou até mesmo compartilhá-los com outras pessoas!
- Até agora neste curso, escrevemos muito código para dizer "olá". Vamos criar um pacote que nos permita dizer "olá" e "adeus". Na janela do terminal, digite `code sayings.py`. No editor de texto, escreva o código a seguir:

      def hello(nome):
          print(f"olá, {nome}")


      def adeus(nome):
          print(f"adeus, {nome}")

  Observe que este código, por si só, não faz nada para o usuário. No entanto, se um programador importar este pacote em seu próprio programa, as capacidades criadas pelas funções acima podem ser implementadas em seu código.

- Vamos ver como podemos implementar um código utilizando o pacote que criamos. Na janela do terminal, digite `code say.py`. Neste novo arquivo no seu editor de texto, digite o seguinte:

      import sys

      from sayings import adeus

      if len(sys.argv) == 2:
          adeus(sys.argv[1])

  Observe que este código importa as capacidades de `adeus` do pacote `sayings`. Se o usuário inseriu pelo menos dois argumentos na linha de comando, ele dirá "adeus" junto com a string inserida na linha de comando.
