## `set` <a id="set"></a>

-   En matemáticas, un conjunto se consideraría un conjunto de números sin duplicados.
-   En la ventana del editor de texto, el código sería el siguiente:

        students = [
            {"name": "Hermione", "house": "Gryffindor"},
            {"name": "Harry", "house": "Gryffindor"},
            {"name": "Ron", "house": "Gryffindor"},
            {"name": "Draco", "house": "Slytherin"},
            {"name": "Padma", "house": "Ravenclaw"},
        ]

        houses = []
        for student in students:
            if student["house"] not in houses:
                houses.append(student["house"])

        for house in sorted(houses):
            print(house)

    Observa cómo tenemos una lista de diccionarios, cada uno representando a un estudiante. Se crea una lista vacía llamada `houses`. Iteramos a través de cada `student` en `students`. Si la `house` del estudiante no está en `houses`, la agregamos a nuestra lista de `houses`.

-   Resulta que podemos usar las funciones incorporadas de `set` para eliminar duplicados.
-   En la ventana del editor de texto, el código sería el siguiente:

        students = [
            {"name": "Hermione", "house": "Gryffindor"},
            {"name": "Harry", "house": "Gryffindor"},
            {"name": "Ron", "house": "Gryffindor"},
            {"name": "Draco", "house": "Slytherin"},
            {"name": "Padma", "house": "Ravenclaw"},
        ]

        houses = set()
        for student in students:
            houses.add(student["house"])

        for house in sorted(houses):
            print(house)

    Observa cómo no es necesario incluir comprobaciones para asegurarnos de que no haya duplicados. El objeto `set` se encarga automáticamente de esto por nosotros.

-   Puedes aprender más en la documentación de Python sobre [`set`](https://docs.python.org/3/library/stdtypes.html#set).

## Variables Globales <a id="variables-globales"></a>

-   En otros lenguajes de programación, existe la noción de variables globales que son accesibles desde cualquier función.
-   Podemos aprovechar esta capacidad en Python. En la ventana del editor de texto, escribe el siguiente código:

        balance = 0


        def main():
            print("Balance:", balance)


        if __name__ == "__main__":
            main()

    Observa cómo creamos una variable global llamada `balance`, fuera de cualquier función.

-   Si ejecutamos el código anterior sin errores, pensaríamos que todo está bien. ¡Pero no es así! En la ventana del editor de texto, escribe el siguiente código:

        balance = 0


        def main():
            print("Balance:", balance)
            depositar(100)
            retirar(50)
            print("Balance:", balance)


        def depositar(n):
            balance += n


        def retirar(n):
            balance -= n


        if __name__ == "__main__":
            main()

    Observa cómo ahora agregamos la funcionalidad para agregar y retirar fondos en `balance`. Sin embargo, al ejecutar este código, ¡aparece un error! Vemos un error llamado `UnboundLocalError`. Puedes suponer que, al menos en la forma en que hemos codificado `balance` y nuestras funciones `depositar` y `retirar`, no podemos reasignarle un valor dentro de una función.

-   Para interactuar con una variable global dentro de una función, la solución es usar la palabra clave `global`. En la ventana del editor de texto, escribe el siguiente código:

        balance = 0


        def main():
            print("Balance:", balance)
            depositar(100)
            retirar(50)
            print("Balance:", balance)


        def depositar(n):
            global balance
            balance += n


        def retirar(n):
            global balance
            balance -= n


        if __name__ == "__main__":
            main()

    Observa cómo la palabra clave `global` le indica a cada función que `balance` no hace referencia a una variable local: en cambio, hace referencia a la variable global que colocamos al principio de nuestro código. Ahora, nuestro código funciona correctamente.

-   Utilizando nuestras habilidades de la programación orientada a objetos, podemos modificar nuestro código para utilizar una clase en lugar de una variable global. En la ventana del editor de texto, escribe el siguiente código:

        class Cuenta:
            def __init__(self):
                self._balance = 0

            @property
            def balance(self):
                return self._balance

            def depositar(self, n):
                self._balance += n

            def retirar(self, n):
                self._balance -= n


        def main():
            cuenta = Cuenta()
            print("Balance:", cuenta.balance)
            cuenta.depositar(100)
            cuenta.retirar(50)
            print("Balance:", cuenta.balance)


        if __name__ == "__main__":
            main()

    Observa cómo utilizamos `cuenta = Cuenta()` para crear una cuenta. Las clases nos permiten resolver este problema de necesitar una variable global de manera más limpia, ya que estas variables de instancia son accesibles para todos los métodos de la clase utilizando `self`.

-   En general, las variables globales se deben utilizar con mucha moderación, si es que se utilizan.

## Constantes <a id="constants"></a>

-   Algunos lenguajes te permiten crear variables que son invariables, llamadas "constantes". Las constantes permiten programar de forma defensiva y reducir las oportunidades de modificar valores importantes.
-   En la ventana del editor de texto, escribe el siguiente código:

        MEOWS = 3

        for _ in range(MEOWS):
            print("meow")

    Observa que `MEOWS` es nuestra constante en este caso. Las constantes generalmente se representan con nombres de variables en mayúsculas y se colocan en la parte superior de nuestro código. Aunque esto _parezca_ una constante, en realidad, ¡Python no tiene un mecanismo para evitar que cambiemos ese valor dentro de nuestro código! En su lugar, estás en el sistema de honor: si el nombre de una variable está escrito en mayúsculas, ¡simplemente no lo cambies!

-   Puedes crear una clase "constante", ahora entre comillas porque sabemos que Python no admite completamente las "constantes". En la ventana del editor de texto, escribe el siguiente código:

        class Cat:
            MEOWS = 3

            def meow(self):
                for _ in range(Cat.MEOWS):
                    print("meow")

        cat = Cat()
        cat.meow()

    Debido a que `MEOWS` se define fuera de un método específico de la clase, todos ellos tienen acceso a ese valor a través de `Cat.MEOWS`.

## Indicaciones de tipo <a id="indicaciones-de-tipo"></a>

-   En otros lenguajes de programación, se expresa explícitamente qué tipo de variable se desea utilizar.
-   Como vimos anteriormente en el curso, Python no requiere la declaración explícita de tipos.
-   Sin embargo, es una buena práctica asegurarse de que todas las variables sean del tipo adecuado.
-   `mypy` es un programa que puede ayudarte a comprobar que todas tus variables sean del tipo correcto.
-   Puedes instalar `mypy` ejecutando en la ventana de la terminal: `pip install mypy`.

En la ventana del editor de texto, escribe el siguiente código:

      def meow(n):
          for _ in range(n):
              print("meow")


      number = input("Número: ")
      meow(number)

Es probable que ya te hayas dado cuenta de que `number = input("Número: )"` devuelve una `cadena`, no un `entero`. ¡Pero `meow` probablemente esperará un `entero`!

-   Se puede agregar una indicación de tipo para darle a Python una pista sobre el tipo de variable que `meow` debería esperar. En la ventana del editor de texto, escribe el siguiente código:

        def meow(n: int):
            for _ in range(n):
                print("meow")


        number = input("Número: ")
        meow(number)

    Sin embargo, observa que nuestro programa todavía produce un error.

-   Después de instalar `mypy`, ejecuta `mypy meows.py` en la ventana de la terminal. `mypy` proporcionará orientación sobre cómo corregir este error.
-   Puedes anotar todas tus variables. En la ventana del editor de texto, escribe el siguiente código:

        def meow(n: int):
            for _ in range(n):
                print("meow")


        number: int = input("Número: ")
        meow(number)

    Observa cómo ahora se le proporciona una indicación de tipo a `number`.

-   Nuevamente, al ejecutar `mypy meows.py` en la ventana de la terminal, obtendrás comentarios mucho más específicos como programador.
-   Podemos corregir nuestro error final simplemente de la siguiente manera:

        def meow(n: int):
            for _ in range(n):
                print("meow")


        number: int = int(input("Número: "))
        meow(number)

    Observa cómo al ejecutar `mypy` ya no se producen errores porque convertimos nuestra entrada en un entero.

-   Introduzcamos un nuevo error al asumir que `meow` nos devolverá una cadena o `str`. En la ventana del editor de texto, escribe el siguiente código:

        def meow(n: int):
            for _ in range(n):
                print("meow")


        number: int = int(input("Número: "))
        meows: str = meow(number)
        print(meows)

    Observa cómo la función `meow` solo tiene un efecto secundario. Debido a que solo intentamos imprimir "meow" y no devolvemos ningún valor, se produce un error cuando intentamos almacenar el valor de retorno de `meow` en `meows`.

-   También podemos usar indicaciones de tipo para verificar errores, esta vez anotando los valores de retorno de las funciones. En la ventana del editor de texto, escribe el siguiente código:

        def meow(n: int) -> None:
            for _ in range(n):
                print("meow")


        number: int = int(input("Número: "))
        meows: str = meow(number)
        print(meows)

    Observa cómo la notación `-> None` le indica a `mypy` que no hay un valor de retorno.

-   Podemos modificar nuestro código para devolver una cadena si así lo deseamos:

        def meow(n: int) -> str:
            return "meow\n" * n


        number: int = int(input("Número: "))
        meows: str = meow(number)
        print(meows, end="")

    Observa cómo almacenamos múltiples `str` en `meows`. Al ejecutar `mypy`, no se producen errores.

-   Puedes obtener más información en la documentación de Python sobre [Indicaciones de Tipo](https://docs.python.org/es/3/library/typing.html).
-   También puedes obtener más información sobre [`mypy`](https://mypy.readthedocs.io/) a través de la documentación del propio programa.

## Docstrings <a id="docstrings"></a>

-   Una forma estándar de comentar el propósito de tu función es utilizar un docstring. En la ventana del editor de texto, escribe el código de la siguiente manera:

        def maullar(n):
            """Maulla n veces."""
            return "maullido\n" * n


        numero = int(input("Número: "))
        maullidos = maullar(numero)
        print(maullidos, end="")

    Observa cómo las tres comillas dobles designan lo que hace la función.

-   Puedes utilizar los docstrings para estandarizar cómo documentas las características de una función. En la ventana del editor de texto, escribe el código de la siguiente manera:

        def maullar(n):
            """
            Maulla n veces.

            :param n: Número de veces que se va a maullar
            :type n: int
            :raise TypeError: Si n no es un entero
            :return: Una cadena de maullidos, uno por línea
            :rtype: str
            """
            return "maullido\n" * n


        numero = int(input("Número: "))
        maullidos = maullar(numero)
        print(maullidos, end="")

    Observa cómo se incluyen varios argumentos en el docstring. Por ejemplo, describe los parámetros que toma la función y lo que devuelve la función.

-   Herramientas establecidas, como [Sphinx](https://www.sphinx-doc.org/en/master/index.html), se pueden utilizar para analizar los docstrings y crear automáticamente documentación en forma de páginas web y archivos PDF para que puedas publicar y compartir con otros.
-   Puedes obtener más información en la documentación de Python sobre [docstrings](https://peps.python.org/pep-0257/).

## `argparse` <a id="argparse"></a>

-   Supongamos que queremos usar argumentos de línea de comandos en nuestro programa. En la ventana del editor de texto, escriba el siguiente código:

        import sys

        if len(sys.argv) == 1:
            print("meow")
        elif len(sys.argv) == 3 and sys.argv[1] == "-n":
            n = int(sys.argv[2])
            for _ in range(n):
                print("meow")
        else:
            print("uso: meows.py [-n NUMERO]")

    Observe cómo se importa `sys`, a partir del cual obtenemos acceso a `sys.argv` - una matriz de argumentos de línea de comandos que se le pasan a nuestro programa cuando se ejecuta. Podemos usar varias declaraciones `if` para verificar si el usuario ha ejecutado correctamente nuestro programa.

-   Supongamos que este programa se va a volver mucho más complicado. ¿Cómo podríamos verificar todos los argumentos que el usuario podría ingresar? ¡Podríamos rendirnos si tenemos más de unos pocos argumentos de línea de comandos!
-   Afortunadamente, `argparse` es una biblioteca que maneja el análisis de cadenas complicadas de argumentos de línea de comandos. En la ventana del editor de texto, escriba el siguiente código:

        import argparse

        parser = argparse.ArgumentParser()
        parser.add_argument("-n")
        args = parser.parse_args()

        for _ in range(int(args.n)):
            print("meow")

    Observe cómo se importa `argparse` en lugar de `sys`. Se crea un objeto llamado `parser` a partir de una clase `ArgumentParser`. El método `add_argument` de esa clase se utiliza para indicarle a `argparse` qué argumentos debemos esperar del usuario cuando ejecuten nuestro programa. Finalmente, al ejecutar el método `parse_args` del analizador se asegura de que todos los argumentos hayan sido incluidos correctamente por el usuario.

-   También podemos programar de manera más limpia para que nuestro usuario pueda obtener información sobre el uso correcto de nuestro código cuando no lo utiliza correctamente. En la ventana del editor de texto, escriba el siguiente código:

        import argparse

        parser = argparse.ArgumentParser(description="Maúlla como un gato")
        parser.add_argument("-n", help="número de veces que debe maullar")
        args = parser.parse_args()

        for _ in range(int(args.n)):
            print("meow")

    Observe cómo se proporciona documentación al usuario. Específicamente, se proporciona un argumento de `help`. Ahora, si el usuario ejecuta `python meows.py --help` o `-h`, se le presentarán algunas pistas sobre cómo usar este programa.

-   Podemos mejorar aún más este programa. En la ventana del editor de texto, escriba el siguiente código:

        import argparse

        parser = argparse.ArgumentParser(description="Maúlla como un gato")
        parser.add_argument("-n", default=1, help="número de veces que debe maullar", type=int)
        args = parser.parse_args()

        for _ in range(args.n):
            print("meow")

    Observe cómo no solo se incluye la documentación de ayuda, sino que también se puede proporcionar un valor `predeterminado` cuando el usuario no proporciona argumentos.

-   Puedes obtener más información en la documentación de Python de [`argparse`](https://docs.python.org/3/library/argparse.html).

## Desempaquetado <a id="desempaquetado"></a>

-   ¿No sería bueno poder dividir una única variable en dos variables? En la ventana del editor de texto, coloca el siguiente código:

        primero, _ = input("¿Cuál es tu nombre? ").split(" ")
        print(f"hola, {primero}")

    Observa cómo este programa intenta obtener el primer nombre del usuario dividiéndolo ingenuamente por un espacio.

-   Resulta que hay otras formas de desempaquetar variables. Puedes escribir un código más potente y elegante al entender cómo desempaquetar variables de formas aparentemente más avanzadas. En la ventana del editor de texto, coloca el siguiente código:

        def total(galeones, sickles, knuts):
            return (galeones * 17 + sickles) * 29 + knuts


        print(total(100, 50, 25), "Knuts")

    Observa cómo esto devuelve el valor total de Knuts.

-   ¿Y si quisiéramos almacenar nuestras monedas en una lista? En la ventana del editor de texto, coloca el siguiente código:

        def total(galeones, sickles, knuts):
            return (galeones * 17 + sickles) * 29 + knuts


        monedas = [100, 50, 25]

        print(total(monedas[0], monedas[1], monedas[2]), "Knuts")

    Observa cómo se crea una lista llamada `monedas`. Podemos pasar cada valor indexando con `0`, `1`, y así sucesivamente.

-   Esto se está volviendo bastante extenso. ¿No sería bueno si pudiéramos simplemente pasar la lista de monedas a nuestra función?
-   Para habilitar la posibilidad de pasar la lista completa, podemos usar el desempaquetado. En la ventana del editor de texto, coloca el siguiente código:

        def total(galeones, sickles, knuts):
            return (galeones * 17 + sickles) * 29 + knuts


        monedas = [100, 50, 25]

        print(total(*monedas), "Knuts")

    Observa cómo el `*` desempaqueta la secuencia de la lista de monedas y pasa cada uno de sus elementos individuales a `total`.

-   ¿Y si pudiéramos pasar los nombres de las monedas en cualquier orden? En la ventana del editor de texto, coloca el siguiente código:

        def total(galeones, sickles, knuts):
            return (galeones * 17 + sickles) * 29 + knuts


        print(total(galeones=100, sickles=50, knuts=25), "Knuts")

    Observa cómo esto sigue calculando correctamente.

-   Cuando empiezas a hablar de "nombres" y "valores", ¡los diccionarios podrían venir a la mente! Puedes implementar esto como un diccionario. En la ventana del editor de texto, coloca el siguiente código:

        def total(galeones, sickles, knuts):
            return (galeones * 17 + sickles) * 29 + knuts


        monedas = {"galeones": 100, "sickles": 50, "knuts": 25}

        print(total(monedas["galeones"], monedas["sickles"], monedas["knuts"]), "Knuts")

    Observa cómo se proporciona un diccionario llamado `monedas`. Podemos acceder a él indexando con las claves, como "galeones" o "sickles".

-   Dado que la función `total` espera tres argumentos, no podemos pasar un diccionario. Podemos usar el desempaquetado para ayudar con esto. En la ventana del editor de texto, coloca el siguiente código:

        def total(galeones, sickles, knuts):
            return (galeones * 17 + sickles) * 29 + knuts


        monedas = {"galeones": 100, "sickles": 50, "knuts": 25}

        print(total(**monedas), "Knuts")

    Observa cómo `**` te permite desempaquetar un diccionario. Al desempaquetar un diccionario, proporciona tanto las claves como los valores.

## `args` y `kwargs` <a id="args-and-kwargs"></a>

-   Recordemos la documentación de `print` que vimos anteriormente en este curso:

        print(*objetos, sep=' ', end='\n', file=sys.stdout, flush=False)

-   `args` son argumentos posicionales, como los que proporcionamos a `print` como `print("Hola", "Mundo")`.
-   `kwargs` son argumentos nombrados, o "argumentos de palabras clave", como los que proporcionamos a `print` como `print(end="")`.
-   Como vemos en el prototipo de la función `print` anterior, podemos indicar a nuestra función que espere un número desconocido de argumentos posicionales. También podemos indicarle que espere un número desconocido de argumentos de palabras clave. En la ventana del editor de texto, ingresamos el siguiente código:

        def f(*args, **kwargs):
            print("Posicional:", args)


        f(100, 50, 25)

    Observa cómo al ejecutar este código se imprimirán los argumentos posicionales.

-   Incluso podemos pasar argumentos nombrados. En la ventana del editor de texto, ingresamos el siguiente código:

        def f(*args, **kwargs):
            print("Nombrado:", kwargs)


        f(galeones=100, sickles=50, knuts=25)

    Observa cómo los valores nombrados se proporcionan en forma de un diccionario.

-   Al pensar en la función `print` anterior, puedes ver cómo `*objetos` toma cualquier número de argumentos posicionales.
-   Puedes obtener más información en la documentación de Python sobre [`print`](https://docs.python.org/3/library/functions.html#print).

## `map` <a id="map"></a>

-   Al principio, comenzamos con la programación procedural.
-   Más tarde descubrimos que Python es un lenguaje de programación orientado a objetos.
-   Vimos indicios de la programación funcional, donde las funciones tienen efectos secundarios sin un valor de retorno. Podemos ilustrar esto en la ventana del editor de texto, escribiendo `code yell.py` y el siguiente código:

        def main():
            yell("This is CS50")


        def yell(word):
            print(word.upper())


        if __name__ == "__main__":
            main()

    Observa cómo la función `yell` simplemente grita.

-   ¿No sería genial gritar una lista de palabras ilimitadas? Modifica tu código de la siguiente manera:

        def main():
            yell(["This", "is", "CS50"])


        def yell(words):
            uppercased = []
            for word in words:
                uppercased.append(word.upper())
            print(*uppercased)


        if __name__ == "__main__":
            main()

    Observa cómo acumulamos las palabras en mayúsculas, iterando sobre cada una de las palabras y convirtiéndolas en mayúsculas. La lista de mayúsculas se imprime utilizando `*` para desempaquetarla.

-   Quitando los corchetes, podemos pasar las palabras como argumentos. En la ventana del editor de texto, escribe el siguiente código:

        def main():
            yell("This", "is", "CS50")


        def yell(*words):
            uppercased = []
            for word in words:
                uppercased.append(word.upper())
            print(*uppercased)


        if __name__ == "__main__":
            main()

    Observa cómo `*words` permite tomar muchos argumentos por la función.

-   `map` te permite aplicar una función a una secuencia de valores. En la práctica, podemos escribir el siguiente código:

        def main():
            yell("This", "is", "CS50")


        def yell(*words):
            uppercased = map(str.upper, words)
            print(*uppercased)


        if __name__ == "__main__":
            main()

    Observa cómo `map` toma dos argumentos. Primero, toma una función que queremos aplicar a cada elemento de una lista. Segundo, toma esa lista en sí misma, a la cual aplicaremos la función mencionada anteriormente. Por lo tanto, todas las palabras en `words` serán pasadas a la función `str.upper` y luego retornadas a `uppercased`.

-   Puedes aprender más en la documentación de Python sobre [`map`](https://docs.python.org/3/library/functions.html#map).

## Comprensión de listas<a id="comprension-de-listas"></a>

-   Las comprensiones de listas te permiten crear una lista sobre la marcha en una elegante línea de código.
-   Podemos implementar esto en nuestro código de la siguiente manera:

        def main():
            yell("This", "is", "CS50")


        def yell(*words):
            uppercased = [arg.upper() for arg in words]
            print(*uppercased)


        if __name__ == "__main__":
            main()

    Observa cómo en lugar de usar `map`, escribimos una expresión en Python dentro de corchetes. Para cada argumento, se aplica el método `.upper`.

-   Llevando este concepto un paso más allá, pivotemos hacia otro programa.
-   En la ventana del editor de texto, escribe `code gryffindors.py` y el código es el siguiente:

        students = [
            {"name": "Hermione", "house": "Gryffindor"},
            {"name": "Harry", "house": "Gryffindor"},
            {"name": "Ron", "house": "Gryffindor"},
            {"name": "Draco", "house": "Slytherin"},
        ]

        gryffindors = []
        for student in students:
            if student["house"] == "Gryffindor":
                gryffindors.append(student["name"])

        for gryffindor in sorted(gryffindors):
            print(gryffindor)

    Observa que tenemos una condición mientras creamos nuestra lista. _Si_ la casa del estudiante es Gryffindor, agregamos el nombre del estudiante a la lista. Finalmente, imprimimos todos los nombres.

-   De manera más elegante, podemos simplificar este código con una comprensión de lista de la siguiente manera:

        students = [
            {"name": "Hermione", "house": "Gryffindor"},
            {"name": "Harry", "house": "Gryffindor"},
            {"name": "Ron", "house": "Gryffindor"},
            {"name": "Draco", "house": "Slytherin"},
        ]

        gryffindors = [
            student["name"] for student in students if student["house"] == "Gryffindor"
        ]

        for gryffindor in sorted(gryffindors):
            print(gryffindor)

    ¡Observa cómo la comprensión de la lista está en una sola línea!

## `filter` <a id="filter"></a>

-   Usar la función `filter` de Python nos permite devolver un subconjunto de una secuencia para la cual se cumple una cierta condición.
-   En la ventana del editor de texto, escribe el siguiente código:

        students = [
            {"name": "Hermione", "house": "Gryffindor"},
            {"name": "Harry", "house": "Gryffindor"},
            {"name": "Ron", "house": "Gryffindor"},
            {"name": "Draco", "house": "Slytherin"},
        ]


        def is_gryffindor(s):
            return s["house"] == "Gryffindor"


        gryffindors = filter(is_gryffindor, students)

        for gryffindor in sorted(gryffindors, key=lambda s: s["name"]):
            print(gryffindor["name"])

    Observa cómo se crea una función llamada `is_gryffindor`. Esta es nuestra función de filtrado que tomará un estudiante `s` y devolverá `True` o `False` dependiendo de si la casa del estudiante es Gryffindor. Puedes ver que la nueva función `filter` toma dos argumentos. Primero, toma la función que se aplicará a cada elemento de una secuencia, en este caso, `is_gryffindor`. Segundo, toma la secuencia a la cual se aplicará la función de filtrado, en este caso, `students`. En `gryffindors`, deberíamos ver solo aquellos estudiantes que están en Gryffindor.

-   `filter` también puede usar funciones lambda de la siguiente manera:

        students = [
            {"name": "Hermione", "house": "Gryffindor"},
            {"name": "Harry", "house": "Gryffindor"},
            {"name": "Ron", "house": "Gryffindor"},
            {"name": "Draco", "house": "Slytherin"},
        ]


        gryffindors = filter(lambda s: s["house"] == "Gryffindor", students)

        for gryffindor in sorted(gryffindors, key=lambda s: s["name"]):
            print(gryffindor["name"])

    Observa cómo se proporciona la misma lista de estudiantes.

-   Puedes obtener más información en la documentación de Python sobre [`filter`](https://docs.python.org/3/library/functions.html#filter).

## Comprensiones de diccionarios <a id="comprensiones-de-diccionarios"></a>

-   Podemos aplicar la misma idea detrás de las comprensiones de listas a los diccionarios. En la ventana del editor de texto, escribe el siguiente código:

        students = ["Hermione", "Harry", "Ron"]

        gryffindors = []

        for student in students:
            gryffindors.append({"name": student, "house": "Gryffindor"})

        print(gryffindors)

    Observa cómo este código todavía no usa ninguna comprensión. En cambio, sigue los mismos paradigmas que hemos visto antes.

-   Ahora podemos aplicar comprensiones de diccionarios modificando nuestro código de la siguiente manera:

        students = ["Hermione", "Harry", "Ron"]

        gryffindors = [{"name": student, "house": "Gryffindor"} for student in students]

        print(gryffindors)

    Observa cómo todo el código anterior se simplifica en una sola línea donde se proporciona la estructura del diccionario para cada `student` en `students`.

-   Incluso podemos simplificar aún más de la siguiente manera:

        students = ["Hermione", "Harry", "Ron"]

        gryffindors = {student: "Gryffindor" for student in students}

        print(gryffindors)

    Observa cómo el diccionario se construirá con pares clave-valor.

## `enumerate` <a id="enumerate"></a>

-   Podemos desear proporcionar una clasificación de cada estudiante. En la ventana del editor de texto, codifica de la siguiente manera:

        students = ["Hermione", "Harry", "Ron"]

        for i in range(len(students)):
            print(i + 1, students[i])

    Observa cómo se enumeran los estudiantes al ejecutar este código.

-   Utilizando la enumeración, podemos hacer lo mismo:

        students = ["Hermione", "Harry", "Ron"]

        for i, student in enumerate(students):
            print(i + 1, student)

    Observa cómo la enumeración presenta el índice y el valor de cada `student`.

-   Puedes obtener más información en la documentación de Python sobre [`enumerate`](https://docs.python.org/3/library/functions.html#enumerate).

## Generadores e Iteradores <a id="generadores-e-iteradores"></a>

-   En Python, hay una forma de protegerse contra que su sistema se quede sin recursos cuando los problemas que están abordando se vuelven demasiado grandes.
-   En los Estados Unidos, es costumbre "contar ovejas" en la mente de uno cuando uno tiene dificultades para conciliar el sueño.
-   En la ventana del editor de texto, escriba `code sleep.py` y el código de la siguiente manera:

        n = int(input("¿Cuál es n? "))
        for i in range(n):
            print("🐑" * i)

    Note cómo este programa contará el número de ovejas que le solicite.

-   Podemos hacer nuestro programa más sofisticado agregando una función `main` de la siguiente manera:

        def main():
            n = int(input("¿Cuál es n? "))
            for i in range(n):
                print("🐑" * i)


        if __name__ == "__main__":
            main()

    Note cómo se proporciona una función `main`.

-   Hemos estado adquiriendo el hábito de abstraer partes de nuestro código.
-   Podemos llamar a una función de ovejas modificando nuestro código de la siguiente manera:

        def main():
            n = int(input("¿Cuál es n? "))
            for i in range(n):
                print(sheep(i))


        def sheep(n):
            return "🐑" * n


        if __name__ == "__main__":
            main()

    Observe cómo la función `main` realiza la iteración.

-   Podemos dotar a la función `sheep` de más habilidades. En la ventana del editor de texto, codifique de la siguiente manera:

        def main():
            n = int(input("¿Cuál es n? "))
            for s in sheep(n):
                print(s)


        def sheep(n):
            flock = []
            for i in range(n):
                flock.append("🐑" * i)
            return flock


        if __name__ == "__main__":
            main()

    Observe cómo creamos un rebaño de ovejas y devolvemos el `rebaño`.

-   Ejecutando nuestro código, puedes probar diferentes números de ovejas como `10`, `1000` y `10000`. ¿Qué pasa si solicitas `1000000` ovejas? Es posible que tu programa se cuelgue o se bloquee por completo. Debido a que has intentado generar una lista masiva de ovejas, tu computadora puede tener dificultades para completar el cálculo.
-   El generador `yield` puede resolver este problema devolviendo un pequeño fragmento de los resultados a la vez. En la ventana del editor de texto, codifique de la siguiente manera:

        def main():
            n = int(input("¿Cuál es n? "))
            for s in sheep(n):
                print(s)


        def sheep(n):
            for i in range(n):
                yield "🐑" * i


        if __name__ == "__main__":
            main()

    Note cómo `yield` proporciona solo un valor a la vez y el bucle `for` sigue funcionando.

-   Puedes obtener más información en la documentación de Python sobre [generadores](https://docs.python.org/3/howto/functional.html#generators).
-   Puedes obtener más información en la documentación de Python sobre [iteradores](https://docs.python.org/3/howto/functional.html#iterators).
