## Ponteiros

-   Da última vez, aprendemos sobre ponteiros, `malloc` e outras ferramentas úteis para trabalhar com memória.
-   Vamos revisar esse trecho de código:

        int main(void)
        {
            int *x;
            int *y;

            x = malloc(sizeof(int));

            *x = 42;
            *y = 13;
        }

    -   Aqui, as primeiras duas linhas de código em nossa função `main` estão declarando dois ponteiros, `x` e `y`. Em seguida, alocamos memória suficiente para um `int` com `malloc` e armazenamos o endereço retornado pelo `malloc` em `x`.
    -   Com `*x = 42;`, vamos para o endereço apontado por `x` e armazenamos o valor `42` nessa localização.
    -   A última linha, porém, possui um erro, pois não sabemos qual é o valor de `y`, já que nunca definimos um valor para ele. Em vez disso, podemos escrever:

            y = x;
            *y = 13;

        -   E isso definirá que `y` aponta para o mesmo local que `x`, e então definirá aquele valor como `13`.

-   Vamos dar uma olhada em uma pequena animação, [Pointer Fun with Binky](https://www.youtube.com/watch?v=3uLKjb973HU), que também explica esse trecho de código de forma animada!

## Redimensionamento de arrays

- Na semana 2, aprendemos sobre arrays, onde podemos armazenar o mesmo tipo de valor em uma lista, lado a lado. Porém, precisamos declarar o tamanho dos arrays quando os criamos e, quando queremos aumentar o tamanho do array, a memória ao seu redor pode ser ocupada por outros dados.
- Uma solução pode ser alocar mais memória em uma área maior que esteja livre e mover nosso array para lá, onde ele terá mais espaço. Porém, precisaremos copiar nosso array, o que se torna uma operação com tempo de execução _O_(_n_), já que precisamos copiar cada um dos _n_ elementos do array.
- Podemos escrever um programa como o seguinte para fazer isso em código:

```c
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    // Aqui, alocamos memória suficiente para comportar três inteiros, e nossa variável
    // list apontará para o primeiro inteiro.
    int *list = malloc(3 * sizeof(int));
    // Devemos verificar se alocamos memória corretamente, já que malloc pode
    // falhar em nos fornecer memória livre suficiente.
    if (list == NULL)
    {
        return 1;
    }

    // Com essa sintaxe, o compilador fará a aritmética de ponteiros para nós e
    // calculará o byte na memória para o qual list[0], list[1] e list[2] apontam,
    // já que inteiros possuem 4 bytes de tamanho.
    list[0] = 1;
    list[1] = 2;
    list[2] = 3;

    // Agora, se quisermos redimensionar nosso array para comportar 4 inteiros, tentaremos alocar
    // memória suficiente para eles e temporariamente usaremos tmp para apontar para o primeiro elemento:
    int *tmp = malloc(4 * sizeof(int));
    if (tmp == NULL)
    {
        return 1;
    }

    // Agora, copiamos os inteiros do array antigo para o novo array ...
    for (int i = 0; i < 3; i++)
    {
        tmp[i] = list[i];
    }

    // ... e adicionamos o quarto inteiro:
    tmp[3] = 4;

    // Devemos liberar a memória original para list, é por isso que precisamos de uma
    // variável temporária para apontar para o novo array ...
    free(list);

    // ... e agora podemos definir nossa variável list para apontar para o novo array para o qual
    // tmp aponta:
    list = tmp;

    // Agora, podemos imprimir o novo array:
    for (int i = 0; i < 4; i++)
    {
        printf("%i\n", list[i]);
    }

    // E, finalmente, liberamos a memória para o novo array.
    free(list);
}
```

- Acontece que existe uma função útil chamada `realloc`, que nos permite realocar memória:

```c
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    int *list = malloc(3 * sizeof(int));
    if (list == NULL)
    {
        return 1;
    }

    list[0] = 1;
    list[1] = 2;
    list[2] = 3;

    // Aqui, passamos para realloc o nosso array original para o qual list aponta,
    // e ele retornará um novo endereço para um novo array, com os dados antigos copiados:
    int *tmp = realloc(list, 4 * sizeof(int));
    if (tmp == NULL)
    {
        return 1;
    }
    // Agora, tudo que precisamos fazer é lembrar da localização do novo array:
    list = tmp;

    list[3] = 4;

    for (int i = 0; i < 4; i++)
    {
        printf("%i\n", list[i]);
    }

    free(list);
}
```

## Estruturas de dados

-   **Estruturas de dados** são construções de programação que nos permitem armazenar informações em diferentes formatos na memória do nosso computador.
-   Para construir uma estrutura de dados, precisaremos de algumas ferramentas que já vimos:
    -   `struct` para criar tipos de dados personalizados
    -   `.` para acessar propriedades em uma estrutura
    -   `*` para ir a um endereço na memória apontado por um ponteiro

## Listas Encadeadas

- Com uma **lista encadeada**, podemos armazenar uma lista de valores que pode ser facilmente expandida ao armazenar valores em partes diferentes da memória:
    ![grade representando a memória, com três dos espaços rotulados com espaços vazios entre eles, cada um rotulado com 1 0x123, 2 0x456 e 3 0x789](https://cs50.harvard.edu/x/2020/notes/5/linked_list.png)
    - Isso é diferente de um array, pois nossos valores não estão mais próximos uns dos outros na memória.
- Podemos ligar nossa lista alocando, para cada elemento, memória suficiente tanto para o valor que queremos armazenar quanto para o endereço do próximo elemento:
    ![três caixas, cada uma dividida em duas e rotulada (1 0x123 e 0x456), (2 0x456 e 0x789) e (3 0x789 e NULL)](https://cs50.harvard.edu/x/2020/notes/5/linked_list_with_addresses.png)
    - A propósito, `NULL` se refere a `\0`, um caractere que termina uma string, e `NULL` se refere a um endereço de zeros ou um ponteiro nulo, que podemos considerar como apontando para nenhum lugar.
- Ao contrário do que podemos fazer com arrays, não podemos mais acessar elementos aleatoriamente em uma lista encadeada. Por exemplo, não podemos mais acessar o quinto elemento da lista calculando onde ele está, em tempo constante. (Já que sabemos que arrays armazenam elementos lado a lado, podemos adicionar 1, ou 4, ou o tamanho do nosso elemento, para calcular os endereços.) Em vez disso, precisamos seguir o ponteiro de cada elemento, um de cada vez. E precisamos alocar o dobro da memória do que precisávamos antes para cada elemento.
- No código, podemos criar nossa própria estrutura chamada `node` (como um nó de um grafo em matemática), e precisamos armazenar tanto um `int` quanto um ponteiro para o próximo `node` chamado `next`:

```c
typedef struct node
{
    int number;
    struct node *next;
}
node;
```

- Começamos essa estrutura com `typedef struct node` para que possamos nos referir a um `node` dentro de nossa estrutura.

- Podemos construir uma lista encadeada no código começando com nossa estrutura. Primeiro, vamos lembrar uma lista vazia, para que possamos usar o ponteiro nulo: `node *list = NULL;`.
- Para adicionar um elemento, primeiro precisamos alocar alguma memória para um nó e definir seus valores:

```c
node *n = malloc(sizeof(node));
// Queremos ter certeza de que o malloc obteve memória para nós:
if (n != NULL)
{
    // Isso é equivalente a (*n).number, onde vamos primeiro ao nó apontado
    // por n e, em seguida, definimos a propriedade number. Em C, também podemos usar essa
    // notação de seta:
    n->number = 2;
    // Em seguida, precisamos armazenar um ponteiro para o próximo nó em nossa lista, mas o
    // novo nó não apontará para nada (por enquanto):
    n->next = NULL;
}
```

- Agora nossa lista pode apontar para este nó: `list = n;`:
    ![uma caixa rotulada lista com seta para fora apontando para duas caixas conectadas, uma com 2 e uma vazia)](https://cs50.harvard.edu/x/2020/notes/5/list_with_one_node.png)
- Para adicionar à lista, criaremos um novo nó da mesma maneira, talvez com o valor 4. Mas agora precisamos atualizar o ponteiro em nosso primeiro nó para apontar para ele.
- Como nosso ponteiro `list` aponta apenas para o primeiro nó (e não podemos ter certeza de que a lista tem apenas um nó), precisamos "seguir as migalhas de pão" e seguir o próximo ponteiro de cada nó:

```c
// Cria um ponteiro temporário para o que a lista está apontando
node *tmp = list;
// Enquanto o nó tiver um próximo ponteiro ...
while (tmp->next != NULL)
{
    // ... defina o temporário para o próximo nó
    tmp = tmp->next;
}
// Agora, tmp aponta para o último nó em nossa lista, e podemos atualizar seu próximo
// ponteiro para apontar para nosso novo nó.
```

- Se quisermos inserir um nó no início de nossa lista encadeada, precisamos atualizar cuidadosamente nosso nó para apontar para o seguinte, antes de atualizar a lista. Caso contrário, perderemos o restante de nossa lista:

```c
// Aqui, estamos inserindo um nó na frente da lista, então queremos que o
// próximo ponteiro aponte para a lista original, antes de apontar a lista para
// n:
n->next = list;
list = n;
```

- E para inserir um nó no meio de nossa lista, podemos percorrer a lista, seguindo cada elemento um de cada vez, comparando seus valores e alterando cuidadosamente os ponteiros `next` também.
- Com alguns voluntários no palco, simulamos uma lista, com cada voluntário atuando como a variável `list` ou um nó. Ao inserir nós na lista, precisamos de um ponteiro temporário para seguir a lista e garantir que não percamos nenhuma parte de nossa lista. Nossa lista encadeada aponta apenas para o primeiro nó em nossa lista, então só podemos olhar para um nó de cada vez, mas podemos alocar mais memória dinamicamente à medida que precisamos expandir nossa lista.
- Agora, mesmo que nossa lista encadeada esteja ordenada, o tempo de execução da pesquisa será _O_(_n_), já que precisamos seguir cada nó para verificar seus valores e não sabemos onde estará o meio da nossa lista.
- Podemos combinar todos os nossos trechos de código em um programa completo:

```c
#include <stdio.h>
#include <stdlib.h>

// Representa um nó
typedef struct node
{
    int number;
    struct node *next;
}
node;

int main(void)
{
    // Lista de tamanho 0, inicialmente não apontando para nada
    node *list = NULL;

    // Adicionar número à lista
    node *n = malloc(sizeof(node));
    if (n == NULL)
    {
        return 1;
    }
    n->number = 1;
    n->next = NULL;
    // Criamos nosso primeiro nó, armazenamos o valor 1 nele e deixamos o próximo
    // ponteiro apontar para nada. Então, nossa variável lista pode apontar para ele.
    list = n;

    // Adicionar número à lista
    n = malloc(sizeof(node));
    if (n == NULL)
    {
        return 1;
    }
    n->number = 2;
    n->next = NULL;
    // Agora, vamos ao nosso primeiro nó que a lista aponta e definimos o próximo ponteiro
    // para apontar para nosso novo nó, adicionando-o ao final da lista:
    list->next = n;

    // Adicionar número à lista
    n = malloc(sizeof(node));
    if (n == NULL)
    {
        return 1;
    }
    n->number = 3;
    n->next = NULL;
    // Podemos seguir vários nós com essa sintaxe, usando o próximo ponteiro
    // repetidamente, para adicionar nosso terceiro novo nó ao final da lista:
    list->next->next = n;
    // Normalmente, porém, queremos um loop e uma variável temporária para adicionar
    // um novo nó à nossa lista.

    // Imprimir lista
    // Aqui podemos iterar sobre todos os nós em nossa lista com uma variável temporária.
    // Primeiro, temos um ponteiro temporário, tmp, que aponta para a lista.
    // Em seguida, nossa condição para continuar é que tmp não seja NULL e,
    // finalmente, atualizamos tmp para o próximo ponteiro de si mesmo.
    for (node *tmp = list; tmp != NULL; tmp = tmp->next)
    {
        // Dentro do nó, vamos apenas imprimir o número armazenado:
        printf("%i\n", tmp->number);
    }

    // Liberar lista
    // Como estamos liberando cada nó à medida que avançamos, usaremos um loop while e
    // siga o próximo ponteiro de cada nó antes de liberá-lo, mas veremos isso com mais detalhes no Problem Set 5.
    while (list != NULL)
    {
        node *tmp = list->next;
        free(list);
        list = tmp;
    }
}
```

## Mais estruturas de dados

- Uma **árvore** é outra estrutura de dados na qual cada nó aponta para outros dois nós, um à esquerda (com um valor menor) e um à direita (com um valor maior):
![árvore com o nó 4 no topo central, seta esquerda para 3 abaixo, seta direita para 6 abaixo; 2 tem seta esquerda para 1 abaixo, seta direita para 3 abaixo; 6 tem seta esquerda para 5 abaixo, seta direita para 7 abaixo](https://cs50.harvard.edu/x/2020/notes/5/binary_search_tree.png)
    - Note que agora existem duas dimensões nesta estrutura de dados, onde alguns nós estão em diferentes "níveis" do que outros. Podemos imaginar a implementação disso com uma versão mais complexa de um nó em uma lista encadeada, onde cada nó possui não uma, mas duas referências, uma para o valor na "metade esquerda" e outra para o valor na "metade direita". E todos os elementos à esquerda de um nó são menores, e todos os elementos à direita são maiores.
    - Isso é chamado de árvore de busca binária porque cada nó tem no máximo dois filhos, ou nós para os quais está apontando, e uma árvore de busca porque é ordenada de forma que nos permite fazer buscas corretamente.
    - E, assim como em uma lista encadeada, vamos querer manter um ponteiro apenas para o início da lista, mas neste caso queremos apontar para a raiz, ou o nó central superior da árvore (o 4).
- Agora, podemos facilmente fazer uma busca binária e, como cada nó está apontando para outro, também podemos inserir nós na árvore sem mover todos eles como teríamos que fazer em um array. A busca recursiva nesta árvore seria algo assim:

```c
typedef struct node
{
    int number;
    struct node *left;
    struct node *right;
} node;

// Aqui, *tree é um ponteiro para a raiz de nossa árvore.
bool search(node *tree)
{
    // Precisamos de um caso base, se a árvore atual (ou parte da árvore) for NULL,
    // para retornar false:
    if (tree == NULL)
    {
        return false;
    }
    // Agora, dependendo se o número no nó atual é maior ou menor,
    // podemos apenas olhar para o lado esquerdo ou direito da árvore:
    else if (50 < tree->number)
    {
        return search(tree->left);
    }
    else if (50 > tree->number)
    {
        return search(tree->right);
    }
    // Caso contrário, o número deve ser igual ao que estamos procurando:
    else {
        return true;
    }
}
```

- O tempo de execução da busca em uma árvore é _O_(log _n_), e a inserção de nós mantendo a árvore equilibrada também é _O_(log _n_). Ao gastar um pouco mais de memória e tempo para manter a árvore, agora temos uma busca mais rápida em comparação com uma simples lista encadeada.
- Uma estrutura de dados com busca quase em tempo constante é uma **tabela hash**, que é uma combinação de um array e uma lista encadeada. Temos um array de listas encadeadas, e cada lista encadeada no array tem elementos de uma determinada categoria. Por exemplo, no mundo real, podemos ter muitos crachás e podemos classificá-los em 26 recipientes, um rotulado com cada letra do alfabeto, para que possamos encontrar crachás procurando apenas em um recipiente.
- Podemos implementar isso em uma tabela hash com um array de 26 ponteiros, cada um apontando para uma lista encadeada para uma letra do alfabeto:
![array vertical com 26 caixas, a primeira com uma seta apontando para uma caixa rotulada Albus, a segunda vazia, a terceira com uma seta apontando para uma caixa rotulada Cedric ... a sétima com uma seta apontando para uma caixa rotulada Ginny, com uma seta dessa caixa apontando para uma caixa rotulada George ...](https://cs50.harvard.edu/x/2020/notes/5/hash_table.png)
- Como temos acesso aleatório com arrays, podemos adicionar elementos rapidamente e também indexar rapidamente um recipiente.
- Um recipiente pode ter vários valores correspondentes, então usaremos uma lista encadeada para armazenar todos eles horizontalmente. (Chamamos isso de colisão, quando dois valores correspondem de alguma maneira.)
- Isso é chamado de tabela hash porque usamos uma função hash, que pega uma entrada e a mapeia para um recipiente em que deve ser inserido. Em nosso exemplo, a função hash está apenas olhando para a primeira letra do nome, então pode retornar `0` para "Albus" e `25` para "Zacharias".
- Mas no pior caso, todos os nomes podem começar com a mesma letra, então podemos acabar com o equivalente a uma única lista encadeada novamente. Podemos olhar para as duas primeiras letras, e alocar recipiente suficiente para 26\*26 valores hash possíveis, ou mesmo olhar para as três primeiras letras, e agora precisaremos de 26\*26\*26 recipientes. Mas ainda podemos ter um pior caso onde todos os nossos valores começam com os mesmos três caracteres, então o tempo de execução para busca é _O_(_n_). Na prática, porém, podemos chegar mais perto de _O_(1) se tivermos aproximadamente o mesmo número de recipientes que os valores possíveis, especialmente se tivermos uma função de hash ideal, onde podemos classificar nossas entradas em recipientes únicos.
- Podemos usar outra estrutura de dados chamada **trie** (pronunciada como "try", e é uma abreviação de "retrieval"):
![array com letras de A-Z em 26 elementos, com H apontando para outro array com todas as 26 letras. O A e o E deste array apontam cada um para mais dois arrays de todas as 26 letras, e isso continua em uma árvore até que os arrays mais inferiores tenham apenas uma letra marcada como válida](https://cs50.harvard.edu/x/2020/notes/5/trie.png)
    - Imagine que queremos armazenar um dicionário de palavras de forma eficiente e ser capazes de acessar cada uma em tempo constante. Um trie é como uma árvore, mas cada nó é um array. Cada array terá cada letra, de A a Z, armazenada. Para cada palavra, a primeira letra apontará para um array, onde a próxima letra válida apontará para outro array, e assim por diante, até chegarmos a algo que indique o final de uma palavra válida. Se nossa palavra não estiver no trie, então um dos arrays não terá um ponteiro ou caractere de término para nossa palavra. Agora, mesmo que nossa estrutura de dados tenha muitas palavras, o tempo de busca será apenas o comprimento da palavra que estamos procurando, e isso pode ser um máximo fixo, então temos _O_(1) para busca e inserção. O custo disso, porém, é 26 vezes mais memória do que precisamos para cada caractere.
- Existem até mesmo construções de nível mais alto, **estruturas de dados abstratas**, onde usamos nossos blocos de construção de arrays, listas encadeadas, tabelas hash e tries para implementar uma solução para algum problema.
- Por exemplo, uma estrutura de dados abstrata é uma **fila**, onde queremos poder adicionar valores e remover valores em uma ordem de entrada-saída (FIFO). Para adicionar um valor, podemos colocá-lo na fila (enqueue), e para remover um valor, desenfileiramos (dequeue). E podemos implementar isso com um array que redimensionamos conforme adicionamos itens, ou com uma lista encadeada onde anexamos os valores ao final.
- Uma estrutura de dados "oposta" seria uma **pilha**, onde os itens adicionados mais recentemente (empilhados) são removidos (desempilhados) primeiro, de uma maneira de último a entrar, primeiro a sair (LIFO). A caixa de entrada de e-mail é uma pilha, onde nossos e-mails mais recentes estão no topo.
- Outro exemplo é um **dicionário**, onde podemos mapear chaves para valores, ou strings para valores, e podemos implementar um com uma tabela hash onde uma palavra vem com alguma outra informação (como sua definição ou significado).
