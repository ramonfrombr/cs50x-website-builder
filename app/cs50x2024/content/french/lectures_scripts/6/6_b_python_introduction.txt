Alors comment arriver à ce point en fin de compte ? Eh bien, rappelez-vous qu'en C, nous avions ce processus de compilation de notre code puis de son exécution, comme avec make ou plus spécifiquement, comme avec clang, puis de son exécution avec le fichier ./hello, représentant un fichier dans votre répertoire de travail actuel.

Aujourd'hui, même ce processus devient un peu plus facile dans la mesure où il ne s'agit plus d'un processus en deux étapes pour écrire et exécuter du code. Il n'y en a plus qu'un. Mais c'est un peu différent du passé, alors que dans le passé, nous avons en effet compilé notre code à partir du code source en code machine, puis fait ./ afin de l'exécuter. Tout comme sur un Mac ou un PC, vous double-cliqueriez sur une icône, Python est utilisé un peu différemment. Et d'autres langages sont utilisés de la même manière également.

Vous n'exécutez pas les programmes directement en soi. Au lieu de cela, littéralement, à partir d'aujourd'hui, vous exécutez un programme qui s'appelle lui-même Python. Et vous lui transmettez en entrée le nom du fichier contenant votre code source. Donc Python lui-même est le programme. Il prend en charge les arguments de ligne de commande.

Et l'un de ces arguments peut être le nom de votre programme même, ce qui signifie que nous n'avons pas à compiler et recompiler notre code de manière très ennuyeuse chaque fois que nous apportons une modification. Si vous souhaitez modifier votre code, il vous suffit de sauvegarder votre fichier et de réexécuter cette commande.
