## Testes Unitários <a id="unit-tests"></a>

- Até agora, é provável que você esteja testando seu próprio código usando instruções `print`.
- Alternativamente, você pode ter contado com o CS50 para testar seu código para você!
- É mais comum na indústria escrever código para testar seus próprios programas.
- Na janela do console, digite `code calculator.py`. Observe que você pode ter codificado este arquivo em uma palestra anterior. No editor de texto, verifique se o código aparece da seguinte forma:

      def main():
          x = int(input("Qual é o valor de x? "))
          print("x ao quadrado é", square(x))


      def square(n):
          return n * n


      if __name__ == "__main__":
          main()

  Observe que você poderia testar o código acima por conta própria usando alguns números óbvios, como `2`. No entanto, pense por que você pode querer criar um teste que garanta que o código acima funcione adequadamente.

- Seguindo a convenção, vamos criar um novo programa de testes digitando `code test_calculator.py` e modifique seu código no editor de texto da seguinte maneira:

      from calculator import square


      def main():
          test_square()


      def test_square():
          if square(2) != 4:
              print("2 ao quadrado não é 4")
          if square(3) != 9:
              print("3 ao quadrado não é 9")


      if __name__ == "__main__":
          main()

  Observe que estamos importando a função `square` do arquivo `square.py` na primeira linha de código. Por convenção, estamos criando uma função chamada `test_square`. Dentro dessa função, definimos algumas condições para testar.

- Na janela do console, digite `python test_calculator.py`. Você notará que nada está sendo exibido. Pode ser que tudo esteja funcionando corretamente! Alternativamente, pode ser que nossa função de teste não tenha descoberto um dos "casos extremos" que poderiam produzir um erro.
- No momento, nosso código testa duas condições. Se quiséssemos testar muitas mais condições, nosso código de teste poderia facilmente ficar confuso. Como poderíamos expandir nossas capacidades de teste sem expandir nosso código de teste?

## `assert` <a id="assert"></a>

- O comando `assert` do Python nos permite dizer ao compilador que algo, alguma afirmação, é verdadeira. Podemos aplicar isso ao nosso código de teste da seguinte forma:

      from calculator import square


      def main():
          test_square()


      def test_square():
          assert square(2) == 4
          assert square(3) == 9


      if __name__ == "__main__":
          main()

  Observe que estamos afirmando de forma definitiva o que `square(2)` e `square(3)` devem ser iguais. Nosso código foi reduzido de quatro linhas de teste para duas.

- Podemos quebrar intencionalmente nosso código da calculadora modificando-o da seguinte forma:

      def main():
          x = int(input("Qual é o valor de x? "))
          print("O quadrado de x é", square(x))


      def square(n):
          return n + n


      if __name__ == "__main__":
          main()

  Observe que mudamos o operador `*` para `+` na função `square`.

- Agora, ao executar `python test_square.py` no console, você perceberá que uma `AssertionError` é gerada pelo compilador. Essencialmente, isso significa que uma de nossas condições não foi atendida.
- Um dos desafios que estamos enfrentando agora é que nosso código pode se tornar ainda mais trabalhoso se quisermos fornecer mensagens de erro mais descritivas aos nossos usuários. Poderíamos codificar da seguinte forma:

      from calculator import square


      def main():
          test_square()


      def test_square():
          try:
              assert square(2) == 4
          except AssertionError:
              print("2 ao quadrado não é 4")
          try:
              assert square(3) == 9
          except AssertionError:
              print("3 ao quadrado não é 9")
          try:
              assert square(-2) == 4
          except AssertionError:
              print("-2 ao quadrado não é 4")
          try:
              assert square(-3) == 9
          except AssertionError:
              print("-3 ao quadrado não é 9")
          try:
              assert square(0) == 0
          except AssertionError:
              print("0 ao quadrado não é 0")


      if __name__ == "__main__":
          main()

  Observe que a execução deste código produzirá vários erros. No entanto, ele não está produzindo todos os erros mencionados anteriormente. Isso ilustra que vale a pena testar vários casos para capturar situações em que haja erros de codificação.

- O código acima ilustra um grande desafio: como podemos tornar mais fácil testar nosso código sem dezenas de linhas de código como as acima?

Você pode aprender mais na documentação do Python sobre o `assert` [`aqui`](https://docs.python.org/3/reference/simple_stmts.html#assert).

## `pytest` <a id="pytest"></a>

- `pytest` é uma biblioteca de terceiros que permite testar unitariamente seu programa. Ou seja, você pode testar suas funções dentro do seu programa.
- Para utilizar `pytest`, digite `pip install pytest` no seu console.
- Antes de aplicar o `pytest` ao nosso próprio programa, modifique sua função `test_calculator` da seguinte forma:

      from calculator import square


      def test_assert():
          assert square(2) == 4
          assert square(3) == 9
          assert square(-2) == 4
          assert square(-3) == 9
          assert square(0) == 0

  Observe como o código acima afirma todas as condições que queremos testar.

- `pytest` nos permite executar nosso programa diretamente através dele, de forma que podemos visualizar mais facilmente os resultados das nossas condições de teste.
- Na janela do terminal, digite `pytest test_calculator.py`. Você imediatamente perceberá que a saída será fornecida. Observe o `F` vermelho próximo ao topo da saída, indicando que algo no seu código falhou. Além disso, observe que o `E` vermelho fornece algumas pistas sobre os erros no seu programa `calculator.py`. Com base na saída, você pode imaginar um cenário em que `3 * 3` tenha fornecido uma saída `6` em vez de `9`. Com base nos resultados deste teste, podemos corrigir nosso código `calculator.py` da seguinte forma:

      def main():
          x = int(input("Qual é o valor de x? "))
          print("O quadrado de x é", square(x))


      def square(n):
          return n * n


      if __name__ == "__main__":
          main()

  Observe que mudamos o operador `+` para um `*` na função square, retornando-a para um estado funcional.

- Ao executar novamente `pytest test_calculator.py`, observe como nenhum erro é produzido. Parabéns!
- No momento, não é ideal que o `pytest` interrompa a execução após o primeiro teste falhado. Novamente, vamos retornar nosso código `calculator.py` ao seu estado quebrado:

      def main():
          x = int(input("Qual é o valor de x? "))
          print("O quadrado de x é", square(x))


      def square(n):
          return n + n


      if __name__ == "__main__":
          main()

  Observe que mudamos o operador `*` para um `+` na função square, retornando-a para um estado quebrado.

- Para melhorar nosso código de teste, vamos modificar `test_calculator.py` para dividir o código em diferentes grupos de testes:

      from calculator import square


      def test_positive():
          assert square(2) == 4
          assert square(3) == 9


      def test_negative():
          assert square(-2) == 4
          assert square(-3) == 9


      def test_zero():
          assert square(0) == 0

  Observe que dividimos os mesmos cinco testes em três funções diferentes. Frameworks de teste como o `pytest` executarão cada função, mesmo que haja falhas em uma delas. Ao executar novamente `pytest test_calculator.py`, você perceberá que muitos mais erros são exibidos. A saída de erro adicional permite explorar ainda mais o que pode estar causando problemas no seu código.

- Após melhorar nosso código de teste, retorne o código `calculator.py` para um estado totalmente funcional:

      def main():
          x = int(input("Qual é o valor de x? "))
          print("O quadrado de x é", square(x))


      def square(n):
          return n * n


      if __name__ == "__main__":
          main()

  Observe que mudamos o operador `+` para um `*` na função square, retornando-a para um estado funcional.

- Ao executar novamente `pytest test_calculator.py`, você perceberá que nenhum erro é encontrado.
- Em resumo, cabe a você, como um programador, definir quantas condições de teste considerar adequadas!

Você pode aprender mais na documentação do Pytest em [`pytest`](https://docs.pytest.org/en/7.1.x/getting-started.html).

## Testando Strings <a id="testing-strings"></a>

- Voltando no tempo, considere o seguinte código `hello.py`:

      def main():
          name = input("Qual é o seu nome? ")
          hello(name)


      def hello(to="mundo"):
          print("olá,", to)


      if __name__ == "__main__":
          main()

  Observe que podemos querer testar o resultado da função `hello`.

- Considere o seguinte código para `test_hello.py`:

      from hello import hello


      def test_hello():
          assert hello("David") == "olá, David"
          assert hello() == "olá, mundo"

  Analisando este código, você acha que essa abordagem de teste funcionará bem? Por que esse teste pode não funcionar bem? Observe que a função `hello` em `hello.py` imprime algo: ou seja, ela não retorna um valor!

- Podemos alterar nossa função `hello` dentro de `hello.py` da seguinte forma:

      def main():
          name = input("Qual é o seu nome? ")
          print(hello(name))


      def hello(to="mundo"):
          return f"olá, {to}"


      if __name__ == "__main__":
          main()

  Observe que alteramos nossa função `hello` para retornar uma string. Isso significa efetivamente que agora podemos usar o `pytest` para testar a função `hello`.

- Executando `pytest test_hello.py`, nosso código passará todos os testes!
- Assim como no nosso caso de teste anterior nesta lição, podemos separar nossos testes separadamente:

      from hello import hello


      def test_default():
          assert hello() == "olá, mundo"


      def test_argument():
          assert hello("David") == "olá, David"

  Observe que o código acima separa nosso teste em várias funções para que todos eles sejam executados, mesmo se um erro for produzido.

## Organizando testes em pastas <a id="organizando-testes-em-pastas"></a>

- É tão comum testar o código usando vários testes que você tem a capacidade de executar uma pasta inteira de testes com um único comando.
- Primeiro, na janela do terminal, execute `mkdir test` para criar uma pasta chamada `test`.
- Então, para criar um teste dentro dessa pasta, digite na janela do terminal `code test/test_hello.py`. Observe que `test/` instrui o terminal a criar `test_hello.py` na pasta chamada `test`.
- Na janela do editor de texto, modifique o arquivo para incluir o seguinte código:

      from hello import hello


      def test_default():
          assert hello() == "olá, mundo"


      def test_argument():
          assert hello("David") == "olá, David"

  Observe que estamos criando um teste da mesma forma que fizemos antes.

- O `pytest` não permitirá que executemos testes como uma pasta apenas com este arquivo (ou um conjunto inteiro de arquivos) sozinho sem um arquivo especial `__init__`. Na janela do seu terminal, crie este arquivo digitando `code test/__init__.py`. Note o `test/` como antes, assim como os underscores duplos em ambos os lados de `init`. Mesmo deixando este arquivo `__init__.py` vazio, o `pytest` é informado de que a pasta inteira que contém o `__init__.py` possui testes que podem ser executados.
- Agora, ao digitar `pytest test` no terminal, você pode executar a pasta inteira de código de teste.

Você pode aprender mais na documentação do Pytest sobre [mecanismos de importação](https://docs.pytest.org/en/7.1.x/explanation/pythonpath.html?highlight=folder#pytest-import-mechanisms-and-sys-path-pythonpath).
