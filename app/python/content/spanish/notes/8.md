## Programación Orientada a Objetos <a id="programacion-orientada-a-objetos"></a>

- Existen diferentes paradigmas de programación. A medida que aprendas otros lenguajes, empezarás a reconocer patrones como estos.
- Hasta este momento, has trabajado de manera procedural paso a paso.
- La programación orientada a objetos (OOP) es una solución convincente a problemas relacionados con la programación.
- Para empezar, escribe `code student.py` en la ventana del terminal y codifica de la siguiente manera:

      name = input("Nombre: ")
      house = input("Casa: ")
      print(f"{name} de la casa {house}")

  Observa que este programa sigue un paradigma procedural, paso a paso, similar a lo que has visto en partes previas de este curso.

- Basándonos en nuestro trabajo de semanas anteriores, podemos crear funciones para abstraer partes de este programa.

      def main():
          name = get_name()
          house = get_house()
          print(f"{name} de la casa {house}")


      def get_name():
          return input("Nombre: ")


      def get_house():
          return input("Casa: ")


      if __name__ == "__main__":
          main()

  Observa cómo `get_name` y `get_house` abstraen algunas necesidades de nuestra función `main`. Además, nota cómo las últimas líneas del código anterior le indican al compilador que ejecute la función `main`.

- Podemos simplificar aún más nuestro programa almacenando al estudiante como una `tupla`. Una `tupla` es una secuencia de valores. A diferencia de una `lista`, una `tupla` no puede ser modificada. En esencia, estamos devolviendo dos valores.

      def main():
          name, house = get_student()
          print(f"{name} de la casa {house}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return name, house


      if __name__ == "__main__":
          main()

  Observa cómo `get_student` devuelve `name, house`.

- Empaquetando esa `tupla`, de modo que podamos devolver ambos elementos a una variable llamada `student`, podemos modificar nuestro código de la siguiente manera.

      def main():
          student = get_student()
          print(f"{student[0]} de la casa {student[1]}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return (name, house)


      if __name__ == "__main__":
          main()

  Nota que `(name, house)` indica explícitamente a quien lee nuestro código que estamos devolviendo dos valores en uno. Además, observa cómo podemos indexar en las `tuplas` usando `student[0]` o `student[1]`.

- Las `tuplas` son inmutables, lo que significa que no podemos cambiar esos valores. La inmutabilidad es una forma mediante la cual podemos programar defensivamente.

      def main():
          student = get_student()
          if student[0] == "Padma":
              student[1] = "Ravenclaw"
          print(f"{student[0]} de la casa {student[1]}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return name, house


      if __name__ == "__main__":
          main()

  Observa que este código produce un error. Dado que las `tuplas` son inmutables, no podemos reasignar el valor de `student[1]`.

- Si quisiéramos brindar flexibilidad a nuestros colegas programadores, podríamos usar una `lista` de la siguiente manera.

      def main():
          student = get_student()
          if student[0] == "Padma":
              student[1] = "Ravenclaw"
          print(f"{student[0]} de la casa {student[1]}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return [name, house]


      if __name__ == "__main__":
          main()

  Ten en cuenta que las listas son mutables. Es decir, el orden de `casa` y `nombre` puede ser intercambiado por un programador. Podrías decidir utilizar esto en algunos casos donde desees brindar más flexibilidad a costa de la seguridad de tu código. Después de todo, si el orden de esos valores es modificable, los programadores que trabajen contigo podrían cometer errores en el futuro.

- Un diccionario también se podría utilizar en esta implementación. Recuerda que los diccionarios proporcionan una pareja clave-valor.

      def main():
          student = get_student()
          print(f"{student['nombre']} de la casa {student['casa']}")


      def get_student():
          student = {}
          student["nombre"] = input("Nombre: ")
          student["casa"] = input("Casa: ")
          return student


      if __name__ == "__main__":
          main()

  Observa en este caso que se devuelven dos pares clave-valor. Una ventaja de este enfoque es que podemos indexar en este diccionario utilizando las claves.

- Aún así, nuestro código puede mejorarse más. Observa que hay una variable innecesaria. Podemos eliminar `student = {}` ya que no necesitamos crear un diccionario vacío.

      def main():
          student = get_student()
          print(f"{student['nombre']} de la casa {student['casa']}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return {"nombre": name, "casa": house}


      if __name__ == "__main__":
          main()

  Observa que podemos utilizar llaves `{}` en la declaración `return` para crear el diccionario y devolverlo todo en la misma línea.

- Podemos incorporar nuestro caso especial con Padma en nuestra versión de código con diccionarios.

      def main():
          student = get_student()
          if student["nombre"] == "Padma":
              student["casa"] = "Ravenclaw"
          print(f"{student['nombre']} de la casa {student['casa']}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return {"nombre": name, "casa": house}


      if __name__ == "__main__":
          main()

  Observa cómo, similar en espíritu a nuestras iteraciones anteriores de este código, podemos utilizar los nombres de clave para indexar en nuestro diccionario de estudiantes.

## Clases <a id="classes"></a>

- Las clases son una forma, en programación orientada a objetos, mediante la cual podemos crear nuestro propio tipo de datos y darles nombres.
- Una clase es como un molde para un tipo de dato, donde podemos inventar nuestro propio tipo de datos y darles un nombre.
- Podemos modificar nuestro código de la siguiente manera para implementar nuestra propia clase llamada `Student`:

      class Student:
          ...


      def main():
          student = get_student()
          print(f"{student.name} de la casa {student.house}")


      def get_student():
          student = Student()
          student.name = input("Nombre: ")
          student.house = input("Casa: ")
          return student


      if __name__ == "__main__":
          main()

  Observa que por convención `Student` está en mayúscula. Además, la `...` simplemente significa que más adelante regresaremos para terminar esa parte de nuestro código. También observa que en `get_student`, podemos crear un `student` de la clase `Student` usando la sintaxis `student = Student()`. Además, notamos que utilizamos la "notación de punto" para acceder a los atributos de esta variable `student` de la clase `Student`.

- Cada vez que creas una clase y utilizas ese plano para crear algo, creas lo que se llama un "objeto" o una "instancia". En el caso de nuestro código, `student` es un objeto.
- Además, podemos establecer algunas bases para los atributos que se esperan dentro de un objeto cuya clase es `Student`. Podemos modificar nuestro código de la siguiente manera:

      class Student:
          def __init__(self, name, house):
              self.name = name
              self.house = house


      def main():
          student = get_student()
          print(f"{student.name} de la casa {student.house}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          student = Student(name, house)
          return student


      if __name__ == "__main__":
          main()

  Observa que dentro de `Student`, estandarizamos los atributos de esta clase. Podemos crear una función dentro de `class Student`, llamada un "método", que determina el comportamiento de un objeto de la clase `Student`. Dentro de esta función, toma el `name` y `house` que se le pasan y asigna estas variables a este objeto. Además, nota cómo el constructor `student = Student(name, house)` llama a esta función dentro de la clase `Student` y crea un `student`. `self` se refiere al objeto actual que acaba de ser creado.

- Podemos simplificar nuestro código de la siguiente manera:

      class Student:
          def __init__(self, name, house):
              self.name = name
              self.house = house


      def main():
          student = get_student()
          print(f"{student.name} de la casa {student.house}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return Student(name, house)


      if __name__ == "__main__":
          main()

  Observa cómo `return Student(name, house)` simplifica la iteración anterior de nuestro código donde la declaración del constructor se ejecutaba en su propia línea.

- Puedes aprender más en la documentación de Python sobre [clases](https://docs.python.org/3/tutorial/classes.html).

## `raise` <a id="raise"></a>

- La programación orientada a objetos te anima a encapsular toda la funcionalidad de una clase dentro de la definición de la clase. ¿Qué sucede si algo sale mal? ¿Qué pasa si alguien intenta escribir algo aleatorio? ¿Qué pasa si alguien intenta crear un estudiante sin nombre? Modifica tu código de la siguiente manera:

      class Student:
          def __init__(self, name, house):
              if not name:
                  raise ValueError("Nombre faltante")
              if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self.name = name
              self.house = house


      def main():
          student = get_student()
          print(f"{student.name} de {student.house}")


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return Student(name, house)


      if __name__ == "__main__":
          main()

  Observa cómo ahora verificamos que se proporcione un nombre y que se designe una casa apropiada. Resulta que podemos crear nuestras propias excepciones que alertan al programador sobre un error potencial creado por el usuario llamado `raise`. En el caso anterior, lanzamos `ValueError` con un mensaje de error específico.

- Resulta que Python te permite crear una función específica mediante la cual puedes imprimir los atributos de un objeto. Modifica tu código de la siguiente manera:

      class Student:
          def __init__(self, name, house, patronus):
              if not name:
                  raise ValueError("Nombre faltante")
              if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self.name = name
              self.house = house
              self.patronus = patronus

          def __str__(self):
              return f"{self.name} de {self.house}"


      def main():
          student = get_student()
          print(student)


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          patronus = input("Patronus: ")
          return Student(name, house, patronus)


      if __name__ == "__main__":
          main()

  Observa cómo `def __str__(self)` proporciona un medio mediante el cual se devuelve un estudiante cuando se llama. Por lo tanto, ahora puedes, como programador, imprimir un objeto, sus atributos o casi cualquier cosa que desees relacionada con ese objeto.

- `__str__` es un método integrado que viene con las clases de Python. Resulta que podemos crear nuestros propios métodos para una clase también. Modifica tu código de la siguiente manera:

      class Student:
          def __init__(self, name, house, patronus=None):
              if not name:
                  raise ValueError("Nombre faltante")
              if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              if patronus and patronus not in ["Stag", "Otter", "Jack Russell terrier"]:
                  raise ValueError("Patronus inválido")
              self.name = name
              self.house = house
              self.patronus = patronus

          def __str__(self):
              return f"{self.name} de {self.house}"

          def charm(self):
              match self.patronus:
                  case "Stag":
                      return "🐴"
                  case "Otter":
                      return "🦦"
                  case "Jack Russell terrier":
                      return "🐶"
                  case _:
                      return "🪄"


      def main():
          student = get_student()
          print("¡Expecto Patronum!")
          print(student.charm())


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          patronus = input("Patronus: ") or None
          return Student(name, house, patronus)


      if __name__ == "__main__":
          main()

  Observa cómo definimos nuestro propio método `charm`. A diferencia de los diccionarios, las clases pueden tener funciones integradas llamadas métodos. En este caso, definimos nuestro método `charm` donde casos específicos tienen resultados específicos. Además, nota que Python tiene la capacidad de utilizar emojis directamente en nuestro código.

- Antes de continuar, eliminemos nuestro código de patronus. Modifica tu código de la siguiente manera:

      class Student:
          def __init__(self, name, house):
              if not name:
                  raise ValueError("Nombre inválido")
              if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self.name = name
              self.house = house

          def __str__(self):
              return f"{self.name} de {self.house}"


      def main():
          student = get_student()
          student.house = "Callejón de la Cabeza de Puerco, número cuatro"
          print(student)


      def get_student():
          name = input("Nombre: ")
          house = input("Casa: ")
          return Student(name, house)


      if __name__ == "__main__":
          main()

  Observa cómo solo tenemos dos métodos: `__init__` y `__str__`.

## Decoradores <a id="decorators"></a>

- Las propiedades pueden ser utilizadas para fortalecer nuestro código. En Python, definimos propiedades utilizando "decoradores" de funciones, los cuales comienzan con `@`. Modifica tu código de la siguiente manera:

      class Estudiante:
          def __init__(self, nombre, casa):
              if not nombre:
                  raise ValueError("Nombre inválido")
              self.nombre = nombre
              self.casa = casa

          def __str__(self):
              return f"{self.nombre} de la casa {self.casa}"

          # Getter para casa
          @property
          def casa(self):
              return self._casa

          # Setter para casa
          @casa.setter
          def casa(self, casa):
              if casa not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self._casa = casa


      def principal():
          estudiante = obtener_estudiante()
          print(estudiante)


      def obtener_estudiante():
          nombre = input("Nombre: ")
          casa = input("Casa: ")
          return Estudiante(nombre, casa)


      if __name__ == "__main__":
          principal()

  Observa cómo hemos escrito `@property` encima de una función llamada `casa`. Al hacerlo, definimos `casa` como una propiedad de nuestra clase. Con `casa` como una propiedad, obtenemos la capacidad de definir cómo se establece y se recupera un atributo de nuestra clase, `_casa`. De hecho, ahora podemos definir una función llamada "setter", a través de `@casa.setter`, la cual será llamada cada vez que la propiedad de casa es establecida, por ejemplo, con `estudiante.casa = "Gryffindor"`. Aquí, hemos hecho que nuestro setter valide los valores de `casa` para nosotros. Observa cómo generamos un `ValueError` si el valor de `casa` no está entre alguna de las casas de Harry Potter, de lo contrario, utilizaremos `casa` para actualizar el valor de `_casa`. ¿Por qué `_casa` y no `casa`? `casa` es una propiedad de nuestra clase, con funciones a través de las cuales un usuario intenta establecer el atributo de nuestra clase. `_casa` es el atributo de esa clase en sí mismo. El guion bajo inicial, `_`, indica a los usuarios que no necesitan (¡y de hecho, no deberían!) modificar este valor directamente. `_casa` solo debe ser establecido a través del setter de `casa`. Observa cómo la propiedad `casa` simplemente devuelve ese valor de `_casa`, nuestro atributo de clase que presuntamente ha sido validado utilizando nuestro setter de `casa`. Cuando un usuario llama a `estudiante.casa`, obtienen el valor de `_casa` a través de nuestro "getter" de `casa`.

- Además del nombre de la casa, también podemos proteger el nombre de nuestro estudiante. Modifica tu código de la siguiente manera:

      class Estudiante:
          def __init__(self, nombre, casa):
              self.nombre = nombre
              self.casa = casa

          def __str__(self):
              return f"{self.nombre} de la casa {self.casa}"

          # Getter para nombre
          @property
          def nombre(self):
              return self._nombre

          # Setter para nombre
          @nombre.setter
          def nombre(self, nombre):
              if not nombre:
                  raise ValueError("Nombre inválido")
              self._nombre = nombre

          @property
          def casa(self):
              return self._casa

          @casa.setter
          def casa(self, casa):
              if casa not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self._casa = casa


      def principal():
          estudiante = obtener_estudiante()
          print(estudiante)


      def obtener_estudiante():
          nombre = input("Nombre: ")
          casa = input("Casa: ")
          return Estudiante(nombre, casa)


      if __name__ == "__main__":
          principal()

  Observa cómo, al igual que en el código anterior, proporcionamos un getter y un setter para el nombre.

## Métodos de Clase <a id="class-methods"></a>

- A veces, queremos agregar funcionalidad a una clase en sí misma, no a las instancias de esa clase.
- `@classmethod` es una función que podemos usar para agregar funcionalidad a una clase en su conjunto.
- Aquí hay un ejemplo de _no_ usar un método de clase. En la ventana de tu terminal, escribe `codigo sombrero.py` y código como sigue:

      import random


      class Sombrero:
          def __init__(self):
              self.casas = ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]

          def ordenar(self, nombre):
              print(nombre, "está en", random.choice(self.casas))


      sombrero = Sombrero()
      sombrero.ordenar("Harry")

  Observa cómo cuando pasamos el nombre del estudiante al sombrero seleccionador, nos dirá a qué casa se le ha asignado al estudiante. Observa que `sombrero = Sombrero()` instancia un `sombrero`. La funcionalidad `ordenar` siempre es manejada por la _instancia_ de la clase `Sombrero`. Al ejecutar `sombrero.ordenar("Harry")`, pasamos el nombre del estudiante al método `ordenar` de la instancia particular de `Sombrero`, a la que hemos llamado `sombrero`.

- Sin embargo, es posible que queramos ejecutar la función `ordenar` sin crear una instancia particular del sombrero seleccionador (¡después de todo, solo hay uno!). Podemos modificar nuestro código de la siguiente manera:

      import random


      class Sombrero:

          casas = ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]

          @classmethod
          def ordenar(cls, nombre):
              print(nombre, "está en", random.choice(cls.casas))


      Sombrero.ordenar("Harry")

  Observa cómo el método `__init__` se elimina porque no necesitamos instanciar un sombrero en ningún lugar de nuestro código. Por lo tanto, `self` deja de ser relevante y se elimina. Especificamos este `ordenar` como un `@classmethod`, reemplazando `self` con `cls`. Finalmente, observa cómo `Sombrero` se escribe en mayúscula al final de este código, porque este es el nombre de nuestra clase.

- Volviendo a `estudiantes.py` podemos modificar nuestro código de la siguiente forma, abordando algunas oportunidades perdidas relacionadas con los `@classmethod`s:

      class Estudiante:
          def __init__(self, nombre, casa):
              self.nombre = nombre
              self.casa = casa

          def __str__(self):
              return f"{self.nombre} de {self.casa}"

          @classmethod
          def obtener(cls):
              nombre = input("Nombre: ")
              casa = input("Casa: ")
              return cls(nombre, casa)


      def principal():
          estudiante = Estudiante.obtener()
          print(estudiante)


      if __name__ == "__main__":
          principal()  

  Observa que `get_student` se elimina y se crea un `@classmethod` llamado `obtener`. Ahora este método puede ser llamado sin necesidad de crear primero un estudiante.

## Herencia <a id="inheritance"></a>

- La herencia es, quizás, la característica más poderosa de la programación orientada a objetos.
- Sucede que puedes crear una clase que "hereda" métodos, variables y atributos de otra clase.
- En el terminal, ejecuta `código wizard.py`. El código es el siguiente:

      class Mago:
          def __init__(self, nombre):
              if not nombre:
                  raise ValueError("Falta el nombre")
              self.nombre = nombre

          ...


      class Estudiante(Mago):
          def __init__(self, nombre, casa):
              super().__init__(nombre)
              self.casa = casa

          ...


      class Profesor(Mago):
          def __init__(self, nombre, materia):
              super().__init__(nombre)
              self.materia = materia

          ...


      mago = Mago("Albus")
      estudiante = Estudiante("Harry", "Gryffindor")
      profesor = Profesor("Severus", "Defensa Contra las Artes Oscuras")
      ...

  Observa que hay una clase llamada `Mago` y una clase llamada `Estudiante` arriba. Además, hay una clase llamada `Profesor`. Tanto los estudiantes como los profesores tienen nombres. Asimismo, tanto los estudiantes como los profesores son magos. Por lo tanto, tanto `Estudiante` como `Profesor` heredan las características de `Mago`. Dentro de la clase "hija" `Estudiante`, `Estudiante` puede heredar de la clase "padre" o "super" `Mago`, ya que la línea `super().__init__(nombre)` ejecuta el método `init` de `Mago`. Por último, observa que las últimas líneas de este código crean un mago llamado Albus, un estudiante llamado Harry, y así sucesivamente.

## Sobrecarga de Operadores <a id="operator-overloading"></a>

- Algunos operadores como `+` y `-` pueden ser "sobrecargados" de manera que pueden tener más habilidades más allá de la simple aritmética.
- En tu ventana de terminal, escribe `code vault.py`. Luego, escribe el código a continuación:

      class Vault:
          def __init__(self, galeones=0, sickles=0, knuts=0):
              self.galeones = galeones
              self.sickles = sickles
              self.knuts = knuts

          def __str__(self):
              return f"{self.galeones} Galeones, {self.sickles} Sickles, {self.knuts} Knuts"

          def __add__(self, other):
              galeones = self.galeones + other.galeones
              sickles = self.sickles + other.sickles
              knuts = self.knuts + other.knuts
              return Vault(galeones, sickles, knuts)


      potter = Vault(100, 50, 25)
      print(potter)

      weasley = Vault(25, 50, 100)
      print(weasley)

      total = potter + weasley
      print(total)

  Observa cómo el método `__str__` devuelve una cadena formateada. Además, observa cómo el método `__add__` permite la suma de los valores de dos bóvedas. `self` es lo que está a la izquierda del operando `+`. `other` es lo que está a la derecha del `+`.

- Puedes aprender más en la documentación de Python sobre [sobrecarga de operadores](https://docs.python.org/3/reference/datamodel.html#special-method-names).
