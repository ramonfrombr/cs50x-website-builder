## Compilação

-   Na última vez, aprendemos a escrever nosso primeiro programa em C. Aprendemos a sintaxe da função `main` em nosso programa, a função `printf` para imprimir no terminal, como criar strings com aspas duplas e como incluir `stdio.h` para a função `printf`.
-   Em seguida, compilamos com `clang hello.c` para ser capaz de executar `./a.out` (o nome padrão), e então `clang -o hello hello.c` (passando um argumento de linha de comando para o nome de saída) para ser capaz de executar `./hello`.
-   Se quisermos usar a biblioteca CS50, por meio de `#include <cs50.h>`, para strings e a função `get_string`, também temos que adicionar uma flag: `clang -o hello hello.c -lcs50`. A flag `-l` vincula o arquivo `cs50`, que já está instalado no CS50 Sandbox, e inclui protótipos ou definições de strings e `get_string` (entre outros) para que nosso programa possa referenciá-los e usá-los.
-   Escrevemos nosso código-fonte em C, mas precisamos compilá-lo para código de máquina, em binário, antes que nossos computadores possam executá-lo.
    -   `clang` é o compilador, e `make` é um utilitário que nos ajuda a executar `clang` sem precisar indicar todas as opções manualmente.
-   "Compilar" código-fonte em código de máquina é composto por etapas menores:
    -   pré-processamento
    -   compilação
    -   montagem
    -   vinculação
-   **Pré-processamento** envolve a análise de linhas que começam com `#`, como `#include`, antes de tudo o mais. Por exemplo, `#include <cs50.h>` informará ao `clang` para procurar esse arquivo de cabeçalho primeiro, pois ele contém conteúdo que desejamos incluir em nosso programa. Em seguida, o `clang` substituirá essencialmente o conteúdo desses arquivos de cabeçalho em nosso programa.
-   Por exemplo ...

        #include <cs50.h>
        #include <stdio.h>

        int main(void)
        {
            string name = get_string("Name: ");
            printf("hello, %s\n", name);
        }

-   ... será pré-processado para:

        string get_string(string prompt);
        int printf(const char *format, ...);

        int main(void)
        {
            string name = get_string("Name: ");
            printf("hello, %s\n", name);
        }

-   **Compilar** pega nosso código-fonte, em C, e o converte em código de montagem, que se parece com isso:

        ...
        main:                         # @main
            .cfi_startproc
        # BB#0:
            pushq    %rbp
        .Ltmp0:
            .cfi_def_cfa_offset 16
        .Ltmp1:
            .cfi_offset %rbp, -16
            movq    %rsp, %rbp
        .Ltmp2:
            .cfi_def_cfa_register %rbp
            subq    $16, %rsp
            xorl    %eax, %eax
            movl    %eax, %edi
            movabsq    $.L.str, %rsi
            movb    $0, %al
            callq    get_string
            movabsq    $.L.str.1, %rdi
            movq    %rax, -8(%rbp)
            movq    -8(%rbp), %rsi
            movb    $0, %al
            callq    printf
            ...

    -   Essas instruções são de nível inferior e estão mais próximas das instruções binárias que a CPU de um computador pode entender diretamente. Elas geralmente operam em bytes, em vez de abstrações como nomes de variáveis.

-   O próximo passo é pegar o código de montagem e traduzi-lo para instruções binárias por meio de **montagem**. As instruções em binário são chamadas de **código de máquina**, que a CPU do computador pode executar diretamente.
-   A última etapa é a **vinculação**, onde o conteúdo de bibliotecas previamente compiladas que queremos vincular, como `cs50.c`, é combinado com o binário do nosso programa. Assim, acabamos com um arquivo binário, `a.out` ou `hello`, que é a versão compilada de `hello.c`, `cs50.c` e `printf.c`.

## Depuração

-   Bugs são erros em programas que não tínhamos a intenção de cometer. E a depuração é o processo de encontrar e corrigir os bugs.

## help50 e printf

- Vamos supor que escrevemos este programa, `buggy0.c`:

    int main(void)
    {
        printf("olá, mundo\n");
    }

    - Vemos um erro (em vermelho) quando tentamos compilar este programa com `make`, que indica que estamos "declarando implicitamente a função da biblioteca 'printf'". Não entendemos muito bem isso, então podemos executar `help50 make buggy0`, o qual nos dirá, no final, que talvez tenhamos esquecido de escrever `#include <stdio.h>`, que contém `printf`.

- Podemos tentar novamente com `buggy1.c`:

    #include <stdio.h>

    int main(void)
    {
        string name = get_string("Qual é o seu nome?\n");
        printf("olá, %s\n", name);
    }

    - Vemos muitos erros, e até mesmo o primeiro não parece fazer muito sentido. Então, podemos executar novamente `help50 make buggy1`, o qual nos indicará que precisamos de `cs50.h`, já que `string` não está definido.

- Para limpar a janela do terminal (para que possamos ver apenas a saída do que queremos executar em seguida), podemos pressionar `control + L` ou digitar `clear` como um comando na janela do terminal.
- Vamos olhar para `buggy2.c`:

    #include <stdio.h>

    int main(void)
    {
        for (int i = 0; i <= 10; i++)
        {
            printf("#\n");
        }
    }

    - Hmm, pretendíamos ver apenas 10 `#`, mas há 11. Se não soubéssemos qual é o problema (já que nosso programa está compilando sem erros e agora temos um erro lógico), poderíamos adicionar outra linha de impressão para nos ajudar:

        #include <stdio.h>

        int main(void)
        {
            for (int i = 0; i <= 10; i++)
            {
                printf("i agora é %i: ", i);
                printf("#\n");
            }
        }

    - Agora, vemos que `i` começou em 0 e continuou até chegar a 10, mas deveríamos parar quando for igual a 10, com `i < 10` em vez de `i <= 10`.

## debug50

-   Hoje também vamos dar uma olhada no CS50 IDE, que é como o CS50 Sandbox, mas com mais recursos. É um ambiente de desenvolvimento online, com um editor de código e uma janela de terminal, mas também ferramentas para depuração e colaboração:  
    ![janela do navegador com CS50 IDE, editor de código na parte superior com buggy2.c, janela de terminal na parte inferior](https://cs50.harvard.edu/x/2020/notes/2/cs50_ide.png)
-   No CS50 IDE, teremos uma outra ferramenta, `debug50`, para nos ajudar a depurar programas.
-   Vamos abrir o `buggy2.c` e tentar `make buggy2`. Mas salvamos o `buggy2.c` em uma pasta chamada `src2`, então precisamos executar `cd src2` para mudar nosso diretório para o correto. E o terminal do CS50 IDE irá nos lembrar em qual diretório estamos, com um prompt como `~/src/ $`. (O `~` indica o diretório padrão, ou diretório principal.)
-   Em vez de usar `printf`, também podemos depurar nosso programa interativamente. Podemos adicionar um _breakpoint_, ou um indicador para uma linha de código onde o depurador deve pausar nosso programa. Por exemplo, podemos clicar à esquerda da linha 5 do nosso código, e um círculo vermelho aparecerá:  
    ![editor de código com ícone vermelho ao lado da linha 5 do código](https://cs50.harvard.edu/x/2020/notes/2/breakpoint.png)
-   Agora, se executarmos `debug50 ./buggy2`, veremos o painel do depurador abrir à direita:  
    ![painel do depurador com controles, variáveis](https://cs50.harvard.edu/x/2020/notes/2/debugger_panel.png)
-   Vemos que a variável que criamos, `i`, está na seção `Variáveis Locais`, e vemos que tem o valor `0`.
-   Nosso breakpoint pausou nosso programa depois da linha 5, logo antes da linha 7, pois é a primeira linha de código que pode ser executada. Para continuar, temos alguns controles no painel do depurador. O triângulo azul continuará nosso programa até chegarmos a outro breakpoint ou ao final do nosso programa. A seta curva à sua direita irá "passar por cima" da linha, executando-a e pausando nosso programa novamente imediatamente depois.
-   Portanto, usaremos a seta curva para executar a próxima linha e ver o que muda em seguida. Estamos na linha `printf`, e pressionando novamente a seta curva, vemos um único `#` impresso na nossa janela de terminal. Com mais um clique da seta, vemos o valor de `i` à direita mudar para `1`. E podemos continuar clicando na seta para observar nosso programa sendo executado, uma linha de cada vez.
-   Para sair do depurador, podemos pressionar `control + C` para parar o programa.
-   Podemos economizar muito tempo no futuro investindo um pouco agora para aprender como usar o `debug50`!

## check50 e style50

- Podemos executar um comando como `check50 cs50/problems/hello`, onde `check50` é um programa que seguirá as instruções identificadas pelo argumento `cs50/problems/hello` para fazer o upload, executar e testar nosso programa nos servidores do CS50. Isso verificará se nosso programa está correto.
    - Ao escrever software no mundo real, os desenvolvedores geralmente escreverão seus próprios testes para garantir que seu código funcione conforme o esperado, especialmente quando mais recursos são adicionados ao mesmo código.
- `style50` é outro programa que verificará nosso código em busca de problemas estéticos, como espaços em branco, para que nosso código seja mais legível e sustentável. Por exemplo, podemos estar com indentação ausente. E o [Guia de Estilo](https://cs50.readthedocs.io/style/c/) incluirá mais explicações sobre o que esperamos.
- Podemos até mesmo usar a depuração com um pato de borracha, um método no qual explicamos o que estamos tentando fazer para um pato de borracha, a fim de percebermos o que estamos tentando fazer e o que precisamos corrigir.
- Também queremos escrever nosso código com um bom design, onde não apenas resolvemos o problema corretamente, mas o fazemos de forma adequada, onde fazemos escolhas razoáveis de como nosso programa é executado e fazemos compensações entre tempo, custo de desenvolvimento e memória.

## Tipos de Dados

-   Em C, temos diferentes tipos de variáveis que podemos usar para armazenar dados:
    -   bool (booleano) 1 byte
    -   char (caractere) 1 byte
    -   int (inteiro) 4 bytes
    -   float (número de ponto flutuante) 4 bytes
    -   long (inteiro longo) 8 bytes
    -   double (número de ponto flutuante duplo) 8 bytes
    -   string (cadeia de caracteres) ? bytes
-   Cada um desses tipos ocupa um certo número de bytes por variável que criamos, e os tamanhos acima são os valores que a sandbox, IDE e provavelmente o seu computador usam para cada tipo em C.

## Memória

- Dentro dos nossos computadores, temos chips chamados RAM, memória de acesso aleatório, que armazenam dados para uso a curto prazo. Podemos salvar um programa ou arquivo no disco rígido (ou SSD) para armazenamento a longo prazo, mas quando o abrimos, ele é copiado primeiro para a RAM. Embora a RAM seja muito menor e temporária (até que a energia seja desligada), ela é muito mais rápida.
- Podemos pensar em bytes, armazenados na RAM, como se estivessem em uma grade:
    ![chip de computador com grade sobreposta](https://cs50.harvard.edu/x/2020/notes/2/ram.png)
    - Na realidade, há milhões ou bilhões de bytes por chip.
- Em C, quando criamos uma variável do tipo `char`, que terá tamanho de um byte, ela será fisicamente armazenada em uma dessas caixas na RAM. Um inteiro, com 4 bytes, ocupará quatro dessas caixas.
- E cada uma dessas caixas é rotulada com algum número, ou endereço, de 0 a 1, de 2 e assim por diante.

## Arrays

- Vamos supor que desejamos armazenar três variáveis:

        #include <stdio.h>

        int main(void)
        {
            char c1 = 'H';
            char c2 = 'I';
            char c3 = '!';
            printf("%c %c %c\n", c1, c2, c3);
        }

    - Observe que usamos aspas simples para indicar um caractere literal e aspas duplas para múltiplos caracteres juntos em uma string.
    - Podemos compilar e executar isso para ver `H I !`.

- E sabemos que caracteres são apenas números, então se alterarmos nosso formato de string para `printf("%i %i %i\n", c1, c2, c3);`, podemos ver os valores numéricos de cada caractere impresso: `72 73 33`.
    - Podemos converter explicitamente, ou fazer a conversão, de cada caractere para um int antes de usá-lo, com `(int) c1`, mas nosso compilador pode fazer isso implicitamente para nós.
- E na memória, podemos ter três caixas, rotuladas como `c1`, `c2` e `c3` de alguma forma, cada uma representando um byte de binário com os valores de cada variável.
- Vamos olhar para `scores0.c`:

        #include <cs50.h>
        #include <stdio.h>

        int main(void)
        {
            // Notas
            int nota1 = 72;
            int nota2 = 73;
            int nota3 = 33;

            // Média
            printf("Média: %i\n", (nota1 + nota2 + nota3) / 3);
        }

    - Podemos imprimir a média de três números, mas agora precisamos criar uma variável para cada nota que queremos incluir e não podemos usá-las facilmente posteriormente.

- Acontece que, na memória, podemos armazenar variáveis uma após a outra, lado a lado. E em C, uma lista de variáveis armazenadas, uma após a outra em um bloco contíguo de memória, é chamada de **array**.
- Por exemplo, podemos usar `int notas[3];` para declarar um array de 3 inteiros.
- E podemos atribuir e usar variáveis em um array com:

        #include <cs50.h>
        #include <stdio.h>

        int main(void)
        {
            // Notas
            int notas[3];
            notas[0] = 72;
            notas[1] = 73;
            notas[2] = 33;

            // Média
            printf("Média: %i\n", (notas[0] + notas[1] + notas[2]) / 3);
        }

    - Observamos que os arrays são indexados a partir de zero, o que significa que o primeiro elemento ou valor tem índice 0.

- E repetimos o valor 3, representando o comprimento do nosso array, em dois lugares diferentes. Então podemos usar uma constante, ou valor fixo, para indicar que deve ser sempre o mesmo em ambos os lugares:

        #include <cs50.h>
        #include <stdio.h>

        const int N = 3;

        int main(void)
        {
            // Notas
            int notas[N];
            notas[0] = 72;
            notas[1] = 73;
            notas[2] = 33;

            // Média
            printf("Média: %i\n", (notas[0] + notas[1] + notas[2]) / N);
        }

    - Podemos usar a palavra-chave `const` para informar ao compilador que o valor de `N` nunca deve ser alterado pelo programa. E por convenção, iremos colocar nossa declaração da variável fora da função `main` e capitalizar seu nome, o que não é necessário para o compilador, mas mostra para outros humanos que esta variável é uma constante e facilita identificar desde o início.

- Com um array, podemos coletar nossas notas em um loop e acessá-las posteriormente em um loop também:

        #include <cs50.h>
        #include <stdio.h>

        float media(int comprimento, int array[]);

        int main(void)
        {
            // Obter número de notas
            int n = get_int("Notas:  ");

            // Obter notas
            int notas[n];
            for (int i = 0; i < n; i++)
            {
                notas[i] = get_int("Nota %i: ", i + 1);
            }

            // Média
            printf("Média: %.1f\n", media(n, notas));
        }

        float media(int comprimento, int array[])
        {
            int soma = 0;
            for (int i = 0; i < comprimento; i++)
            {
                soma += array[i];
            }
            return (float) soma / (float) comprimento;
        }

    - Primeiro, perguntaremos ao usuário o número de notas que ele possui, criaremos um array com espaço suficiente para o número de notas que ele possui e usaremos um loop para coletar todas as notas.
    - Em seguida, escreveremos uma função auxiliar `media` para retornar um `float`, ou um valor decimal. Passaremos o comprimento e um array de inteiros (que pode ter qualquer tamanho) e usaremos outro loop dentro de nossa função auxiliar para somar os valores em uma soma. Usamos `(float)` para converter tanto `soma` quanto `comprimento` em floats, para que o resultado da divisão dos dois também seja um float.
    - Por fim, quando imprimimos o resultado que obtemos, usamos `%.1f` para exibir apenas um lugar após a vírgula.

- Na memória, nosso array agora está armazenado como mostrado abaixo, onde cada valor ocupa não um, mas quatro bytes:
  ![grade com 72 rotulado como nota1, 73 rotulado como nota2, 33 rotulado como nota3, cada um ocupando quatro caixas, e muitas caixas vazias seguindo](https://cs50.harvard.edu/x/2020/notes/2/memory_with_array.png)

## Strings

- As strings são na verdade apenas arrays de caracteres. Se tivéssemos uma string `s`, cada caractere pode ser acessado com `s[0]`, `s[1]` e assim por diante.
- E acontece que uma string termina com um caractere especial, '\\0' ou um byte com todos os bits definidos como 0. Esse caractere é chamado de caractere nulo ou caractere terminador nulo. Portanto, na verdade, precisamos de quatro bytes para armazenar nossa string "HI!":
    ![grid with H labeled s[0], I labeled s[1], ! labeled s[2], \0 labeled s[3], each of which takes up one box, and many empty boxes following](https://cs50.harvard.edu/x/2020/notes/2/memory_with_string.png)
- Agora, vejamos como quatro strings em um array podem ser:

        string names[4];
        names[0] = "EMMA";
        names[1] = "RODRIGO";
        names[2] = "BRIAN";
        names[3] = "DAVID";

        printf("%s\n", names[0]);
        printf("%c%c%c%c\n", names[0][0], names[0][1], names[0][2], names[0][3]);

    - Podemos imprimir o primeiro valor em `names` como uma string, ou podemos obter a primeira string e obter cada caractere individual nessa string usando `[]` novamente. (Podemos pensar nisso como `(names[0])[0]`, embora não seja necessário os parênteses.)
    - E embora saibamos que o primeiro nome tinha quatro caracteres, `printf` provavelmente usou um loop para analisar cada caractere na string, imprimindo-os um de cada vez até atingir o caractere nulo que marca o final da string. E, na verdade, podemos imprimir `names[0][4]` como um `int` com `%i` e ver um `0` sendo impresso.

- Podemos visualizar cada caractere com seu próprio rótulo na memória:
    ![grid with E labeled names[0][0], M labeled names[0][1], and so on, until names[3][5] with a \0, each of which takes up one box, and empty boxes following](https://cs50.harvard.edu/x/2020/notes/2/memory_with_string_array.png)
- Podemos tentar experimentar com `string0.c`:

        #include <cs50.h>
        #include <stdio.h>
        #include <string.h>

        int main(void)
        {
            string s = get_string("Input:  ");
            printf("Output: ");
            for (int i = 0; i < strlen(s); i++)
            {
                printf("%c", s[i]);
            }
            printf("\n");
        }

    - Podemos usar a condição `s[i] != '\0'`, onde podemos verificar o caractere atual e imprimi-lo apenas se não for o caractere nulo.
    - Também podemos usar o comprimento da string, mas primeiro, precisamos de uma nova biblioteca, `string.h`, para `strlen`, que nos diz o comprimento de uma string.

- Podemos melhorar o design de nosso programa. `string0` foi um pouco ineficiente, pois verificamos o comprimento da string depois que cada caractere é impresso, em nossa condição. Mas como o comprimento da string não muda, podemos verificar o comprimento da string uma vez:

        #include <cs50.h>
        #include <stdio.h>
        #include <string.h>

        int main(void)
        {
            string s = get_string("Input: ");
            printf("Output:\n");
            for (int i = 0, n = strlen(s); i < n; i++)
            {
                printf("%c\n", s[i]);
            }
        }

    - Agora, no início de nosso loop, inicializamos tanto uma variável `i` quanto uma variável `n` e lembramos o comprimento de nossa string em `n`. Em seguida, podemos verificar os valores a cada vez, sem precisar realmente calcular o comprimento da string.
    - E nós precisamos usar um pouco mais de memória para `n`, mas isso nos economiza algum tempo ao não precisarmos verificar o comprimento da string a cada vez.

- Agora podemos combinar o que vimos para escrever um programa que pode capitalizar letras:

        #include <cs50.h>
        #include <stdio.h>
        #include <string.h>

        int main(void)
        {
            string s = get_string("Before: ");
            printf("After:  ");
            for (int i = 0, n = strlen(s); i < n; i++)
            {
                if (s[i] >= 'a' && s[i] <= 'z')
                {
                    printf("%c", s[i] - 32);
                }
                else
                {
                    printf("%c", s[i]);
                }
            }
            printf("\n");
        }

    - Primeiro, obtemos uma string `s`. Em seguida, para cada caractere na string, se for minúsculo (seu valor está entre o de `a` e `z`), o convertemos para maiúsculo. Caso contrário, apenas o imprimimos.
    - Podemos converter uma letra minúscula em seu equivalente em maiúscula, subtraindo a diferença entre seus valores ASCII. (Sabemos que as letras minúsculas têm um valor ASCII maior que as letras maiúsculas, e a diferença é convenientemente a mesma entre as mesmas letras, então podemos subtrair essa diferença para obter uma letra maiúscula a partir de uma letra minúscula.)

- Podemos usar as [**páginas do manual**](https://man.cs50.io/), ou manual do programador, para encontrar funções de biblioteca que podemos usar para realizar a mesma coisa:

        #include <cs50.h>
        #include <ctype.h>
        #include <stdio.h>
        #include <string.h>

        int main(void)
        {
            string s = get_string("Before: ");
            printf("After:  ");
            for (int i = 0, n = strlen(s); i < n; i++)
            {
                printf("%c", toupper(s[i]));
            }
            printf("\n");
        }

    - Ao pesquisar nas páginas do manual, vemos que `toupper()` é uma função, entre outras, de uma biblioteca chamada `ctype`, que podemos usar.

## Argumentos da linha de comando

- Utilizamos programas como `make` e `clang`, que recebem palavras extras após o seu nome na linha de comando. Acontece que os nossos próprios programas também podem receber **argumentos da linha de comando**.
- No arquivo `argv.c`, modificamos a aparência da nossa função `main`:

```c
    #include <cs50.h>
    #include <stdio.h>

    int main(int argc, string argv[])
    {
        if (argc == 2)
        {
            printf("olá, %s\n", argv[1]);
        }
        else
        {
            printf("olá, mundo\n");
        }
    }
```

    - `argc` e `argv` são duas variáveis que a nossa função `main` receberá quando o nosso programa for executado a partir da linha de comando. `argc` é a contagem de argumentos, ou número de argumentos, e `argv` é uma matriz de strings que são os argumentos. E o primeiro argumento, `argv[0]`, é o nome do nosso programa (a primeira palavra digitada, como `./hello`). Neste exemplo, verificamos se temos dois argumentos e imprimimos o segundo, caso exista.
    - Por exemplo, se executarmos `./argv David`, receberemos `olá, David` impresso, já que digitamos `David` como a segunda palavra em nosso comando.

- Acontece que podemos indicar erros em nosso programa retornando um valor da nossa função `main` (como indicado pelo `int` antes da nossa função `main`). Por padrão, nossa função `main` retorna `0` para indicar que nada deu errado, mas podemos escrever um programa para retornar um valor diferente:

```c
    #include <cs50.h>
    #include <stdio.h>

    int main(int argc, string argv[])
    {
        if (argc != 2)
        {
            printf("faltando argumento da linha de comando\n");
            return 1;
        }
        printf("olá, %s\n", argv[1]);
        return 0;
    }
```

   - O valor de retorno do `main` no nosso programa é chamado de código de saída.

- À medida que escrevemos programas mais complexos, esses códigos de erro podem nos ajudar a determinar o que deu errado, mesmo que não seja visível ou significativo para o usuário.

## Criptografia

- Se quisermos enviar uma mensagem para alguém, podemos querer **criptografá-la** ou embaralhá-la de alguma forma para que seja difícil para os outros lerem. A mensagem original, ou entrada em nosso algoritmo, é chamada de **texto simples**, e a mensagem criptografada, ou saída, é chamada de **texto cifrado**.
- Uma mensagem como `OI!` poderia ser convertida para ASCII como `72 73 33`. Mas qualquer pessoa poderia converter isso de volta para letras.
- Um algoritmo de criptografia geralmente requer outra entrada, além do texto simples. É necessário uma **chave**, e às vezes é apenas um número que é mantido em segredo. Com a chave, o texto simples pode ser convertido, por meio de algum algoritmo, em texto cifrado e vice-versa.
- Por exemplo, se quisermos enviar uma mensagem como `EU AMO VOCÊ`, podemos primeiro convertê-la para ASCII: `73 76 79 86 69 89 79 85`. Em seguida, podemos criptografá-la com uma chave de apenas `1` e um algoritmo simples, onde apenas adicionamos a chave a cada valor: `74 77 80 87 70 90 80 86`. Então, alguém que converter esse ASCII de volta para texto verá `J M P W F Z P V`. Para descriptografar isso, alguém precisará saber a chave.
