Ensemble de problèmes 4

La collaboration sur les ensembles de problèmes n'est pas autorisée, sauf dans la mesure où vous pouvez demander de l'aide à vos camarades de classe et à d'autres personnes, à condition que cette aide ne se réduise pas à quelqu'un d'autre qui fait votre travail pour vous, conformément à la politique du cours sur l'honnêteté académique.

L'équipe effectue des vérifications aléatoires des soumissions à CS50x. Les étudiants qui enfreignent cette politique seront exclus du cours. Les étudiants qui ont déjà terminé CS50x et qui sont reconnus coupables d'infraction verront leur certificat CS50 révoqué de manière permanente.

Que faire

Assurez-vous d'avoir terminé le Laboratoire 4 avant de commencer cet ensemble de problèmes.

  1. Connectez-vous à code.cs50.io en utilisant votre compte GitHub.
  2. Exécutez update50 dans la fenêtre du terminal de votre espace de code pour vous assurer que votre espace de code est à jour et, lorsque vous y êtes invité, cliquez sur Reconstruire maintenant.
  3. Soumettez l'une des options suivantes :
    • cette version de Filter si vous vous sentez moins à l'aise
    • cette version de Filter si vous vous sentez plus à l'aise
  4. Soumettez l'une des options suivantes :
    • Recover
    • Reverse

Si vous soumettez les deux versions de Filter, nous enregistrerons la meilleure de vos deux notes.

Quand le faire

Avant le 2023-12-31T23:59:00+00:00.

Conseils

  • Essayez les programmes de David de la Semaine 4.
  • Si vous voyez des erreurs lors de la compilation de votre code avec make, concentrez-vous d'abord sur la réparation de la toute première erreur que vous voyez, en faisant défiler vers le haut si nécessaire. Si vous n'êtes pas sûr de ce que cela signifie, essayez de demander de l'aide à help50. Par exemple, si vous essayez de compiler filter, et que 
    make filter

    produit des erreurs, essayez plutôt d'exécuter

    help50 make filter
