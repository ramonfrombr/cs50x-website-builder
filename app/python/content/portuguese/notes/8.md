## Programação Orientada a Objetos <a id="programação-orientada-a-objetos"></a>

- Existem diferentes paradigmas de programação. À medida que você aprende outras linguagens, começará a reconhecer padrões como esses.
- Até este ponto, você trabalhou de forma procedural, passo a passo.
- A programação orientada a objetos (OOP) é uma solução convincente para problemas relacionados à programação.
- Para começar, digite `code student.py` na janela do terminal e escreva o código da seguinte forma:

      name = input("Nome: ")
      house = input("Casa: ")
      print(f"{name} da casa {house}")

  Observe que este programa segue um paradigma procedural, passo a passo: assim como você viu nas partes anteriores deste curso.

- Com base no nosso trabalho das semanas anteriores, podemos criar funções para abstrair partes desse programa.

      def main():
          name = obter_nome()
          house = obter_casa()
          print(f"{name} da casa {house}")


      def obter_nome():
          return input("Nome: ")


      def obter_casa():
          return input("Casa: ")


      if __name__ == "__main__":
          main()

  Observe como `obter_nome` e `obter_casa` abstraem parte das necessidades de nossa função `main`. Além disso, observe como as últimas linhas de código acima dizem ao compilador para executar a função `main`.

- Podemos simplificar ainda mais nosso programa armazenando o aluno como uma `tuple`. Uma `tuple` é uma sequência de valores. Ao contrário de uma `list`, uma `tuple` não pode ser modificada. Em essência, estamos retornando dois valores.

      def main():
          name, house = obter_aluno()
          print(f"{name} da casa {house}")


      def obter_aluno():
          name = input("Nome: ")
          house = input("Casa: ")
          return name, house


      if __name__ == "__main__":
          main()

  Observe como `obter_aluno` retorna `name, house`.

- Empacotando essa `tuple`, de forma que possamos retornar ambos os itens para uma variável chamada `student`, podemos modificar nosso código da seguinte forma.

      def main():
          student = obter_aluno()
          print(f"{student[0]} da casa {student[1]}")


      def obter_aluno():
          name = input("Nome: ")
          house = input("Casa: ")
          return (name, house)


      if __name__ == "__main__":
          main()

  Observe que `(name, house)` explicitamente informa a qualquer pessoa que esteja lendo nosso código que estamos retornando dois valores em um só. Além disso, observe como podemos acessar as `tuple`s usando `student[0]` ou `student[1]`.

- `tuple`s são imutáveis, o que significa que não podemos alterar esses valores. Imutabilidade é uma maneira pela qual podemos programar defensivamente.

      def main():
          student = obter_aluno()
          if student[0] == "Padma":
              student[1] = "Ravenclaw"
          print(f"{student[0]} da casa {student[1]}")


      def obter_aluno():
          name = input("Nome: ")
          house = input("Casa: ")
          return name, house


      if __name__ == "__main__":
          main()

  Observe que esse código produz um erro. Como as `tuple`s são imutáveis, não podemos atribuir um novo valor a `student[1]`.

- Se quiséssemos fornecer flexibilidade aos nossos colegas programadores, poderíamos utilizar uma `list` da seguinte forma.

      def main():
          student = obter_aluno()
          if student[0] == "Padma":
              student[1] = "Ravenclaw"
          print(f"{student[0]} da casa {student[1]}")


      def obter_aluno():
          name = input("Nome: ")
          house = input("Casa: ")
          return [name, house]


      if __name__ == "__main__":
          main()

  Observe que listas são mutáveis. Ou seja, a ordem de `house` e `name` pode ser alterada por um programador. Você pode decidir utilizar isso em alguns casos em que deseja fornecer mais flexibilidade ao custo da segurança do seu código. Afinal, se a ordem desses valores é modificável, programadores que trabalham com você podem cometer erros no futuro.

- Um dicionário também pode ser utilizado nessa implementação. Lembre-se de que os dicionários fornecem um par de chave-valor.

      def main():
          student = obter_aluno()
          print(f"{student['name']} da casa {student['house']}")


      def obter_aluno():
          student = {}
          student["name"] = input("Nome: ")
          student["house"] = input("Casa: ")
          return student


      if __name__ == "__main__":
          main()

  Observe que, neste caso, dois pares de chave-valor são retornados. Uma vantagem dessa abordagem é que podemos acessar esse dicionário usando as chaves.

- Ainda assim, nosso código pode ser melhorado. Note que há uma variável desnecessária. Podemos remover `student = {}` porque não precisamos criar um dicionário vazio.

      def main():
          student = obter_aluno()
          print(f"{student['name']} da casa {student['house']}")


      def obter_aluno():
          name = input("Nome: ")
          house = input("Casa: ")
          return {"name": name, "house": house}


      if __name__ == "__main__":
          main()

  Observe que podemos usar as chaves `{}` na instrução `return` para criar o dicionário e retorná-lo na mesma linha.

- Podemos fornecer nosso caso especial com Padma na versão do nosso código que usa dicionário.

      def main():
          student = obter_aluno()
          if student["name"] == "Padma":
              student["house"] = "Ravenclaw"
          print(f"{student['name']} da casa {student['house']}")


      def obter_aluno():
          name = input("Nome: ")
          house = input("Casa: ")
          return {"name": name, "house": house}


      if __name__ == "__main__":
          main()

  Observe como, de maneira semelhante às nossas iterações anteriores deste código, podemos usar os nomes das chaves para acessar nosso dicionário de aluno.

## Classes <a id="classes"></a>

- As classes são uma forma pela qual, na programação orientada a objetos, podemos criar nosso próprio tipo de dado e atribuí-los nomes.
- Uma classe é como um molde para um tipo de dado - onde podemos inventar nosso próprio tipo de dado e atribuir um nome a ele.
- Podemos modificar nosso código da seguinte forma para implementar nossa própria classe chamada `Student`:

      class Student:
          ...


      def main():
          student = get_student()
          print(f"{student.name} de {student.house}")


      def get_student():
          student = Student()
          student.name = input("Nome: ")
          student.house = input("Casa: ")
          return student


      if __name__ == "__main__":
          main()

  Perceba que por convenção, `Student` é escrito com a primeira letra maiúscula. Além disso, observe que `...` significa apenas que retornaremos mais tarde para finalizar aquela parte do código. Além disso, observe que em `get_student`, podemos criar um `student` da classe `Student` usando a sintaxe `student = Student()`. Adicionalmente, observe que utilizamos a "notação de ponto" para acessar os atributos dessa variável `student` da classe `Student`.

- Toda vez que você cria uma classe e a utiliza para criar algo, você cria o que é chamado de "objeto" ou "instância". No caso do nosso código, `student` é um objeto.
- Além disso, podemos estabelecer algumas informações básicas sobre os atributos esperados em um objeto cuja classe é `Student`. Podemos modificar nosso código da seguinte forma:

      class Student:
          def __init__(self, name, house):
              self.name = name
              self.house = house


      def main():
          student = get_student()
          print(f"{student.name} de {student.house}")


      def get_student():
          name = input("Nome: ")
          house = input("Casa: ")
          student = Student(name, house)
          return student


      if __name__ == "__main__":
          main()

  Observe que dentro de `Student`, padronizamos os atributos dessa classe. Podemos criar uma função dentro da `class Student`, chamada "método", que determina o comportamento de um objeto da classe `Student`. Dentro dessa função, ela recebe o `name` e o `house` passados para ela e atribui essas variáveis a esse objeto. Além disso, note como o construtor `student = Student(name, house)` chama essa função dentro da classe `Student` e cria um `student`. `self` se refere ao objeto atual que acabou de ser criado.

- Podemos simplificar nosso código da seguinte forma:

      class Student:
          def __init__(self, name, house):
              self.name = name
              self.house = house


      def main():
          student = get_student()
          print(f"{student.name} de {student.house}")


      def get_student():
          name = input("Nome: ")
          house = input("Casa: ")
          return Student(name, house)


      if __name__ == "__main__":
          main()

  Observe como `return Student(name, house)` simplifica a iteração anterior do nosso código, onde a declaração do construtor era executada em uma linha separada.

- Você pode obter mais informações na documentação do Python sobre [classes](https://docs.python.org/3/tutorial/classes.html).

## `raise` <a id="raise"></a>

- O programa orientado a objetos encoraja a encapsulação de todas as funcionalidades de uma classe dentro da definição da classe. E se algo der errado? E se alguém tentar digitar algo aleatório? E se alguém tentar criar um estudante sem um nome? Modifique seu código da seguinte forma:

```python
     class Student:
         def **init**(self, name, house):
             if not name:
                 raise ValueError("Nome ausente")
             if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                 raise ValueError("Casa inválida")
             self.name = name
             self.house = house

def main():
         student = get_student()
         print(f"{student.name} de {student.house}")

def get_student():
         name = input("Nome: ")
         house = input("Casa: ")
         return Student(name, house)

if **name** == "**main**":
         main()
```

Note como verificamos agora se um nome é fornecido e uma casa adequada é designada. Acontece que podemos criar nossas próprias exceções que alertam o programador para um possível erro criado pelo usuário, chamadas de `raise`. No caso acima, estamos levantando `ValueError` com uma mensagem de erro específica.

- Acontece que o Python permite que você crie uma função específica pela qual você pode imprimir os atributos de um objeto. Modifique seu código da seguinte forma:

```python
     class Student:
         def **init**(self, name, house, patronus):
             if not name:
                 raise ValueError("Nome ausente")
             if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                 raise ValueError("Casa inválida")
             self.name = name
             self.house = house
             self.patronus = patronus

def **str**(self):
             return f"{self.name} de {self.house}"

def main():
         student = get_student()
         print(student)

def get_student():
         name = input("Nome: ")
         house = input("Casa: ")
         patronus = input("Patrono: ")
         return Student(name, house, patronus)

if **name** == "**main**":
         main()
```

Note como `def __str__(self)` fornece um meio pelo qual um estudante é retornado quando chamado. Portanto, você, como programador, pode imprimir um objeto, seus atributos ou quase qualquer coisa relacionada a esse objeto.

- `__str__` é um método embutido que vem com as classes do Python. Acontece que também podemos criar nossos próprios métodos para uma classe! Modifique seu código da seguinte forma:

```python
     class Student:
         def **init**(self, name, house, patronus=None):
             if not name:
                 raise ValueError("Nome ausente")
             if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                 raise ValueError("Casa inválida")
             if patronus and patronus not in ["Javali", "Ourico-cacheiro", "Rottweiler"]:
                 raise ValueError("Patrono inválido")
             self.name = name
             self.house = house
             self.patronus = patronus

def **str**(self):
             return f"{self.name} de {self.house}"

def charm(self):
             match self.patronus:
                 case "Javali":
                     return "🐴"
                 case "Ourico-cacheiro":
                     return "🦦"
                 case "Rottweiler":
                     return "🐶"
                 case \_:
                     return "🪄"

def main():
         student = get_student()
         print("Expecto Patronum!")
         print(student.charm())

def get_student():
         name = input("Nome: ")
         house = input("Casa: ")
         patronus = input("Patrono: ") or None
         return Student(name, house, patronus)

if **name** == "**main**":
         main()
```

Note como definimos nosso próprio método `charm`. Ao contrário de dicionários, as classes podem ter funções embutidas chamadas métodos. Neste caso, definimos nosso método `charm` onde casos específicos têm resultados específicos. Além disso, observe que o Python tem a capacidade de usar emojis diretamente em nosso código.

- Antes de avançar, vamos remover nosso código de patrono. Modifique seu código da seguinte forma:


Note como temos apenas dois métodos: `__init__` e `__str__`.

## Decoradores <a id="decorators"></a>

- Propriedades podem ser utilizadas para fortalecer nosso código. Em Python, definimos propriedades usando "decoradores" de função, que começam com `@`. Modifique seu código da seguinte forma:

      class Student:
          def __init__(self, name, house):
              if not name:
                  raise ValueError("Nome inválido")
              self.name = name
              self.house = house

          def __str__(self):
              return f"{self.name} da {self.house}"

          # Getter para house
          @property
          def house(self):
              return self._house

          # Setter para house
          @house.setter
          def house(self, house):
              if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self._house = house


      def main():
          student = get_student()
          print(student)


      def get_student():
          name = input("Nome: ")
          house = input("Casa: ")
          return Student(name, house)


      if __name__ == "__main__":
          main()

  Observe como escrevemos `@property` acima de uma função chamada `house`. Fazendo isso, definimos `house` como uma propriedade da nossa classe. Com `house` como uma propriedade, ganhamos a capacidade de definir como algum atributo da nossa classe, `_house`, deve ser definido e obtido. De fato, agora podemos definir uma função chamada "setter", via `@house.setter`, que será chamada sempre que a propriedade `house` for definida - por exemplo, com `student.house = "Gryffindor"`. Aqui, fizemos com que nosso setter valide os valores de `house` para nós. Observe como levantamos um `ValueError` se o valor de `house` não for nenhuma das casas de Harry Potter, caso contrário, usaremos `house` para atualizar o valor de `_house`. Por que `_house` e não `house`? `house` é uma propriedade da nossa classe, com funções através das quais um usuário tenta definir o atributo da nossa classe. `_house` é o próprio atributo da classe. O traço de sublinhado inicial, `_`, indica aos usuários que eles não precisam (e na verdade, não devem!) modificar esse valor diretamente. `_house` deve ser definido apenas através do setter `house`. Observe como a propriedade `house` simplesmente retorna o valor de `_house`, nosso atributo de classe que foi presumivelmente validado usando nosso setter `house`. Quando um usuário chama `student.house`, ele está obtendo o valor de `_house` através do nosso "getter" `house`.

- Além do nome da casa, também podemos proteger o nome do nosso estudante. Modifique seu código da seguinte forma:

      class Student:
          def __init__(self, name, house):
              self.name = name
              self.house = house

          def __str__(self):
              return f"{self.name} da {self.house}"

          # Getter para name
          @property
          def name(self):
              return self._name

          # Setter para name
          @name.setter
          def name(self, name):
              if not name:
                  raise ValueError("Nome inválido")
              self._name = name

          @property
          def house(self):
              return self._house

          @house.setter
          def house(self, house):
              if house not in ["Gryffindor", "Hufflepuff", "Ravenclaw", "Slytherin"]:
                  raise ValueError("Casa inválida")
              self._house = house


      def main():
          student = get_student()
          print(student)


      def get_student():
          name = input("Nome: ")
          house = input("Casa: ")
          return Student(name, house)


      if __name__ == "__main__":
          main()

  Observe como, assim como no código anterior, fornecemos um getter e setter para o nome.

- Você pode aprender mais na documentação de [métodos](https://docs.python.org/3/tutorial/classes.html) do Python.

## Métodos de Classe <a id="class-methods"></a>

- Às vezes, queremos adicionar funcionalidade a uma classe em si, não às instâncias dessa classe.
- `@classmethod` é uma função que podemos usar para adicionar funcionalidade a uma classe inteira.
- Aqui está um exemplo de _não_ uso de um método de classe. Na janela do terminal, digite `code hat.py` e o código a seguir:

      import random


      class Hat:
          def __init__(self):
              self.houses = ["Grifinória", "Lufa-Lufa", "Corvinal", "Sonserina"]

          def sort(self, name):
              print(name, "está em", random.choice(self.houses))


      hat = Hat()
      hat.sort("Harry")

  Observe como, ao passarmos o nome do estudante para o chapéu seletor, ele nos dirá em qual casa o estudante está designado. Observe que `hat = Hat()` instância um `hat`. A funcionalidade `sort` é sempre tratada pela _instância_ da classe `Hat`. Ao executar `hat.sort("Harry")`, passamos o nome do estudante para o método `sort` da instância específica de `Hat`, que chamamos de `hat`.

- No entanto, podemos querer executar a função `sort` sem criar uma instância específica do chapéu seletor (afinal, há apenas um!). Podemos modificar nosso código da seguinte maneira:

      import random


      class Hat:

          houses = ["Grifinória", "Lufa-Lufa", "Corvinal", "Sonserina"]

          @classmethod
          def sort(cls, name):
              print(name, "está em", random.choice(cls.houses))


      Hat.sort("Harry")

  Observe como o método `__init__` é removido porque não precisamos instanciar um chapéu em nenhum lugar do nosso código. Portanto, `self` não é mais relevante e é removido. Especificamos esse `sort` como um `@classmethod`, substituindo `self` por `cls`. Por fim, observe como `Hat` está em maiúscula por convenção perto do final deste código, porque esse é o nome da nossa classe.

- Voltando para `students.py`, podemos modificar nosso código da seguinte maneira, abordando algumas oportunidades perdidas relacionadas aos `@classmethod`s:

      class Student:
          def __init__(self, name, house):
              self.name = name
              self.house = house

          def __str__(self):
              return f"{self.name} de {self.house}"

          @classmethod
          def get(cls):
              name = input("Nome: ")
              house = input("Casa: ")
              return cls(name, house)


      def main():
          student = Student.get()
          print(student)


      if __name__ == "__main__":
          main()

  Observe que `get_student` é removido e um `@classmethod` chamado `get` é criado. Agora, esse método pode ser chamado sem ter que criar um aluno primeiro.

## Herança <a id="herança"></a>

- A herança é, talvez, a funcionalidade mais poderosa da programação orientada a objetos.
- Por acaso, você pode criar uma classe que "herda" métodos, variáveis ​​e atributos de outra classe.
- No terminal, execute `python wizard.py`. Código a seguir:

      class Wizard:
          def __init__(self, name):
              if not name:
                  raise ValueError("Nome não informado")
              self.name = name

          ...


      class Student(Wizard):
          def __init__(self, name, house):
              super().__init__(name)
              self.house = house

          ...


      class Professor(Wizard):
          def __init__(self, name, subject):
              super().__init__(name)
              self.subject = subject

          ...


      wizard = Wizard("Albus")
      student = Student("Harry", "Gryffindor")
      professor = Professor("Severus", "Defesa Contra as Artes das Trevas")
      ...

  Observe que há uma classe chamada `Wizard` acima e uma classe chamada `Student`. Além disso, observe que há uma classe chamada `Professor`. Tanto os alunos quanto os professores têm nomes. Além disso, tanto os alunos quanto os professores são feiticeiros. Portanto, tanto `Student` quanto `Professor` herdam as características de `Wizard`. Dentro da classe "filha" `Student`, `Student` pode herdar da classe "pai" ou "super" `Wizard`, pois a linha `super().__init__(name)` executa o método `init` de `Wizard`. Por fim, observe que as últimas linhas deste código criam um feiticeiro chamado Albus, um estudante chamado Harry e assim por diante.

## Sobrecarga de Operadores <a id="sobrecarga-de-operadores"></a>

- Alguns operadores, como `+` e `-`, podem ser "sobrecarregados" de forma que tenham mais funcionalidades além da simples aritmética.
- Na sua janela do terminal, digite `code vault.py`. Em seguida, codifique da seguinte forma:

      class Vault:
          def __init__(self, galeões=0, sickles=0, knuts=0):
              self.galeões = galeões
              self.sickles = sickles
              self.knuts = knuts

          def __str__(self):
              return f"{self.galeões} Galeões, {self.sickles} Sickles, {self.knuts} Knuts"

          def __add__(self, other):
              galeões = self.galeões + other.galeões
              sickles = self.sickles + other.sickles
              knuts = self.knuts + other.knuts
              return Vault(galeões, sickles, knuts)


      potter = Vault(100, 50, 25)
      print(potter)

      weasley = Vault(25, 50, 100)
      print(weasley)

      total = potter + weasley
      print(total)

  Observe como o método `__str__` retorna uma string formatada. Além disso, observe como o método `__add__` permite a adição dos valores de dois cofres. `self` é o que está à esquerda do operador `+`. `other` é o que está à direita do `+`.

- Você pode aprender mais na documentação do Python sobre [sobrecarga de operadores](https://docs.python.org/3/reference/datamodel.html#special-method-names).
