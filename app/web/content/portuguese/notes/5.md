## JavaScript <a id="javascript"></a>

Vamos começar revisitando um diagrama de algumas aulas atrás:

![Diagrama Cliente-Servidor](https://cs50.harvard.edu/web/2020/notes/5/images/client_server.png)

Lembre-se de que na maioria das interações online, temos um cliente/usuário que envia uma Requisição HTTP para um servidor, que envia de volta uma Resposta HTTP. Todo o código em Python que escrevemos até agora usando Django estava rodando em um servidor. O JavaScript nos permitirá executar código no lado do cliente, o que significa que nenhuma interação com o servidor é necessária enquanto ele está em execução, permitindo que nossos sites se tornem muito mais interativos.

Para adicionar um pouco de JavaScript à nossa página, podemos adicionar um par de tags `<script>` em algum lugar da nossa página HTML. Usamos as tags `<script>` para sinalizar para o navegador que qualquer coisa que escrevermos entre as duas tags é código JavaScript que desejamos executar quando um usuário visita nosso site. Nosso primeiro programa pode se parecer com isso:

    alert('Olá, mundo!');

A função `alert` em JavaScript exibe uma mensagem ao usuário que ele pode então dispensar. Para mostrar onde isso se encaixaria em um documento HTML real, aqui está um exemplo de uma página simples com algum JavaScript:

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Olá</title>
            <script>
                alert('Olá, mundo!');
            </script>
        </head>
        <body>
            <h1>Olá!</h1>
        </body>
    </html>

## Eventos <a id="events"></a>

Uma característica do JavaScript que o torna útil para a programação web é que ele suporta a [Programação Orientada a Eventos](https://medium.com/@vsvaibhav2016/introduction-to-event-driven-programming-28161b79c223).

A Programação Orientada a Eventos é um paradigma de programação que se baseia na detecção de eventos e nas ações que devem ser tomadas quando um evento é detectado.

Um evento pode ser quase qualquer coisa, incluindo um botão sendo clicado, o cursor sendo movido, uma resposta sendo digitada ou uma página sendo carregada. Praticamente tudo o que um usuário faz para interagir com uma página da web pode ser considerado um evento. Em JavaScript, usamos [Ouvintes de Eventos](https://www.w3schools.com/js/js_htmldom_eventlistener.asp) que aguardam a ocorrência de eventos específicos e, em seguida, executam algum código.

Vamos começar transformando nosso JavaScript acima em uma [função](https://www.w3schools.com/js/js_functions.asp) chamada `hello`:

    function hello() {
        alert('Olá, mundo!')
    }

Agora, vamos trabalhar em executar esta função sempre que um botão for clicado. Para fazer isso, vamos criar um botão HTML em nossa página com um atributo `onclick`, que dá ao navegador instruções sobre o que deve acontecer quando o botão for clicado:

    <button onclick="hello()">Clique Aqui</button>

Essas mudanças nos permitem esperar para executar partes do nosso código JavaScript até que um determinado evento ocorra.

## Variáveis <a id="variables"></a>

JavaScript é uma linguagem de programação, assim como Python, C ou qualquer outra linguagem com a qual você já trabalhou, o que significa que possui muitas das mesmas características que outras linguagens, incluindo variáveis. Existem três palavras-chave que podemos usar para atribuir valores em JavaScript:

-   `var`: usada para definir uma variável globalmente

```
var idade = 20;
```

-   `let`: usada para definir uma variável que é limitada ao escopo do bloco atual, como uma função ou loop

```
let contador = 1;
```

-   `const`: usada para definir um valor que não mudará

```
const PI = 3.14;
```

Para um exemplo de como podemos usar uma variável, vamos dar uma olhada em uma página que mantém o controle de um contador:

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Contagem</title>
            <script>
                let contador = 0;
                function contar() {
                    contador++;
                    alert(contador);
                }
            </script>
        </head>
        <body>
            <h1>Olá!</h1>
            <button onclick="contar()">Contar</button>
        </body>
    </html>

## `querySelector` <a id="queryselector"></a>

Além de nos permitir exibir mensagens por meio de alertas, o JavaScript também nos permite alterar elementos na página. Para fazer isso, primeiro devemos introduzir uma função chamada `document.querySelector`. Esta função pesquisa e retorna elementos do DOM. Por exemplo, podemos usar:

    let heading = document.querySelector('h1');

para extrair um cabeçalho. Em seguida, para manipular o elemento que encontramos recentemente, podemos alterar sua propriedade `innerHTML`:

    heading.innerHTML = `Adeus!`;

Assim como em Python, também podemos aproveitar [condições](https://www.w3schools.com/js/js_if_else.asp) em JavaScript. Por exemplo, vamos dizer que, em vez de sempre alterar nosso cabeçalho para `Adeus!`, desejamos alternar entre `Olá!` e `Adeus!`. Nossa página pode ser parecida com a mostrada abaixo. Observe que em JavaScript, usamos `===` como uma comparação mais forte entre dois itens, que também verifica se os objetos são do mesmo tipo. Geralmente queremos usar `===` sempre que possível.

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Contador</title>
            <script>
                function hello() {
                    const header = document.querySelector('h1');
                    if (header.innerHTML === 'Olá!') {
                        header.innerHTML = 'Adeus!';
                    }
                    else {
                        header.innerHTML = 'Olá!';
                    }
                }
            </script>
        </head>
        <body>
            <h1>Olá!</h1>
            <button onclick="hello()">Clique Aqui</button>
        </body>
    </html>

![alternar](https://cs50.harvard.edu/web/2020/notes/5/images/toggle.gif)

## Manipulação do DOM <a id="dom-manipulation"></a>

Vamos usar essa ideia de manipulação do DOM para melhorar nossa página de contador:

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Contagem</title>
            <script>
                let contador = 0;
                function contar() {
                    contador++;
                    document.querySelector('h1').innerHTML = contador;
                }
            </script>
        </head>
        <body>
            <h1>0</h1>
            <button onclick="contar()">Contar</button>
        </body>
    </html>

![contagem 2](https://cs50.harvard.edu/web/2020/notes/5/images/count2.gif)

Podemos tornar essa página ainda mais interessante, exibindo um alerta toda vez que o contador alcançar um múltiplo de dez. Nesse alerta, queremos formatar uma string para personalizar a mensagem, o que podemos fazer em JavaScript usando [template literals](https://developer.mozilla.org/pt-BR/docs/Web/JavaScript/Reference/Template_literals). Template literals requerem que haja crases (`` ` ``) em torno de toda a expressão e um $ e chaves em torno de quaisquer substituições. Por exemplo, vamos alterar nossa função de contagem:

    function contar() {
        contador++;
        document.querySelector('h1').innerHTML = contador;

        if (contador % 10 === 0) {
            alert(`Contagem agora é ${contador}`)
        }
    }

![contagem com alerta](https://cs50.harvard.edu/web/2020/notes/5/images/count3.gif)

Agora, vamos ver algumas formas de melhorar o design dessa página. Primeiramente, assim como tentamos evitar estilização em linha com CSS, queremos evitar JavaScript em linha tanto quanto possível. Podemos fazer isso em nosso exemplo de contador adicionando uma linha de script que altera o atributo `onclick` de um botão na página e removendo o atributo `onclick` de dentro da tag `button`.

    document.querySelector('button').onclick = contar;

Uma coisa a se observar sobre o que acabamos de fazer é que não estamos chamando a função `contar` adicionando parênteses a seguir, mas apenas nomeando a função. Isso especifica que só queremos chamar essa função quando o botão é clicado. Isso funciona porque, assim como em Python, o JavaScript suporta a programação funcional, então as funções podem ser tratadas como valores em si mesmas.

A mudança acima sozinha não é o suficiente, como podemos ver ao inspecionar a página e olhar para o console do nosso navegador:

![console de erro](https://cs50.harvard.edu/web/2020/notes/5/images/error0.png)

Esse erro apareceu porque, ao procurar um elemento usando `document.querySelector('button')`, não encontrou nada. Isso aconteceu porque demora um pouco para a página carregar e nosso código JavaScript rodou antes do botão ter sido renderizado. Para contornar isso, podemos especificar que o código será executado somente após a página ter carregado usando a função [addEventListener](https://www.w3schools.com/jsref/met_document_addeventlistener.asp). Esta função recebe dois argumentos:

1. Um evento a ser ouvido (por exemplo: `'click'`)
2. Uma função a ser executada quando o evento é detectado (por exemplo: `hello` de cima)

Podemos usar a função para rodar o código somente após todo o conteúdo ter carregado:

    document.addEventListener('DOMContentLoaded', function() {
        // Algum código aqui
    });

No exemplo acima, usamos uma função [anônima](https://www.w3schools.com/js/js_function_definition.asp), que é uma função a qual não foi dado um nome. Colocando tudo isso junto, nosso JavaScript agora se parece com isso:

    let contador = 0;

    function contar() {
        contador++;
        document.querySelector('h1').innerHTML = contador;

        if (contador % 10 === 0) {
            alert(`Contagem agora é ${contador}`)
        }
    }

    document.addEventListener('DOMContentLoaded', function() {
        document.querySelector('button').onclick = contar;
    });

Outra forma de melhorar nosso design é movendo nosso JavaScript para um arquivo separado. A forma como fazemos isso é muito semelhante à maneira como separamos nosso CSS em um arquivo separado para estilizar:

1. Escreva todo o seu código JavaScript em um arquivo separado que termina com `.js`, talvez `index.js`.
2. Adicione um atributo `src` à tag `<script>` apontando para este novo arquivo.

Para nossa página de contador, poderíamos ter um arquivo chamado `contador.html` que se parece com isso:

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Contagem</title>
            <script src="contador.js"></script>
        </head>
        <body>
            <h1>0</h1>
            <button>Contar</button>
        </body>
    </html>

E um arquivo chamado `contador.js` que se parece com isso:

    let contador = 0;

    function contar() {
        contador++;
        document.querySelector('h1').innerHTML = contador;

        if (contador % 10 === 0) {
            alert(`Contagem agora é ${contador}`)
        }
    }

    document.addEventListener('DOMContentLoaded', function() {
        document.querySelector('button').onclick = contar;
    });

Ter o JavaScript em um arquivo separado é útil por várias razões:

-   Aparência visual: Nossos arquivos individuais de HTML e JavaScript se tornam mais legíveis.
-   Acesso entre arquivos HTML: Agora podemos ter vários arquivos HTML que compartilham o mesmo JavaScript.
-   Colaboração: Agora podemos facilmente ter uma pessoa trabalhando no JavaScript enquanto outra trabalha no HTML.
-   Importação: Podemos importar bibliotecas JavaScript que outras pessoas já escreveram. Por exemplo [Bootstrap](https://getbootstrap.com/docs/4.5/getting-started/introduction/#js) tem sua própria biblioteca JavaScript que você pode incluir para deixar seu site mais interativo.

Vamos começar com outro exemplo de uma página que pode ser um pouco mais interativa. Abaixo, vamos criar uma página onde o usuário pode digitar seu nome para obter uma saudação personalizada.

    <!DOCTYPE html>
    <html lang="pt">
    <head>
        <title>Olá</title>
        <script>
            document.addEventListener('DOMContentLoaded', function() {
                document.querySelector('form').onsubmit = function() {
                    const nome = document.querySelector('#name').value;
                    alert(`Olá, ${nome}`);
                };
            });
        </script>
    </head>
    <body>
        <form>
            <input autofocus id="name" placeholder="Nome" type="text">
            <input type="submit">
        </form>
    </body>
    </html>

![Demo de saudação](https://cs50.harvard.edu/web/2020/notes/5/images/greet.gif)

Alguns pontos sobre a página acima:

-   Usamos o campo `autofocus` no input `name` para indicar que o cursor deve ser colocado dentro desse input assim que a página for carregada.
-   Usamos `#name` dentro de `document.querySelector` para encontrar um elemento com um `id` de `name`. Podemos usar os mesmos seletores nesta função como poderíamos em CSS.
-   Usamos o atributo `value` de um campo de input para obter o que está sendo digitado.

Podemos fazer mais do que apenas adicionar HTML à nossa página usando JavaScript, podemos também alterar o estilo de uma página! Na página abaixo, usamos botões para mudar a cor do nosso cabeçalho.

    <!DOCTYPE html>
    <html lang="pt">
    <head>
         <title>Cores</title>
         <script>
             document.addEventListener('DOMContentLoaded', function() {
                document.querySelectorAll('button').forEach(function(button) {
                    button.onclick = function() {
                        document.querySelector("#hello").style.color = button.dataset.color;
                    }
                });
             });
         </script>
    </head>
    <body>
        <h1 id="hello">Olá</h1>
        <button data-color="red">Vermelho</button>
        <button data-color="blue">Azul</button>
        <button data-color="green">Verde</button>
    </body>
    </html>

![Demo de cores](https://cs50.harvard.edu/web/2020/notes/5/images/colors.gif)

Alguns pontos sobre a página acima:

-   Alteramos o estilo de um elemento usando o atributo `style.SOMETHING`.
-   Usamos o atributo `data-SOMETHING` para atribuir dados a um elemento HTML. Mais tarde, podemos acessar esses dados em JavaScript usando a propriedade `dataset` do elemento.
-   Usamos a função `querySelectorAll` para obter uma [Node List](https://www.w3schools.com/js/js_htmldom_nodelist.asp) (semelhante a uma lista em Python ou um [array em JavaScript](https://www.w3schools.com/js/js_arrays.asp)) com todos os elementos que correspondem à consulta.
-   A função [forEach](https://www.w3schools.com/jsref/jsref_foreach.asp) em JavaScript recebe outra função e aplica essa função a cada elemento em uma lista ou array.

### Funções de Seta <a id="arrow-functions"></a>

Além da notação de função tradicional que já vimos, o JavaScript agora nos dá a capacidade de usar [Funções de Seta](https://www.w3schools.com/js/js_arrow_function.asp) onde temos uma entrada (ou parênteses quando não há entrada) seguida de `=>` seguido por algum código a ser executado. Por exemplo, podemos alterar nosso script acima para usar uma função de seta anônima:

    document.addEventListener('DOMContentLoaded', () => {
        document.querySelectorAll('button').forEach(button => {
            button.onclick = () => {
                document.querySelector("#hello").style.color = button.dataset.color;
            }
        });
    });

Também podemos ter funções nomeadas que usam setas, como nesta reescrita da função de contagem:

    contar = () => {
        contador++;
        document.querySelector('h1').innerHTML = contador;

        if (contador % 10 === 0) {
            alert(`Contagem agora é ${contador}`)
        }
    }

Para ter uma ideia sobre alguns outros eventos que podemos usar, vamos ver como podemos implementar nosso alternador de cores usando um menu suspenso em vez de três botões separados. Podemos detectar alterações em um elemento `select` usando o atributo `onchange`. Em JavaScript, [this](https://www.w3schools.com/js/js_this.asp) é uma palavra-chave que muda com base no contexto em que é usada. No caso de um manipulador de eventos, `this` se refere ao objeto que acionou o evento.

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Cores</title>
            <script>
                document.addEventListener('DOMContentLoaded', function() {
                    document.querySelector('select').onchange = function() {
                        document.querySelector('#hello').style.color = this.value;
                    }
                });
            </script>
        </head>
        <body>
            <h1 id="hello">Olá</h1>
            <select>
                <option value="black">Preto</option>
                <option value="red">Vermelho</option>
                <option value="blue">Azul</option>
                <option value="green">Verde</option>
            </select>

        </body>
    </html>

### Lista de Tarefas <a id="todo-list"></a>

Para juntar algumas coisas que aprendemos nesta aula, vamos trabalhar em criar uma lista de tarefas totalmente em JavaScript. Vamos começar escrevendo o layout HTML da página. Observe abaixo como deixamos espaço para uma lista não ordenada, mas ainda não adicionamos nada a ela. Observe também que adicionamos um link para `tasks.js` onde escreveremos nosso JavaScript.

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Tarefas</title>
            <script src="tasks.js"></script>
        </head>
        <body>
            <h1>Tarefas</h1>
            <ul id="tasks"></ul>
            <form>
                <input id="task" placeholder = "Nova Tarefa" type="text">
                <input id="submit" type="submit">
            </form>
        </body>
    </html>

Agora, aqui está nosso código que podemos manter em `tasks.js`. Algumas observações sobre o que você verá abaixo:

-   Esse código é um pouco diferente do código da aula. Aqui, consultamos nosso botão de envio e campo de tarefa de input apenas uma vez no início e armazenamos esses dois valores nas variáveis `submit` e `novaTarefa`.
-   Podemos habilitar/desabilitar um botão configurando seu atributo `disabled` para `false`/`true`.
-   Em JavaScript, usamos `.length` para encontrar o comprimento de objetos como strings e arrays.
-   No final do script, adicionamos a linha `return false`. Isso impede o envio padrão do formulário, que envolve recarregar a página atual ou redirecionar para uma nova.
-   Em JavaScript, podemos criar elementos HTML usando a função [createElement](https://www.w3schools.com/jsref/met_document_createelement.asp). Podemos então adicionar esses elementos ao DOM usando a função `append`.

```javascript
// Aguardar carregamento da página
document.addEventListener("DOMContentLoaded", function () {
    // Selecionar o botão de envio e o input para serem usados posteriormente
    const submit = document.querySelector("#submit");
    const novaTarefa = document.querySelector("#task");

    // Desativar o botão de envio por padrão:
    submit.disabled = true;

    // Ouvir a digitação no campo de input
    novaTarefa.onkeyup = () => {
        if (novaTarefa.value.length > 0) {
            submit.disabled = false;
        } else {
            submit.disabled = true;
        }
    };

    // Ouvir o envio do formulário
    document.querySelector("form").onsubmit = () => {
        // Encontrar a tarefa que o usuário acabou de enviar
        const tarefa = novaTarefa.value;

        // Criar um item de lista para a nova tarefa e adicionar a tarefa a ele
        const li = document.createElement("li");
        li.innerHTML = tarefa;

        // Adicionar novo elemento à nossa lista não ordenada:
        document.querySelector("#tasks").append(li);

        // Limpar o campo de input:
        novaTarefa.value = "";

        // Desativar o botão de envio novamente:
        submit.disabled = true;

        // Impedir o formulário de ser enviado
        return false;
    };
});
```

## Intervalos <a id="intervalos"></a>

Além de especificar que as funções são executadas quando um evento é acionado, também podemos configurar funções para serem executadas após um determinado período de tempo. Por exemplo, vamos retornar ao script da página do nosso contador e adicionar um intervalo para que, mesmo se o usuário não clicar em nada, o contador incremente a cada segundo. Para fazer isso, utilizamos a função [setInterval](https://www.w3schools.com/jsref/met_win_setinterval.asp), que recebe como argumento uma função a ser executada e um tempo (em milissegundos) entre as execuções da função.

    let contador = 0;

    function contar() {
        contador++;
        document.querySelector('h1').innerHTML = contador;
    }

    document.addEventListener('DOMContentLoaded', function() {
        document.querySelector('button').onclick = contar;

        setInterval(contar, 1000);
    });

## Armazenamento Local

Uma coisa a notar em todos os nossos sites até agora é que toda vez que recarregamos a página, todas as nossas informações são perdidas. A cor do título volta para preto, o contador volta para 0 e todas as tarefas são apagadas. Às vezes isso é o que pretendemos, mas outras vezes queremos ser capazes de armazenar informações que possamos usar quando um usuário retornar ao site.

Uma forma de fazer isso é usando [Armazenamento Local](https://www.w3schools.com/jsref/prop_win_localstorage.asp), ou armazenar informações no navegador da web do usuário que podemos acessar posteriormente. Essas informações são armazenadas como um conjunto de pares de chave-valor, quase como um dicionário Python. Para usar o armazenamento local, empregaremos duas funções principais:

-   `localStorage.getItem(chave)`: Esta função procura por uma entrada no armazenamento local com uma dada chave e retorna o valor associado a essa chave.
-   `localStorage.setItem(chave, valor)`: Esta função define uma entrada no armazenamento local, associando a chave a um novo valor.

Vamos ver como podemos usar essas novas funções para atualizar nosso contador! No código abaixo,

    // Verifica se já existe um valor no armazenamento local
    if (!localStorage.getItem('contador')) {

        // Se não, define o contador como 0 no armazenamento local
        localStorage.setItem('contador', 0);
    }

    function contar() {
        // Recupera o valor do contador do armazenamento local
        let contador = localStorage.getItem('contador');

        // atualiza contador
        contador++;
        document.querySelector('h1').innerHTML = contador;

        // Armazena o contador no armazenamento local
        localStorage.setItem('contador', contador);
    }

    document.addEventListener('DOMContentLoaded', function() {
        // Define o título com o valor atual dentro do armazenamento local
        document.querySelector('h1').innerHTML = localStorage.getItem('contador');
        document.querySelector('button').onclick = contar;
    });

## APIs <a id="apis"></a>

### Objetos JavaScript <a id="objetos-javascript"></a>

Um [Objeto JavaScript](https://www.w3schools.com/js/js_objects.asp) é muito semelhante a um dicionário Python, pois nos permite armazenar pares de chave-valor. Por exemplo, eu poderia criar um Objeto JavaScript representando Harry Potter:

    let person = {
        first: 'Harry',
        last: 'Potter'
    };

Então, posso acessar ou alterar partes desse objeto usando a notação de colchetes ou de ponto:

![Harry Potter](https://cs50.harvard.edu/web/2020/notes/5/images/console.png)

Uma maneira pela qual os Objetos JavaScript são realmente úteis é na transferência de dados de um site para outro, especialmente ao usar [APIs](https://www.mulesoft.com/resources/api/what-is-an-api).

Uma API, ou Interface de Programação de Aplicações, é uma forma estruturada de comunicação entre duas aplicações diferentes.

Por exemplo, podemos querer que nossa aplicação obtenha informações do Google Maps, Amazon ou de algum serviço meteorológico. Podemos fazer isso fazendo chamadas para a API do serviço, que nos retornará dados estruturados, frequentemente no formato [JSON](https://www.w3schools.com/js/js_json_intro.asp) (Notação de Objetos JavaScript). Por exemplo, um voo no formato JSON pode se parecer com isso:

    {
        "origin": "Nova Iorque",
        "destination": "Londres",
        "duration": 415
    }

Os valores dentro de um JSON não precisam ser apenas strings e números como no exemplo acima. Também podemos armazenar listas, ou até mesmo outros Objetos JavaScript:

    {
        "origin": {
            "city": "Nova Iorque",
            "code": "JFK"
        },
        "destination": {
            "city": "Londres",
            "code": "LHR"
        },
        "duration": 415
    }

### Troca de Moeda <a id="currency-exchange"></a>

Para mostrar como podemos usar APIs em nossas aplicações, vamos trabalhar na construção de uma aplicação onde podemos encontrar as taxas de câmbio entre duas moedas. Ao longo do exercício, estaremos utilizando a [API de Taxas de Câmbio do Banco Central Europeu](https://exchangeratesapi.io). Ao visitar o site deles, você verá a documentação da API, que é geralmente um bom ponto de partida quando você deseja utilizar uma API. Podemos testar esta API visitando a URL: [https://api.exchangeratesapi.io/latest?base=USD](https://api.exchangeratesapi.io/latest?base=USD). Quando visitar esta página, você verá a taxa de câmbio entre o Dólar Americano e muitas outras moedas, escritas no formato JSON. Você também pode alterar o parâmetro GET na URL de `USD` para qualquer outro código de moeda para mudar as taxas que você obtém.

Vamos ver como implementar esta API em uma aplicação criando um novo arquivo HTML chamado `currency.html` e vinculando a ele um arquivo JavaScript, mas deixando o corpo vazio:

    <!DOCTYPE html>
    <html lang="pt-br">
        <head>
            <title>Troca de Moeda</title>
            <script src="currency.js"></script>
        </head>
        <body></body>
    </html>

Agora, vamos usar algo chamado [AJAX](https://www.w3schools.com/js/js_ajax_intro.asp), ou JavaScript Assíncrono e XML, que nos permite acessar informações de páginas externas mesmo depois que nossa página foi carregada. Para fazer isso, vamos usar a função [fetch](https://javascript.info/fetch), que nos permitirá enviar uma requisição HTTP. A função `fetch` retorna uma [promessa](https://web.dev/promises/). Não vamos falar sobre os detalhes do que é uma promessa aqui, mas podemos pensar nela como um valor que virá em algum momento, mas não necessariamente imediatamente. Lidamos com promessas dando a elas um atributo `.then` descrevendo o que deve ser feito quando obtermos uma `response`. O trecho de código abaixo irá registrar nossa resposta no console.

    document.addEventListener('DOMContentLoaded', function() {
        // Enviar uma requisição GET para a URL
        fetch('https://api.exchangeratesapi.io/latest?base=USD')
        // Colocar a resposta em formato JSON
        .then(response => response.json())
        .then(data => {
            // Registrar os dados no console
            console.log(data);
        });
    });

![Log de Moeda](https://cs50.harvard.edu/web/2020/notes/5/images/curr_log.png)

Um ponto importante sobre o código acima é que o argumento de `.then` é sempre uma função. Embora pareça que estamos criando as variáveis `response` e `data`, essas variáveis são apenas os parâmetros de duas funções anônimas.

Em vez de simplesmente registrar esses dados, podemos usar JavaScript para exibir uma mensagem na tela, como no código abaixo:

    document.addEventListener('DOMContentLoaded', function() {
        // Enviar uma requisição GET para a URL
        fetch('https://api.exchangeratesapi.io/latest?base=USD')
        // Colocar a resposta em formato JSON
        .then(response => response.json())
        .then(data => {

            // Obter a taxa dos dados
            const rate = data.rates.EUR;

            // Exibir mensagem na tela
            document.querySelector('body').innerHTML = `1 USD é igual a ${rate.toFixed(3)} EUR.`;
        });
    });

![Moeda](https://cs50.harvard.edu/web/2020/notes/5/images/exhange.png)

Agora, vamos tornar o site um pouco mais interativo, permitindo que o usuário escolha qual moeda ele gostaria de ver. Vamos começar alterando nosso HTML para permitir que o usuário insira uma moeda:

    <!DOCTYPE html>
    <html lang="pt-br">
        <head>
            <title>Troca de Moeda</title>
            <script src="currency.js"></script>
        </head>
        <body>
            <form>
                <input id="currency" placeholder="Moeda" type="text">
                <input type="submit" value="Converter">
            </form>
            <div id="result"></div>
        </body>
    </html>

Agora, faremos algumas alterações em nosso JavaScript para que ele só mude quando o formulário for submetido e para levar em conta a entrada do usuário. Também incluiremos alguma verificação de erros aqui:

    document.addEventListener('DOMContentLoaded', function() {
        document.querySelector('form').onsubmit = function() {

            // Enviar uma requisição GET para a URL
            fetch('https://api.exchangeratesapi.io/latest?base=USD')
            // Colocar a resposta em formato JSON
            .then(response => response.json())
            .then(data => {
                // Obter a moeda da entrada do usuário e converter para maiúsculas
                const currency = document.querySelector('#currency').value.toUpperCase();

                // Obter a taxa dos dados
                const rate = data.rates[currency];

                // Verificar se a moeda é válida:
                if (rate !== undefined) {
                    // Exibir a taxa na tela
                    document.querySelector('#result').innerHTML = `1 USD é igual a ${rate.toFixed(3)} ${currency}.`;
                }
                else {
                    // Exibir um erro na tela
                    document.querySelector('#result').innerHTML = 'Moeda Inválida.';
                }
            })
            // Capturar quaisquer erros e registrá-los no console
            .catch(error => {
                console.log('Erro:', error);
            });
            // Impedir a submissão padrão
            return false;
        }
    });
