¿Qué tal otra característica? Supongamos que no tenemos solo una prueba, sino muchas pruebas diferentes en su lugar, y queremos comenzar a organizar esas pruebas en múltiples archivos e incluso una carpeta. Pytest y otros frameworks también admiten ese paradigma. De hecho, déjame ir y probar hello.py usando una carpeta de pruebas, con técnicamente solo una prueba, pero que sería representativa de tener aún más en esa carpeta.

Voy a proceder y crear una nueva carpeta llamada test usando mkdir en mi línea de comando. Y luego, dentro de esa carpeta, voy a proceder y crear un archivo llamado test_hello.py. Dentro de este archivo, mientras tanto, voy a probar lo mismo. Así que voy a proceder y desde hello, importar hello.

Y voy a proceder y definir una función llamada test_default que simplemente prueba el escenario donde hello sin argumentos devuelve hello, coma, mundo. Y voy a tener esa otra función donde pruebo que se pasa un argumento. Y en este caso, elegiré un argumento como afirmar que hello, comillas, comillas, David, es igual a, de hecho, hello, coma, no mundo, sino David.

Así que en este caso, acabo de recrear la misma prueba de antes, pero ahora están en un archivo en una carpeta llamada test. Pytest me permite ejecutar estas pruebas aquí también. Pero para hacerlo, en realidad necesito crear otro archivo. Dentro de mi directorio de pruebas, necesito crear un archivo llamado __init__.py, que tiene el efecto, incluso si este archivo está vacío, de decirle a Python que trate esa carpeta no solo como un módulo, sino como un paquete, por decirlo así.

Un paquete es un módulo de Python o múltiples módulos que se organizan dentro de una carpeta. Y este archivo, __init__.py, es simplemente un indicador visual para Python de que realmente debe tratar esa carpeta como un paquete. Si tuviera más código en esta carpeta, podría hacer aún más cosas con este archivo. Pero por ahora, es solo una pista de que realmente se supone que es un paquete y no solo un módulo o archivo independiente.

Lo que puedo hacer ahora para cerrar es ejecutar pytest, no solo en ese archivo específico, sino en toda una carpeta de pruebas. Entonces, si ejecuto pytest de test, donde test es el nombre de esa carpeta, pytest buscará automáticamente todas las pruebas posibles en esa carpeta, siempre y cuando haya solo esas dos en este archivo, pero cuando lo ejecuto ahora con Enter, todavía pasaré esas pruebas. Todavía obtendré 100%. Y ahora tengo un mecanismo, en última instancia, para probar mi propio código.
