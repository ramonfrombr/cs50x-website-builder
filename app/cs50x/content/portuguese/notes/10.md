## Segurança cibernética

- Hoje será uma visão geral de alto nível de alguns tópicos relacionados à segurança cibernética.
- A segurança cibernética envolve entender como nossos dados estão "seguros" ou "não seguros".

## Senhas

- Uma preocupação de cibersegurança está relacionada às nossas senhas.
- As senhas são um método usado para proteger dados online.
- Existem senhas comuns que as pessoas usam:

        1. 123456
        2. admin
        3. 12345678
        4. 123456789
        5. 1234
        6. 12345
        7. password
        8. 123
        9. Aa123456
        10. 1234567890

- Se você tem uma das senhas acima, é muito provável que milhões de pessoas tenham a mesma senha que você!
- Os adversários no mundo começarão com essa lista.
- Os indivíduos mal-intencionados também podem adivinhar a maioria das heurísticas que você usa para adicionar símbolos à sua senha.
- Os adversários podem usar _ataques de força bruta_, usando um dicionário de senhas para tentar simplesmente todas as senhas possíveis.
- Sua senha provavelmente não é tão segura quanto você pensa que é.

## Segurança de telefone

- Muitos telefones são protegidos por um código de quatro dígitos.
- A forma mais simples de ataque seria tentar todas as senhas possíveis por força bruta.
- Existem 10.000 senhas possíveis ao usar um código de quatro dígitos.
- Se levar um palpite por segundo, levará 10.000 segundos para quebrar a senha.
- No entanto, se um programador criar um programa para gerar todos os códigos possíveis, o tempo necessário seria mínimo. Considere o seguinte código em Python:

    ```python
    from string import digits
    from itertools import product

    for passcode in product(digits, repeat=4):
        print(passcode)
    ```

- Deve ser bastante desconcertante que o código acima possa levar apenas alguns segundos (no máximo!) para descobrir sua senha.
- Poderíamos melhorar nossa segurança mudando para uma senha de quatro letras. Isso resultaria em 7.311.616 senhas possíveis.
- Incluir caracteres maiúsculos e minúsculos criaria mais de 78 milhões de possibilidades.
- Considere como poderíamos modificar seu código para descobrir essas senhas:

    ```python
    from string import ascii_letters
    from itertools import product

    for passcode in product(ascii_letters, repeat=4):
        print(passcode)
    ```

- Podemos até adicionar a capacidade de examinar todas as possíveis senhas de oito dígitos com letras, números e pontuações:

    ```python
    from string import ascii_letters, digits, punctuation
    from itertools import product

    for passcode in product(ascii_letters + digits + punctuation, repeat=8):
        print(passcode)
    ```

- Expandindo para oito caracteres, incluindo letras maiúsculas e minúsculas, números e símbolos, resultaria em 6.095.689.385.410.816 combinações possíveis.
- No mundo digital, você simplesmente deseja que sua senha seja melhor do que as senhas de outras pessoas para que outras sejam atacadas antes de você, pois você é um alvo muito menos conveniente.
- Uma desvantagem de usar uma senha tão longa é a dificuldade de lembrá-la.
- Por isso, existem outras defesas que poderiam ser empregadas para retardar um atacante. Por exemplo, alguns fabricantes de telefone bloqueiam aqueles que adivinham uma senha incorretamente.
- A segurança envolve encontrar um "ponto ideal" entre os compromissos de segurança aprimorada ao mesmo tempo em que mantém a conveniência.

## Gerenciadores de Senhas

-   Os gerenciadores de senhas podem ser usados para criar senhas muito desafiadoras e lembrá-las para você.
-   A probabilidade de uma senha protegida por um gerenciador de senhas ser quebrada é muito, muito baixa.
-   Espera-se que esses gerenciadores de senhas sejam seguros. No entanto, se alguém ganha acesso ao seu gerenciador de senhas, eles teriam acesso a todas as suas senhas.
-   No final, você é muito menos propenso a estar em risco por aqueles com quem você convive - e muito mais provável de estar em risco pelos bilhões de outras pessoas na internet.
-   Conforme mencionado anteriormente, você pode tomar uma decisão com base em um equilíbrio entre segurança e conveniência.

## Autenticação em duas etapas

-   Adicionar outro meio pelo qual você deve autenticar adiciona mais segurança. No entanto, há um custo humano, pois você pode não ter acesso ao seu segundo fator.
-   Esses são implementados como senhas únicas que são enviadas para um e-mail, dispositivo ou número de telefone.
-   Sempre, as políticas de segurança tentam equilibrar as necessidades de segurança e a conveniência humana.

## Hashing

- Suas informações de conta e outros dados sensíveis não devem ser armazenados como texto puro em um banco de dados online.
- Se um banco de dados for comprometido e todas as credenciais forem armazenadas como texto simples, as credenciais de outros serviços em outros sites também serão comprometidas.
- Por isso, algoritmos de hash, como discutido anteriormente neste curso, são usados para armazenar apenas valores hash das senhas.
- O hash unidirecional permite que serviços online nunca armazenem a senha original digitada pelo usuário: apenas o valor hash dessas senhas. Portanto, se houver uma violação, apenas o valor hash será conhecido.
- Tabelas de arco-íris são grandes dicionários que adversários usam para tentar pré-criar possíveis senhas como um meio de tentar quebrar o algoritmo de hash.
- Como um processo adicional de segurança aprimorada, os programadores às vezes podem introduzir o sal onde se torna improvável que vários usuários tenham o mesmo valor hash para representar suas senhas. Você pode imaginar isso da seguinte maneira:

## Criptografia

-   Similar ao hashing, um algoritmo de cifra pode usar uma _chave pública_ e texto para criar o texto cifrado.
-   Por sua vez, uma _chave privada_ e o texto cifrado podem ser inseridos no algoritmo para descriptografar o texto.

## Chaves de acesso

- As chaves de acesso são uma nova tecnologia que surgiu apenas nos últimos meses.
- Através de chaves privadas e um desafio fornecido a um algoritmo, os sites podem autenticar você através da assinatura única criada pelo seu dispositivo.

    ![chave pública e desafio sendo fornecidos a um algoritmo resultando em uma assinatura](cs50Week10Slide125.png)

- Portanto, senhas e nomes de usuário podem em breve se tornar obsoletos.

## Criptografia

- A criptografia é uma forma pela qual os dados são obscurecidos de modo que apenas o remetente e o destinatário pretendido possam lê-los.
- No início deste curso, aprendemos um algoritmo muito simples para "deslocar" o texto por um ou mais caracteres como uma forma rudimentar de criptografia.
- A _criptografia de ponta a ponta_ é uma forma pela qual a criptografia e a descriptografia são feitas no mesmo sistema, sem intermediários. Isso impede que o intermediário ou um ator malicioso possa bisbilhotar seus dados. Zoom e Apple Messages podem usar a criptografia de ponta a ponta.

## Exclusão

-   Ao descartar um arquivo no seu computador ou esvaziar a lixeira, os bits reais do arquivo não são realmente excluídos do seu computador.
-   Em vez disso, restos dos arquivos são deixados.

    ![restos de um arquivo em um disco rígido](cs50Week10Slide136.png)

-   _Exclusão segura_ é quando os restos desses arquivos são transformados em zeros e uns.
-   Ainda assim, alguns restos podem permanecer devido ao que é tornado inacessível pelo sistema operacional.
-   _Criptografia de disco completo_ permite que todo o seu disco rígido seja criptografado. Assim, seus arquivos excluídos são menos acessíveis aos adversários.
-   Considerando a criptografia, é essa mesma tecnologia que os adversários usam para criar _ransomware_ que pode, literalmente, manter seu disco rígido como refém.
