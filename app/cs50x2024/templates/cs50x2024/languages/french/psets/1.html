Ensemble de problèmes 1

À faire

  1. Connectez-vous à submit.cs50.io en utilisant votre compte GitHub et cliquez sur Autoriser cs50, puis fermez l'onglet.
  2. Connectez-vous à cs50.dev en utilisant votre compte GitHub pour accéder à votre propre « codespace ».
  3. Une fois votre codespace chargé, fermez tous les onglets Bienvenue qui pourraient s'être ouverts par défaut.
  4. Exécutez update50 dans la fenêtre du terminal de votre codespace pour vous assurer que votre codespace est à jour et, si vous y êtes invité, cliquez sur Reconstruire maintenant.
  5. Complétez Hello, World.
  6. Soumettez Hello, It’s Me.
  7. Soumettez l'un des exercices suivants :
    • cette version de Mario, si vous vous sentez moins à l'aise
    • cette version de Mario, si vous vous sentez plus à l'aise
  8. Soumettez l'un des exercices suivants :
    • Cash, si vous vous sentez moins à l'aise
    • Credit, si vous vous sentez plus à l'aise

Si vous soumettez les deux versions de Mario, nous enregistrerons le score le plus élevé des deux. Si vous soumettez à la fois Cash et Credit, nous enregistrerons le score le plus élevé des deux.

Quand le faire

Avant 2024-12-31T23:59:00-05:00.

Conseil

  • Essayez l'un des programmes de David du cours via le code source de la semaine 1.
  • Consultez le guide de style de CS50 pour C pour obtenir des conseils sur la façon d'améliorer le style de votre code.
  • Si vous rencontrez des erreurs lors de la compilation de votre code avec make, concentrez-vous d'abord sur la correction de la toute première erreur que vous voyez, en faisant défiler vers le haut si nécessaire. Si vous ne savez pas ce que cela signifie, essayez de demander de l'aide à help50. Par exemple, si vous essayez de compiler hello, et 
    make hello
    génère des erreurs, essayez d'exécuter :

    help50 make hello

    à la place !
