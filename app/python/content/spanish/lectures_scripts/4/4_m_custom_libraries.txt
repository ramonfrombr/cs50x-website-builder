Desafortunadamente, eso es cierto incluso si estoy importando este archivo o simplemente una función de este archivo como lo estoy haciendo aquí en mi programa say.py. Desafortunadamente, para cuando Python haya leído el archivo de arriba abajo, de izquierda a derecha, esa última línea de código, recuerden que es para llamar a main. Se llamará a main pase lo que pase. Así que realmente, la forma correcta de usar una función principal, que resuelve ese problema de asegurarse de que podemos ordenar nuestras funciones como queramos, y todas las funciones se definirán en el momento en que se invoquen, no debería llamar incondicionalmente a main en la parte inferior de este o realmente cualquier otro de mis programas.

En su lugar, debo usar esta técnica. Debo decir if __name__=="__main__", entonces y solo entonces deberías llamar realmente a main. Pero están ahí a la izquierda y a la derecha.

Esta es una variable especial cuyo valor es automáticamente establecido por Python como "main" cuando ejecutas un archivo desde la línea de comandos, como al ejecutar Python sayings.py. Así que vean lo que sucede ahora con esta condicional adicional en sayings.py. Pero noten esto. Name no se establecerá como "main". Se establecerá como algo más, técnicamente el nombre del módulo cuando en su lugar importo el archivo como lo hago aquí. Así que esta línea de código resaltada, aunque hará que Python busque sayings.py, lo lea de arriba abajo, de izquierda a derecha, va a ignorar la llamada a main esta vez porque está envuelta en esa condicional.

En este caso, cuando estoy importando un archivo y no ejecutándolo directamente desde la línea de comandos, main no se ejecutará por definición del valor de ese nombre. Así que voy a probar esto.
