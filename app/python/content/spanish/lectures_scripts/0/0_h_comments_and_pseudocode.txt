Resulta que Python y muchos lenguajes de programación también admiten algo llamado comentarios. Los comentarios son notas para ti mismo en tu código y los incluyes mediante un símbolo especial, en Python va a ser el símbolo de numeral, también llamado de símbolo de almohadilla, y eso te permite escribir el equivalente a una nota para ti mismo pero de una manera que no va a romper tu código. La computadora realmente ignora tu comentario. Solo está ahí para ti, solo está allí para tu profesor, solo está allí para tu colega con el que estás compartiendo ese código en última instancia.

Lo bueno de los comentarios también es esto: los comentarios también pueden servir como una especie de lista de tareas pendientes para ti mismo.

Existe esta noción en la programación de pseudocódigo. El pseudocódigo no es algo formal. No es un lenguaje específico. Es solo usar inglés o tu propio idioma humano para expresar tus pensamientos de manera concisa, metódica, algorítmica, por así decirlo. Pero el pseudocódigo, por lo tanto, porque no es Python y no necesariamente es inglés, simplemente te permite describir tu programa incluso antes. Entonces, por ejemplo, si hoy no estaba seguro de cómo quería escribir este programa, pero no sabía qué quería hacer, podría haber comenzado hoy escribiendo esto en Hello.py.
