## Excepciones <a id="exceptions"></a>

- Las excepciones son cosas que salen mal dentro de nuestro código.
- En nuestro editor de texto, escribe `code hello.py` para crear un archivo nuevo. Escribe lo siguiente (con los errores intencionales incluidos):

      print("hello, world)

  Nota que intencionalmente omitimos una comilla.

- Al ejecutar `python hello.py` en nuestra ventana de terminal, se muestra un error. El compilador indica que es un "error de sintaxis". Los errores de sintaxis son aquellos que requieren que verifiques dos veces que hayas escrito correctamente en tu código.
- Puedes aprender más en la documentación de Python sobre [Errores y Excepciones](https://docs.python.org/3/tutorial/errors.html).

## Errores de tiempo de ejecución <a id="errores-de-tiempo-de-ejecucion"></a>

- Los errores de tiempo de ejecución se refieren a aquellos creados por un comportamiento inesperado dentro de tu código. Por ejemplo, tal vez tenías la intención de que un usuario ingresara un número, pero en cambio ingresó un carácter. Tu programa puede arrojar un error debido a esta entrada inesperada del usuario.
- En tu ventana de terminal, ejecuta `code number.py`. El código en tu editor de texto se ve así:

      x = int(input("¿Cuál es el valor de x? "))
      print(f"x es {x}")

  Observa que al incluir la `f`, le indicamos a Python que interpolate lo que está entre llaves como el valor de `x`. Además, al probar tu código, puedes imaginar cómo alguien podría fácilmente ingresar una cadena o un carácter en lugar de un número. Incluso, un usuario podría no ingresar nada, simplemente presionando la tecla "Enter".

- Como programadores, debemos ser defensivos para asegurar que nuestros usuarios ingresen lo que esperábamos. Podríamos considerar "casos límite" como `-1`, `0`, o `gato`.
- Si ejecutamos este programa e ingresamos "gato", de repente veremos `ValueError: invalid literal for int() with base 10: 'gato'`. Básicamente, el intérprete de Python no acepta que pasemos "gato" a la función `print`.
- Una estrategia efectiva para corregir este error potencial sería implementar "manejo de errores" para asegurarnos de que el usuario se comporte como esperamos.
- Puedes aprender más en la documentación de Python sobre [Errores y Excepciones](https://docs.python.org/3/tutorial/errors.html).

## `try` <a id="try"></a>

- En Python, `try` y `except` son formas de probar la entrada del usuario antes de que algo salga mal. Modifica tu código de la siguiente manera:

      try:
          x = int(input("¿Cuál es x?"))
          print(f"x es {x}")
      except ValueError:
          print("x no es un entero")

  Observa cómo, al ejecutar este código, se aceptará la entrada de `50`. Sin embargo, si se escribe `gato`, se producirá un error visible para el usuario, indicándoles por qué su entrada no fue aceptada.

- Aún así, esta no es la mejor forma de implementar este código. Observa que estamos tratando de hacer dos líneas de código. Para seguir las mejores prácticas, solo deberíamos `try` la menor cantidad de líneas de código posible que podrían fallar. Ajusta tu código de la siguiente manera:

      try:
          x = int(input("¿Cuál es x?"))
      except ValueError:
          print("x no es un entero")

      print(f"x es {x}")

  Observa que, si bien esto logra nuestro objetivo de intentar la menor cantidad de líneas posible, ¡ahora nos enfrentamos a un nuevo error! Nos encontramos con un `NameError` donde `x no está definido`. Observa este código y considera: ¿Por qué x no está definido en algunos casos?

- De hecho, si examinas el orden de las operaciones en `x = int(input("¿Cuál es x?"))`, trabajando de derecha a izquierda, podría tomar un carácter incorrectamente introducido e intentar asignarlo como un entero. Si esto falla, la asignación del valor de `x` nunca ocurre. Por lo tanto, no hay ningún x para imprimir en nuestra última línea de código.

## `else` <a id="else"></a>

- Resulta que hay otra manera de implementar `try` que podría capturar errores de esta naturaleza.
- Ajusta tu código de la siguiente manera:

      try:
          x = int(input("¿Cuál es x?"))
      except ValueError:
          print("x no es un entero")
      else:
          print(f"x es {x}")

  Observa que si no ocurre ninguna excepción, entonces se ejecutará el bloque de código dentro de `else`. Al ejecutar `python number.py` y proporcionar `50`, notarás que se imprimirá el resultado. Al intentarlo nuevamente, esta vez proporcionando `cat`, notarás que el programa ahora captura el error.

- Considerando mejorar nuestro código, observa que estamos siendo un poco descorteses con nuestro usuario. Si nuestro usuario no coopera, actualmente simplemente finalizamos nuestro programa. Considera cómo podemos usar un bucle para solicitar al usuario el valor de `x` ¡y si no lo proporciona, pedir nuevamente! Mejora tu código de la siguiente manera:

      while True:
          try:
              x = int(input("¿Cuál es x?"))
          except ValueError:
              print("x no es un entero")
          else:
              break

      print(f"x es {x}")

  Observa que `while True` se ejecutará en bucle infinitamente. Si el usuario logra proporcionar la entrada correcta, podemos salir del bucle y luego imprimir la salida. Ahora, un usuario que ingrese algo incorrectamente será solicitado a ingresar nuevamente.

## Creando una función para obtener un entero <a id="creando-una-función-para-obtener-un-entero"></a>

- Seguramente, hay muchas ocasiones en las que querríamos obtener un entero de nuestro usuario. Modifica tu código de la siguiente manera:

      def main():
          x = get_int()
          print(f"x es {x}")


      def get_int():
          while True:
              try:
                  x = int(input("¿Cuál es x?"))
              except ValueError:
                  print("x no es un número entero")
              else:
                  break
          return x


      main()

  Observa que estamos manifestando muchas propiedades excelentes. Primero, hemos abstraído la capacidad de obtener un entero. Ahora, este programa se reduce a las primeras tres líneas del programa.

- Aun así, podemos mejorar este programa. Considera qué más podrías hacer para mejorarlo. Modifica tu código de la siguiente manera:

      def main():
          x = get_int()
          print(f"x es {x}")


      def get_int():
          while True:
              try:
                  x = int(input("¿Cuál es x?"))
              except ValueError:
                  print("x no es un número entero")
              else:
                  return x


      main()

  Observa que `return` no solo te sacará de un bucle, sino que también devolverá un valor.

- Algunas personas podrían argumentar que podrías hacer lo siguiente:

      def main():
          x = get_int()
          print(f"x es {x}")


      def get_int():
          while True:
              try:
                  return int(input("¿Cuál es x?"))
              except ValueError:
                  print("x no es un número entero")


      main()

  Observa que esto hace lo mismo que la iteración anterior de nuestro código, simplemente con menos líneas.

## `pass` <a id="pass"></a>

- Podemos modificar nuestro código para que no advierta al usuario, sino simplemente le vuelva a preguntar nuestra pregunta de inicio, de la siguiente manera:

      def main():
          x = get_int()
          print(f"x es {x}")


      def get_int():
          while True:
              try:
                  return int(input("¿Qué es x?"))
              except ValueError:
                  pass


      main()

  Observa que nuestro código seguirá funcionando pero no informará al usuario repetidamente sobre su error. En algunos casos, querrás ser muy claro con el usuario sobre el error que se está produciendo. En otros casos, podrías decidir simplemente pedirles la entrada nuevamente.

- Una última mejora que podría mejorar la implementación de esta función `get_int`. Ahora, observa que actualmente estamos confiando en el sistema de honor de que la variable `x` está presente en las funciones `main` y `get_int`. Probablemente queramos pasar un mensaje para que el usuario lo vea cuando se le pida la entrada. Modifica tu código de la siguiente manera.

      def main():
          x = get_int("¿Qué es x? ")
          print(f"x es {x}")


      def get_int(prompt):
          while True:
              try:
                  return int(input(prompt))
              except ValueError:
                  pass


      main()

- Puedes aprender más en la documentación de Python sobre [`pass`](https://docs.python.org/3/tutorial/controlflow.html#pass-statements).
