## Variáveis <a id="variaveis"></a>

Uma parte fundamental de qualquer linguagem de programação é a capacidade de criar e manipular variáveis. Para atribuir um valor a uma variável em Python, a sintaxe é a seguinte:

    a = 28
    b = 1.5
    c = "Olá!"
    d = True
    e = None

Cada uma dessas linhas está atribuindo o valor à direita do `=`, e armazenando-o no nome da variável à esquerda.

Ao contrário de algumas outras linguagens de programação, os tipos de variáveis em Python são inferidos, o que significa que embora cada variável tenha um tipo, não precisamos declarar explicitamente qual tipo é quando criamos a variável. Alguns dos tipos de variáveis mais comuns são:

- **int**: Um número inteiro
- **float**: Um número decimal
- **str**: Uma string, ou sequência de caracteres
- **bool**: Um valor que é ou `True` ou `False`
- **NoneType**: Um valor especial (`None`) que indica a ausência de valor.

Agora, vamos trabalhar em escrever um programa mais interessante que pode receber entrada do usuário e cumprimentar esse usuário. Para isso, vamos usar outra função integrada chamada `input`, que exibe um prompt ao usuário e retorna o que o usuário fornece como entrada. Por exemplo, podemos escrever o seguinte em um arquivo chamado `nome.py`:

    nome = input("Nome: ")
    print("Olá, " + nome)

Quando executado no terminal, é assim que o programa parece:

![Olá](https://cs50.harvard.edu/web/2020/notes/2/images/hello.png)

Algumas coisas para destacar aqui:

- Na primeira linha, em vez de atribuir o nome da variável a um valor explícito, estamos atribuindo-a ao que a função `input` retorna.
- Na segunda linha, estamos usando o operador `+` para combinar, ou **concatenar**, duas strings. Em Python, o operador `+` pode ser usado para adicionar números ou concatenar strings e listas.

## Formatação de Strings <a id="formatting-strings"></a>

- Enquanto podemos usar o operador `+` para combinar strings como fizemos anteriormente, nas últimas versões do Python, existem maneiras ainda mais fáceis de trabalhar com strings, conhecidas como [strings formatadas](https://realpython.com/python-f-strings/), ou **f-strings** para abreviar.
- Para indicar que estamos usando strings formatadas, simplesmente adicionamos um `f` antes das aspas. Por exemplo, em vez de usar `"Olá, " + nome` como fizemos anteriormente, poderíamos escrever `f"Olá, {nome}"` para o mesmo resultado. Podemos até inserir uma função nessa string, se quisermos, e transformar nosso programa acima em uma única linha:

```
print(f"Olá, {input("Nome: ")}")
```

## Condições <a id="conditions"></a>

- Assim como em outras linguagens de programação, o Python nos dá a capacidade de executar diferentes segmentos de código com base em diferentes [condições](https://realpython.com/python-conditional-statements/). Por exemplo, no programa abaixo, iremos alterar nossa saída dependendo do número que o usuário digitar:

```
num = input("Número: ")
if num > 0:
    print("Número é positivo")
elif num < 0:
    print("Número é negativo")
else:
    print("Número é 0")
```

- Entrando em como o programa acima funciona, as condicionais em python contêm uma palavra-chave (`if`, `elif` ou `else`) e então (exceto no caso de `else`) uma expressão booleana, ou uma expressão que avalia para `True` ou `False`. Em seguida, todo o código que queremos executar se uma determinada expressão for verdadeira é **indentado** diretamente abaixo da declaração. A endentação é obrigatória como parte da sintaxe do Python.
- No entanto, ao executar este programa, nos deparamos com uma [exceção](https://docs.python.org/3/tutorial/errors.html) que se parece com isso:

![condição](https://cs50.harvard.edu/web/2020/notes/2/images/cond.png)

- Uma exceção é o que acontece quando um erro ocorre enquanto estamos executando nosso código Python, e com o tempo você ficará melhor em interpretar esses erros, o que é uma habilidade muito valiosa de se ter.
- Vamos olhar um pouco mais de perto esta exceção específica: Se olharmos na parte inferior, veremos que encontramos um `TypeError`, que geralmente significa que o Python esperava que uma determinada variável fosse de um tipo, mas descobriu que era de outro tipo. Neste caso, a exceção nos diz que não podemos usar o símbolo `>` para comparar uma `str` e um `int`, e então acima podemos ver que essa comparação ocorre na linha 2.
- Neste caso, é óbvio que `0` é um inteiro, então deve ser o caso de que nossa variável `num` é uma string. Isso está acontecendo porque a função `input` sempre retorna uma string, e precisamos especificar que ela deve ser convertida (ou **convertida**) em um inteiro usando a função `int`. Isso significa que nossa primeira linha agora seria assim:

```
num = int(input("Número: "))
```

- Agora, o programa funcionará exatamente como pretendíamos!

## Sequências <a id="sequences"></a>

Uma das partes mais poderosas da linguagem Python é sua capacidade de trabalhar com **sequências** de dados, além de variáveis individuais.

Existem vários tipos de sequências que são semelhantes de certas maneiras, mas diferentes em outras. Ao explicar essas diferenças, utilizaremos os termos **mutável/imutável** e **ordenado/não ordenado**. **Mutável** significa que uma vez que uma sequência foi definida, podemos alterar elementos individuais dessa sequência, e **ordenado** significa que a ordem dos objetos é importante.

### Strings <a id="strings"></a>

**Ordenado**: Sim

**Mutável**: Não

Já vimos um pouco sobre strings, mas em vez de apenas variáveis, podemos pensar em uma string como uma sequência de caracteres. Isso significa que podemos acessar elementos individuais dentro da string! Por exemplo:

    name = "Harry"
    print(name[0])
    print(name[1])

imprime o primeiro (ou índice 0) caractere na string, que neste caso é `H`, e em seguida imprime o segundo (ou índice 1) caractere, que é `a`.

### Listas <a id="lists"></a>

**Ordenado**: Sim

**Mutável**: Sim

Uma [lista Python](https://www.w3schools.com/python/python_lists.asp) permite armazenar qualquer tipo de variável. Criamos uma lista usando colchetes e vírgulas, conforme mostrado abaixo. De forma semelhante às strings, podemos imprimir uma lista inteira, ou alguns elementos individuais. Também podemos adicionar elementos a uma lista usando `append`, e classificar uma lista usando `sort`.

    # Este é um comentário Python
    names = ["Harry", "Ron", "Hermione"]
    # Imprimir a lista inteira:
    print(names)
    # Imprimir o segundo elemento da lista:
    print(names[1])
    # Adicionar um novo nome à lista:
    names.append("Draco")
    # Classificar a lista:
    names.sort()
    # Imprimir a nova lista:
    print(names)

![Nomes](https://cs50.harvard.edu/web/2020/notes/2/images/list.png)

### Tuplas <a id="tuples"></a>

**Ordenado**: Sim

**Mutável**: Não

[Tuplas](https://www.w3schools.com/python/python_tuples.asp) são geralmente usadas quando é necessário armazenar apenas dois ou três valores juntos, como os valores de x e y para um ponto. No código Python, usamos parênteses:

    point = (12.5, 10.6)

### Conjuntos <a id="sets"></a>

**Ordenado**: Não

**Mutável**: N/A

[Conjuntos](https://www.w3schools.com/python/python_sets.asp) são diferentes de listas e tuplas por serem **não ordenados**. Também são diferentes porque, enquanto é possível ter dois ou mais dos mesmos elementos em uma lista/tupla, um conjunto armazenará cada valor apenas uma vez. Podemos definir um conjunto vazio usando a função `set`. Em seguida, podemos usar `add` e `remove` para adicionar e remover elementos desse conjunto, e a função `len` para encontrar o tamanho do conjunto. Observe que a função `len` funciona em todas as sequências em Python. Também observe que, apesar de adicionar `4` e `3` ao conjunto duas vezes, cada item só pode aparecer uma vez em um conjunto.

    # Criar um conjunto vazio:
    s = set()

    # Adicionar alguns elementos:
    s.add(1)
    s.add(2)
    s.add(3)
    s.add(4)
    s.add(3)
    s.add(1)

    # Remover o 2 do conjunto
    s.remove(2)

    # Imprimir o conjunto:
    print(s)

    # Encontrar o tamanho do conjunto:
    print(f"O conjunto tem {len(s)} elementos.")

    """ Este é um comentário Python de várias linhas:
    Saída:
    {1, 3, 4}
    O conjunto tem 3 elementos.
    """

### Dicionários <a id="dictionaries"></a>

**Ordenado**: Não

**Mutável**: Sim

[Os dicionários Python](https://www.w3schools.com/python/python_dictionaries.asp) ou `dict`s, serão especialmente úteis neste curso. Um dicionário é um conjunto de **pares chave-valor**, onde cada chave tem um valor correspondente, assim como em um dicionário, cada palavra (a chave) tem uma definição correspondente (o valor). Em Python, usamos chaves para definir um dicionário, e dois pontos para indicar chaves e valores. Por exemplo:

    # Definir um dicionário
    houses = {"Harry": "Gryffindor", "Draco": "Slytherin"}
    # Imprimir a casa do Harry
    print(houses["Harry"])
    # Adicionando valores a um dicionário:
    houses["Hermione"] = "Gryffindor"
    # Imprimir a Casa da Hermione:
    print(houses["Hermione"])

    """ Saída:
    Grifinória
    Grifinória
    """

### Loops <a id="loops"></a>

Os loops são uma parte incrivelmente importante de qualquer linguagem de programação, e em Python, existem dois tipos principais: [for loops](https://www.w3schools.com/python/python_for_loops.asp) e [while loops](https://www.w3schools.com/python/python_while_loops.asp). Por enquanto, vamos nos concentrar nos Loops For.

- Os loops For são usados para iterar sobre uma sequência de elementos, executando um bloco de código (indentado abaixo) para cada elemento em uma sequência. Por exemplo, o código a seguir imprimirá os números de 0 a 5:

```
for i in [0, 1, 2, 3, 4, 5]:
    print(i)

""" Saída:
0
1
2
3
4
5
"""
```

- Podemos condensar este código usando a função `range` do Python, que nos permite obter facilmente uma sequência de números. O código a seguir dá exatamente o mesmo resultado que o nosso código anterior:

```
for i in range(6):
    print(i)

""" Saída:
0
1
2
3
4
5
"""
```

- Esse tipo de loop pode funcionar para qualquer sequência! Por exemplo, se quisermos imprimir cada nome em uma lista, poderíamos escrever o código abaixo:

```
# Criar uma lista:
names = ["Harry", "Ron", "Hermione"]

# Imprimir cada nome:
for name in names:
    print(name)

""" Saída:
Harry
Ron
Hermione
"""
```

- Podemos ser ainda mais específicos se quisermos e iterar por cada caractere em um único nome!

```
name = "Harry"
for char in name:
    print(char)

""" Saída:
H
a
r
r
y
"""
```

## Módulos <a id="modules"></a>

À medida que nossos projetos crescem, será útil poder escrever funções em um arquivo e executá-las em outro. No caso acima, poderíamos criar um arquivo chamado `functions.py` com o código:

    def square(x):
        return x * x

E outro arquivo chamado `square.py` com o código:

    for i in range(10):
        print(f"O quadrado de {i} é {square(i)}")

No entanto, quando tentamos executar `square.py`, encontramos o seguinte erro:

![Erro de Nome](https://cs50.harvard.edu/web/2020/notes/2/images/NameError.png)

Enfrentamos esse problema porque, por padrão, os arquivos Python não sabem sobre os outros, então precisamos `importar` explicitamente a função square do **módulo** `functions` que acabamos de escrever. Agora, quando `square.py` estiver assim:

    from functions import square

    for i in range(10):
        print(f"O quadrado de {i} é {square(i)}")

Alternativamente, podemos optar por importar o módulo inteiro `functions` e usar a notação de ponto para acessar a função `square`:

    import functions

    for i in range(10):
        print(f"O quadrado de {i} é {functions.square(i)}")

Existem muitos módulos integrados do Python que podemos importar, como `math` ou `csv`, que nos dão acesso a ainda mais funções. Além disso, podemos baixar ainda mais módulos para obter funcionalidades adicionais! Passaremos muito tempo usando o módulo `Django`, o qual discutiremos na próxima aula.

## Programação Orientada a Objetos <a id="object-oriented-programming"></a>

A [Programação Orientada a Objetos](https://pt.wikipedia.org/wiki/Programa%C3%A7%C3%A3o_orientada_a_objetos) é um paradigma de programação, ou uma forma de pensar sobre programação, que se concentra em objetos que podem armazenar informações e realizar ações.

- **Classes**: Já vimos alguns tipos diferentes de variáveis em Python, mas e se quisermos criar nosso próprio tipo? Uma [Classe Python](https://www.w3schools.com/python/python_classes.asp) é essencialmente um modelo para um novo tipo de objeto que pode armazenar informações e realizar ações. Aqui está uma classe que define um ponto bidimensional:

```
class Point():
    # Um método que define como criar um ponto:
    def __init__(self, x, y):
        self.x = x
        self.y = y
```

- Observe que no código acima, usamos a palavra-chave `self` para representar o objeto com o qual estamos trabalhando. `self` deve ser o primeiro argumento para qualquer método dentro de uma classe Python.

Agora, vamos ver como podemos realmente usar a classe acima para criar um objeto:

    p = Point(2, 8)
    print(p.x)
    print(p.y)

    """ Saída:
    2
    8
    """

Agora, vamos ver um exemplo mais interessante, onde, em vez de armazenar apenas as coordenadas de um Ponto, criamos uma classe que representa um voo de companhia aérea:

    class Flight():
        # Método para criar um novo voo com capacidade fornecida
        def __init__(self, capacity):
            self.capacity = capacity
            self.passengers = []

        # Método para adicionar um passageiro ao voo:
        def add_passenger(self, name):
            self.passengers.append(name)

No entanto, essa classe é falha porque, embora tenhamos definido uma capacidade, ainda poderíamos adicionar passageiros demais. Vamos melhorá-la para que, antes de adicionar um passageiro, verifiquemos se há lugar no voo:

    class Flight():
        # Método para criar um novo voo com capacidade fornecida
        def __init__(self, capacity):
            self.capacity = capacity
            self.passengers = []

        # Método para adicionar um passageiro ao voo:
        def add_passenger(self, name):
            if not self.open_seats():
                return False
            self.passengers.append(name)
            return True

        # Método para retornar o número de assentos disponíveis
        def open_seats(self):
            return self.capacity - len(self.passengers)

Observe que acima, usamos a linha `if not self.open_seats()` para determinar se há assentos disponíveis ou não. Isso funciona porque em Python, o número 0 pode ser interpretado como significando `False`, e também podemos usar a palavra-chave `not` para indicar o oposto da declaração seguinte, então `not True` é `False` e `not False` é `True`. Portanto, se `open_seats` retornar 0, toda a expressão será avaliada como `True`.

Agora, vamos testar a classe que criamos instanciando alguns objetos:

    # Criar um novo voo com capacidade de até 3 passageiros
    flight = Flight(3)

    # Criar uma lista de pessoas
    pessoas = ["Harry", "Ron", "Hermione", "Ginny"]

    # Tentar adicionar cada pessoa da lista a um voo
    for pessoa in pessoas:
        if flight.add_passenger(pessoa):
            print(f"{pessoa} adicionado ao voo com sucesso")
        else:
            print(f"Não há assentos disponíveis para {pessoa}")

    """ Saída:
    Harry adicionado ao voo com sucesso
    Ron adicionado ao voo com sucesso
    Hermione adicionado ao voo com sucesso
    Não há assentos disponíveis para Ginny
    """

## Programação Funcional <a id="functional-programming"></a>

Além de suportar a Programação Orientada a Objetos, o Python também suporta o [Paradigma da Programação Funcional](https://en.wikipedia.org/wiki/Functional_programming), no qual as funções são tratadas como valores assim como qualquer outra variável.

### Decoradores <a id="decorators"></a>

Uma coisa possibilitada pela programação funcional é a ideia de um decorador, que é uma função de ordem superior que pode modificar outra função. Por exemplo, vamos escrever um decorador que anuncia quando uma função está prestes a começar e quando termina. Então, podemos aplicar este decorador usando o símbolo `@`.

    def announce(f):
        def wrapper():
            print("Prestes a executar a função")
            f()
            print("Concluído com a função")
        return wrapper

    @announce
    def hello():
        print("Olá, mundo!")

    hello()

    """ Saída:
    Prestes a executar a função
    Olá, mundo!
    Concluído com a função
    """

### Funções Lambda <a id="lambda-functions"></a>

As funções lambda fornecem outra maneira de criar funções em python. Por exemplo, se quisermos definir a mesma função `square` que fizemos anteriormente, podemos escrever:

    square = lambda x: x * x

Onde a entrada está à esquerda do `:` e a saída está à direita.

Isso pode ser útil quando não queremos escrever uma função separada para um uso único e pequeno. Por exemplo, se quisermos classificar alguns objetos onde não está claro a princípio como classificá-los. Imagine que temos uma lista de pessoas, mas com nomes e casas em vez de apenas nomes que desejamos classificar:

    pessoas = [
        {"nome": "Harry", "casa": "Grifinória"},
        {"nome": "Cho", "casa": "Corvinal"},
        {"nome": "Draco", "casa": "Sonserina"}
    ]

    pessoas.sort()

    print(pessoas)

No entanto, isso nos deixa com o erro:

![Erro de Tipo](https://cs50.harvard.edu/web/2020/notes/2/images/TypeError.png)

O qual ocorre porque o Python não sabe como comparar dois dicionários para verificar se um é menor que o outro.

Podemos resolver esse problema incluindo um argumento `key` na função de classificação, que especifica qual parte do dicionário queremos usar para classificar:

    pessoas = [
        {"nome": "Harry", "casa": "Grifinória"},
        {"nome": "Cho", "casa": "Corvinal"},
        {"nome": "Draco", "casa": "Sonserina"}
    ]

    def f(pessoa):
        return pessoa["nome"]

    pessoas.sort(key=f)

    print(pessoas)

    """ Saída:
    [{'nome': 'Cho', 'casa': 'Corvinal'}, {'nome': 'Draco', 'casa': 'Sonserina'}, {'nome': 'Harry', 'casa': 'Grifinória'}]
    """

Embora isso funcione, tivemos que escrever uma função inteira que estamos usando apenas uma vez, podemos tornar nosso código mais legível usando uma função lambda:

    pessoas = [
        {"nome": "Harry", "casa": "Grifinória"},
        {"nome": "Cho", "casa": "Corvinal"},
        {"nome": "Draco", "casa": "Sonserina"}
    ]

    pessoas.sort(key=lambda pessoa: pessoa["nome"])

    print(pessoas)

    """ Saída:
    [{'nome': 'Cho', 'casa': 'Corvinal'}, {'nome': 'Draco', 'casa': 'Sonserina'}, {'nome': 'Harry', 'casa': 'Grifinória'}]
    """

## Exceções <a id="exceptions"></a>

Durante esta aula, nos deparamos com algumas exceções diferentes, então agora vamos investigar algumas novas formas de lidar com elas.

No trecho de código a seguir, vamos pegar dois inteiros do usuário e tentar dividi-los:

```python
x = int(input("x: "))
y = int(input("y: "))

result = x / y

print(f"{x} / {y} = {result}")
```

Em muitos casos, este programa funciona bem:

No entanto, encontraremos problemas ao tentar dividir por 0:

![Mau exemplo](https://cs50.harvard.edu/web/2020/notes/2/images/dividebad.png)

Podemos lidar com esse erro confuso usando [Tratamento de Exceção](https://www.w3schools.com/python/python_try_except.asp). No bloco de código a seguir, vamos tentar dividir os dois números, mas tratar quando ocorrer um `ZeroDivisionError`:

```python
import sys

x = int(input("x: "))
y = int(input("y: "))

try:
    result = x / y
except ZeroDivisionError:
    print("Erro: Não é possível dividir por 0.")
    # Sair do programa
    sys.exit(1)

print(f"{x} / {y} = {result}")
```

Entretanto, ainda encontramos um erro quando o usuário insere valores não numéricos para x e y:

![Erro de valor](https://cs50.harvard.edu/web/2020/notes/2/images/valueError.png)

Podemos resolver esse problema de maneira semelhante!

```python
import sys

try:
    x = int(input("x: "))
    y = int(input("y: "))
except ValueError:
    print("Erro: Entrada inválida")
    sys.exit(1)

try:
    result = x / y
except ZeroDivisionError:
    print("Erro: Não é possível dividir por 0.")
    # Sair do programa
    sys.exit(1)

print(f"{x} / {y} = {result}")
```
