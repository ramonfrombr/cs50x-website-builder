## `set` <a id="set"></a>

- Em matemática, um conjunto seria considerado um conjunto de números sem duplicatas.
- Na janela do editor de texto, o código é o seguinte:

      students = [
          {"name": "Hermione", "house": "Gryffindor"},
          {"name": "Harry", "house": "Gryffindor"},
          {"name": "Ron", "house": "Gryffindor"},
          {"name": "Draco", "house": "Slytherin"},
          {"name": "Padma", "house": "Ravenclaw"},
      ]

      houses = []
      for student in students:
          if student["house"] not in houses:
              houses.append(student["house"])

      for house in sorted(houses):
          print(house)

  Observe como temos uma lista de dicionários, cada um representando um estudante. Uma lista vazia chamada `houses` é criada. Iteramos por cada `student` em `students`. Se a `house` de um estudante não estiver em `houses`, adicionamos à nossa lista de `houses`.

- Acontece que podemos usar as funcionalidades embutidas do `set` para eliminar duplicatas.
- Na janela do editor de texto, o código é o seguinte:

      students = [
          {"name": "Hermione", "house": "Gryffindor"},
          {"name": "Harry", "house": "Gryffindor"},
          {"name": "Ron", "house": "Gryffindor"},
          {"name": "Draco", "house": "Slytherin"},
          {"name": "Padma", "house": "Ravenclaw"},
      ]

      houses = set()
      for student in students:
          houses.add(student["house"])

      for house in sorted(houses):
          print(house)

  Observe como não é necessário fazer nenhuma verificação para garantir que não haja duplicatas. O objeto `set` cuida disso automaticamente para nós.

- Você pode aprender mais na documentação do Python sobre o [`set`](https://docs.python.org/3/library/stdtypes.html#set).

## Variáveis Globais <a id="global-variables"></a>

- Em outras linguagens de programação, há a noção de variáveis globais que são acessíveis por qualquer função.
- Podemos aproveitar essa capacidade no Python. Na janela do editor de texto, escreva o código a seguir:

      balance = 0


      def main():
          print("Saldo:", balance)


      if __name__ == "__main__":
          main()

  Note como criamos uma variável global chamada `balance`, fora de qualquer função.

- Como nenhum erro é apresentado ao executar o código acima, você pode pensar que está tudo bem. No entanto, não está! Na janela do editor de texto, escreva o código a seguir:

      balance = 0


      def main():
          print("Saldo:", balance)
          deposit(100)
          withdraw(50)
          print("Saldo:", balance)


      def deposit(n):
          balance += n


      def withdraw(n):
          balance -= n


      if __name__ == "__main__":
          main()

  Note como agora adicionamos a funcionalidade de adicionar e retirar fundos em `balance`. No entanto, ao executar esse código, é apresentado um erro! Vemos um erro chamado `UnboundLocalError`. Você pode deduzir que, pelo menos da maneira como codificamos `balance` e nossas funções `deposit` e `withdraw`, não podemos atribuir a ele um valor dentro de uma função.

- Para interagir com uma variável global dentro de uma função, a solução é usar a palavra-chave `global`. Na janela do editor de texto, escreva o código a seguir:

      balance = 0


      def main():
          print("Saldo:", balance)
          deposit(100)
          withdraw(50)
          print("Saldo:", balance)


      def deposit(n):
          global balance
          balance += n


      def withdraw(n):
          global balance
          balance -= n


      if __name__ == "__main__":
          main()

  Note como a palavra-chave `global` informa a cada função que `balance` não se refere a uma variável local: em vez disso, se refere à variável global que colocamos originalmente no topo do nosso código. Agora, nosso código funciona!

- Utilizando nossos conhecimentos em programação orientada a objetos, podemos modificar nosso código para usar uma classe em vez de uma variável global. Na janela do editor de texto, escreva o código a seguir:

      class Conta:
          def __init__(self):
              self._saldo = 0

          @property
          def saldo(self):
              return self._saldo

          def depositar(self, n):
              self._saldo += n

          def sacar(self, n):
              self._saldo -= n


      def main():
          conta = Conta()
          print("Saldo:", conta.saldo)
          conta.depositar(100)
          conta.sacar(50)
          print("Saldo:", conta.saldo)


      if __name__ == "__main__":
          main()

  Note como usamos `conta = Conta()` para criar uma conta. As classes nos permitem resolver melhor esse problema de precisar de uma variável global, porque essas variáveis de instância são acessíveis a todos os métodos dessa classe utilizando `self`.

- Em geral, variáveis globais devem ser usadas com muita parcimônia, se é que devem ser usadas!

## Constantes <a id="constantes"></a>

- Algumas linguagens permitem criar variáveis que são imutáveis, chamadas de "constantes". As constantes permitem programar defensivamente e reduzir as oportunidades de alteração de valores importantes.
- Na janela do editor de texto, escreva o código da seguinte forma:

      MEOWS = 3

      for _ in range(MEOWS):
          print("meow")

  Observe que `MEOWS` é a nossa constante neste caso. As constantes são normalmente indicadas por nomes de variáveis em maiúsculas e são colocadas no topo do nosso código. Embora isso _pareça_ uma constante, na realidade, o Python não tem um mecanismo para impedir que mudemos esse valor dentro do nosso código! Em vez disso, você deve seguir a ética: se um nome de variável estiver escrito todo em maiúsculas, não o altere!

- Pode-se criar uma "constante" de classe, agora entre aspas porque sabemos que o Python não suporta "constantes". Na janela do editor de texto, escreva o código da seguinte forma:

      class Cat:
          MEOWS = 3

          def meow(self):
              for _ in range(Cat.MEOWS):
                  print("meow")


      cat = Cat()
      cat.meow()

  Como `MEOWS` é definido fora de qualquer método específico da classe, todos eles têm acesso a esse valor por meio de `Cat.MEOWS`.

## Dicas de Tipos <a id="type-hints"></a>

- Em outras linguagens de programação, você expressa explicitamente qual tipo de variável deseja usar.
- Como vimos anteriormente no curso, o Python não requer a declaração explícita de tipos.
- No entanto, é uma boa prática garantir que todas as suas variáveis sejam do tipo correto.
- `mypy` é um programa que pode ajudar a testar se todas as suas variáveis são do tipo correto.
- Você pode instalar o `mypy` executando no seu terminal: `pip install mypy`.

No editor de texto, escreva o código a seguir:

      def miar(n):
          for _ in range(n):
              print("miau")


      numero = input("Número: ")
      miar(numero)

Você provavelmente já percebeu que `numero = input("Número: ")` retorna uma `string`, não um `int`. Mas `miar` provavelmente deseja um `int`!

- Uma dica de tipo pode ser adicionada para dar ao Python uma dica de que tipo de variável `miar` deve esperar. No editor de texto, escreva o seguinte código:

      def miar(n: int):
          for _ in range(n):
              print("miau")


      numero = input("Número: ")
      miar(numero)

  Note, porém, que nosso programa ainda gera um erro.

- Após instalar o `mypy`, execute `mypy miados.py` na janela do terminal. O `mypy` fornecerá algumas orientações sobre como corrigir esse erro.
- Você pode anotar todas as suas variáveis. No editor de texto, escreva o seguinte código:

      def miar(n: int):
          for _ in range(n):
              print("miau")


      numero: int = input("Número: ")
      miar(numero)

  Observe como `numero` agora recebe uma dica de tipo.

- Novamente, executar `mypy miados.py` na janela do terminal fornece um feedback mais específico para você, o programador.
- Podemos corrigir nosso último erro escrevendo o seguinte código:

      def miar(n: int):
          for _ in range(n):
              print("miau")


      numero: int = int(input("Número: "))
      miar(numero)

  Observe como executar `mypy` agora não gera erros porque convertemos nossa entrada para um inteiro.

- Vamos introduzir um novo erro ao assumir que `miar` nos retornará uma string, ou `str`. No editor de texto, escreva o seguinte código:

      def miar(n: int):
          for _ in range(n):
              print("miau")


      numero: int = int(input("Número: "))
      miados: str = miar(numero)
      print(miados)

  Observe como a função `miar` possui apenas um efeito colateral. Como só tentamos imprimir "miau", e não retornar um valor, um erro é gerado quando tentamos armazenar o valor de retorno de `miar` em `miados`.

- Podemos usar as dicas de tipo para verificar erros, anotando também os valores de retorno das funções. No editor de texto, escreva o seguinte código:

      def miar(n: int) -> None:
          for _ in range(n):
              print("miau")


      numero: int = int(input("Número: "))
      miados: str = miar(numero)
      print(miados)

  Observe como a notação `-> None` informa ao `mypy` que não há valor de retorno.

- Podemos modificar nosso código para retornar uma string se assim desejarmos:

      def miar(n: int) -> str:
          return "miau\n" * n


      numero: int = int(input("Número: "))
      miados: str = miar(numero)
      print(miados, end="")

  Observe como armazenamos em `miados` várias `str`s. Executando `mypy` não gera erros.

- Você pode aprender mais na documentação do Python sobre [Dicas de Tipos](https://docs.python.org/3/library/typing.html).
- Você pode aprender mais sobre o [`mypy`](https://mypy.readthedocs.io/) através da própria documentação do programa.

## Docstrings <a id="docstrings"></a>

- Uma forma padrão de comentar o propósito de uma função é usar um docstring. No editor de texto, escreva o código da seguinte forma:

      def meow(n):
          """Miar n vezes."""
          return "miar\n" * n


      número = int(input("Número: "))
      miados = meow(número)
      print(miados, end="")

  Note como as três aspas duplas designam o que a função faz.

- Você pode usar docstrings para padronizar a documentação das características de uma função. No editor de texto, escreva o código da seguinte forma:

      def meow(n):
          """
          Miar n vezes.

          :param n: Número de vezes para miar
          :type n: int
          :raise TypeError: Se n não for um int
          :return: Uma string com n miados, um por linha
          :rtype: str
          """
          return "miar\n" * n


      número = int(input("Número: "))
      miados = meow(número)
      print(miados, end="")

  Note como múltiplos argumentos de docstring são incluídos. Por exemplo, ele descreve os parâmetros recebidos pela função e o que é retornado pela função.

- Ferramentas estabelecidas, como o [Sphinx](https://www.sphinx-doc.org/en/master/index.html), podem ser usadas para analisar docstrings e criar automaticamente documentação para nós na forma de páginas da web e arquivos PDF, que podem ser publicados e compartilhados com outros.
- Você pode aprender mais na documentação do [docstrings](https://peps.python.org/pep-0257/) do Python.

## `argparse` <a id="argparse"></a>

- Suponha que desejamos usar argumentos de linha de comando em nosso programa. Na janela do editor de texto, escreva o seguinte código:

      import sys

      if len(sys.argv) == 1:
          print("meow")
      elif len(sys.argv) == 3 and sys.argv[1] == "-n":
          n = int(sys.argv[2])
          for _ in range(n):
              print("meow")
      else:
          print("usage: meows.py [-n NUMBER]")

  Observe como `sys` é importado, a partir do qual obtemos acesso a `sys.argv` - uma matriz de argumentos de linha de comando fornecidos ao nosso programa quando ele é executado. Podemos usar várias declarações `if` para verificar se o usuário executou nosso programa corretamente.

- Vamos supor que este programa se torne muito mais complicado. Como poderíamos verificar todos os argumentos que podem ser inseridos pelo usuário? Poderíamos desistir se tivermos mais de alguns argumentos de linha de comando!
- Felizmente, `argparse` é uma biblioteca que lida com toda a análise de sequências complicadas de argumentos de linha de comando. Na janela do editor de texto, escreva o seguinte código:

      import argparse

      parser = argparse.ArgumentParser()
      parser.add_argument("-n")
      args = parser.parse_args()

      for _ in range(int(args.n)):
          print("meow")

  Observe como `argparse` é importado em vez de `sys`. É criado um objeto chamado `parser` a partir de uma classe `ArgumentParser`. O método `add_argument` dessa classe é usado para informar ao `argparse` quais argumentos devemos esperar do usuário quando eles executam nosso programa. Por fim, a execução do método `parse_args` do analisador garante que todos os argumentos tenham sido incluídos corretamente pelo usuário.

- Também podemos programar de maneira mais limpa, para que o usuário obtenha informações sobre o uso correto de nosso código quando não utilizá-lo corretamente. Na janela do editor de texto, escreva o seguinte código:

      import argparse

      parser = argparse.ArgumentParser(description="Meow like a cat")
      parser.add_argument("-n", help="número de vezes que será chamado o 'meow'")
      args = parser.parse_args()

      for _ in range(int(args.n)):
          print("meow")

  Observe como é fornecida alguma documentação ao usuário. Especificamente, um argumento `help` é fornecido. Agora, se o usuário executar `python meows.py --help` ou `-h`, ele será apresentado com algumas dicas sobre como usar esse programa.

- Podemos melhorar ainda mais esse programa. Na janela do editor de texto, escreva o seguinte código:

      import argparse

      parser = argparse.ArgumentParser(description="Meow like a cat")
      parser.add_argument("-n", default=1, help="número de vezes que será chamado o 'meow'", type=int)
      args = parser.parse_args()

      for _ in range(args.n):
          print("meow")

  Observe como, além de incluir documentação de ajuda, você pode fornecer um valor `default` quando nenhum argumento é fornecido pelo usuário.

- Você pode aprender mais na documentação do Python sobre [`argparse`](https://docs.python.org/3/library/argparse.html).

## Descompactando <a id="descompactando"></a>

- Não seria bom dividir uma única variável em duas variáveis? Na janela do editor de texto, escreva o código como segue:

  primeiro, \_ = input("Qual é o seu nome? ").split(" ")
  print(f"Olá, {primeiro}")

  Observe como este programa tenta obter o primeiro nome do usuário dividindo ingenuamente em um espaço único.

- Acontece que existem outras maneiras de descompactar variáveis. Você pode escrever um código mais poderoso e elegante entendendo como descompactar variáveis de maneiras aparentemente mais avançadas. Na janela do editor de texto, escreva o código como segue:

  def total(galeões, sicles, knuts):
  return (galeões _ 17 + sicles) _ 29 + knuts

  print(total(100, 50, 25), "Knuts")

  Observe como isso retorna o valor total de Knuts.

- E se quiséssemos armazenar nossas moedas em uma lista? Na janela do editor de texto, escreva o código como segue:

  def total(galeões, sicles, knuts):
  return (galeões _ 17 + sicles) _ 29 + knuts

  moedas = [100, 50, 25]

  print(total(moedas[0], moedas[1], moedas[2]), "Knuts")

  Observe como uma lista chamada `moedas` é criada. Podemos passar cada valor indexando usando `0`, `1`, e assim por diante.

- Isso está ficando bastante verboso. Não seria bom se pudéssemos simplesmente passar a lista de moedas para nossa função?
- Para habilitar a possibilidade de passar a lista inteira, podemos usar a descompactação. Na janela do editor de texto, escreva o código como segue:

  def total(galeões, sicles, knuts):
  return (galeões _ 17 + sicles) _ 29 + knuts

  moedas = [100, 50, 25]

  print(total(\*moedas), "Knuts")

  Observe como o `*` descompacta a sequência da lista de moedas e passa cada um de seus elementos individuais para `total`.

- Suponha que pudéssemos passar os nomes da moeda em qualquer ordem? Na janela do editor de texto, escreva o código como segue:

  def total(galeões, sicles, knuts):
  return (galeões _ 17 + sicles) _ 29 + knuts

  print(total(galeões=100, sicles=50, knuts=25), "Knuts")

  Observe como isso ainda calcula corretamente.

- Quando você começa a falar sobre "nomes" e "valores", pode ser que as dicionários venham à mente! Você pode implementar isso como um dicionário. Na janela do editor de texto, escreva o código como segue:

  def total(galeões, sicles, knuts):
  return (galeões _ 17 + sicles) _ 29 + knuts

  moedas = {"galeões": 100, "sicles": 50, "knuts": 25}

  print(total(moedas["galeões"], moedas["sicles"], moedas["knuts"]), "Knuts")

  Observe como é fornecido um dicionário chamado `moedas`. Podemos indexar nele usando chaves, como "galeões" ou "sicles".

- Como a função `total` espera três argumentos, não podemos passar um dicionário diretamente. Podemos usar a descompactação para ajudar com isso. Na janela do editor de texto, escreva o código como segue:

  def total(galeões, sicles, knuts):
  return (galeões _ 17 + sicles) _ 29 + knuts

  moedas = {"galeões": 100, "sicles": 50, "knuts": 25}

  print(total(\*\*moedas), "Knuts")

  Observe como `**` permite descompactar um dicionário. Ao descompactar um dicionário, ele fornece tanto as chaves quanto os valores.

## `args` e `kwargs` <a id="args-and-kwargs"></a>

- Lembre-se da documentação do `print` que vimos anteriormente neste curso:

      print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)

- `args` são argumentos posicionais, como aqueles que fornecemos para imprimir como `print("Olá", "Mundo")`.
- `kwargs` são argumentos nomeados, ou "argumentos de palavras-chave", como aqueles que fornecemos para imprimir como `print(end="")`.
- Como vemos no protótipo da função `print` acima, podemos dizer à nossa função para esperar um número atualmente desconhecido de argumentos posicionais. Também podemos dizer a ela para esperar um número atualmente desconhecido de argumentos de palavra-chave. Na janela do editor de texto, escreva o seguinte código:

      def f(*args, **kwargs):
          print("Posicionais:", args)


      f(100, 50, 25)

  Observe como a execução deste código imprimirá como argumentos posicionais.

- Até podemos passar argumentos nomeados. Na janela do editor de texto, escreva o seguinte código:

      def f(*args, **kwargs):
          print("Nomeados:", kwargs)


      f(galeões=100, sickles=50, knuts=25)

  Observe como os valores nomeados são fornecidos na forma de um dicionário.

- Pensando na função `print` acima, você pode ver como `*objetos` aceita qualquer número de argumentos posicionais.
- Você pode aprender mais na documentação do Python do [`print`](https://docs.python.org/3/library/functions.html#print).

## `map` <a id="map"></a>

- No início, começamos com a programação procedural.
- Mais tarde, descobrimos que Python é uma linguagem de programação orientada a objetos.
- Vimos indícios de programação funcional, onde as funções têm efeitos colaterais sem um valor de retorno. Podemos ilustrar isso na janela do editor de texto, digitando `code yell.py` e o código a seguir:

      def main():
          yell("This is CS50")


      def yell(word):
          print(word.upper())


      if __name__ == "__main__":
          main()

  Observe como a função `yell` é simplesmente gritada.

- Não seria bom poder gritar uma lista ilimitada de palavras? Modifique seu código da seguinte maneira:

      def main():
          yell(["This", "is", "CS50"])


      def yell(words):
          uppercased = []
          for word in words:
              uppercased.append(word.upper())
          print(*uppercased)


      if __name__ == "__main__":
          main()

  Observe que acumulamos as palavras em maiúsculas, percorrendo cada uma das palavras e colocando-as em maiúsculas. A lista em maiúsculas é impressa utilizando o `*` para desempacotá-la.

- Removendo os colchetes, podemos passar as palavras como argumentos. Na janela do editor de texto, digite o seguinte código:

      def main():
          yell("This", "is", "CS50")


      def yell(*words):
          uppercased = []
          for word in words:
              uppercased.append(word.upper())
          print(*uppercased)


      if __name__ == "__main__":
          main()

  Observe como `*words` permite que muitos argumentos sejam passados para a função.

- `map` permite mapear uma função a uma sequência de valores. Na prática, podemos codificar da seguinte forma:

      def main():
          yell("This", "is", "CS50")


      def yell(*words):
          uppercased = map(str.upper, words)
          print(*uppercased)


      if __name__ == "__main__":
          main()

  Observe como `map` recebe dois argumentos. Primeiro, ele recebe uma função que queremos aplicar a cada elemento de uma lista. Segundo, ele recebe essa lista em si, à qual aplicaremos a função mencionada anteriormente. Portanto, todas as palavras em `words` serão passadas para a função `str.upper` e retornadas para `uppercased`.

- Você pode aprender mais na documentação do Python sobre [`map`](https://docs.python.org/3/library/functions.html#map).

## Compreensões de Lista <a id="list-comprehensions"></a>

- Compreensões de lista permitem que você crie uma lista rapidamente em uma única linha elegante.
- Podemos implementar isso em nosso código da seguinte forma:

      def main():
          yell("This", "is", "CS50")


      def yell(*words):
          uppercased = [arg.upper() for arg in words]
          print(*uppercased)


      if __name__ == "__main__":
          main()

  Observe como, em vez de usar `map`, escrevemos uma expressão Python entre colchetes. Para cada argumento, `.upper` é aplicado a ele.

- Levando esse conceito adiante, vamos direcionar nossa atenção para outro programa.
- Na janela do editor de texto, digite `code gryffindors.py` e digite o seguinte código:

      students = [
          {"name": "Hermione", "house": "Gryffindor"},
          {"name": "Harry", "house": "Gryffindor"},
          {"name": "Ron", "house": "Gryffindor"},
          {"name": "Draco", "house": "Slytherin"},
      ]

      gryffindors = []
      for student in students:
          if student["house"] == "Gryffindor":
              gryffindors.append(student["name"])

      for gryffindor in sorted(gryffindors):
          print(gryffindor)

  Observe que temos uma condição enquanto estamos criando nossa lista. _Se_ a casa do estudante for Grifinória, nós adicionamos o estudante à lista de nomes. Por fim, imprimimos todos os nomes.

- De forma mais elegante, podemos simplificar esse código com uma compreensão de lista da seguinte forma:

      students = [
          {"name": "Hermione", "house": "Gryffindor"},
          {"name": "Harry", "house": "Gryffindor"},
          {"name": "Ron", "house": "Gryffindor"},
          {"name": "Draco", "house": "Slytherin"},
      ]

      gryffindors = [
          student["name"] for student in students if student["house"] == "Gryffindor"
      ]

      for gryffindor in sorted(gryffindors):
          print(gryffindor)

  Observe como a compreensão de lista está em uma única linha!

## `filter` <a id="filter"></a>

- Usar a função `filter` do Python nos permite retornar um subconjunto de uma sequência para a qual uma determinada condição é verdadeira.
- Na janela do editor de texto, escreva o seguinte código:

      students = [
          {"name": "Hermione", "house": "Gryffindor"},
          {"name": "Harry", "house": "Gryffindor"},
          {"name": "Ron", "house": "Gryffindor"},
          {"name": "Draco", "house": "Slytherin"},
      ]


      def is_gryffindor(s):
          return s["house"] == "Gryffindor"


      gryffindors = filter(is_gryffindor, students)

      for gryffindor in sorted(gryffindors, key=lambda s: s["name"]):
          print(gryffindor["name"])

  Observe como uma função chamada `is_gryffindor` é criada. Esta é nossa função de filtragem que receberá um estudante `s` e retornará `True` ou `False` dependendo se a casa do estudante é Gryffindor. Você pode ver que a nova função `filter` recebe dois argumentos. Primeiro, ela recebe a função que será aplicada a cada elemento de uma sequência - neste caso, `is_gryffindor`. Segundo, ela recebe a sequência à qual a função de filtragem será aplicada - neste caso, `students`. Em `gryffindors`, deveríamos ver apenas os estudantes que estão em Gryffindor.

- `filter` também pode usar funções lambda da seguinte forma:

      students = [
          {"name": "Hermione", "house": "Gryffindor"},
          {"name": "Harry", "house": "Gryffindor"},
          {"name": "Ron", "house": "Gryffindor"},
          {"name": "Draco", "house": "Slytherin"},
      ]


      gryffindors = filter(lambda s: s["house"] == "Gryffindor", students)

      for gryffindor in sorted(gryffindors, key=lambda s: s["name"]):
          print(gryffindor["name"])

  Observe como a mesma lista de estudantes é fornecida.

- Você pode aprender mais na documentação do Python sobre [`filter`](https://docs.python.org/pt-br/3/library/functions.html#filter).

## Compreensões de dicionário <a id="dictionary-comprehensions"></a>

- Podemos aplicar a mesma ideia por trás das compreensões de lista aos dicionários. Na janela do editor de texto, escreva o seguinte código:

      students = ["Hermione", "Harry", "Ron"]

      gryffindors = []

      for student in students:
          gryffindors.append({"name": student, "house": "Gryffindor"})

      print(gryffindors)

  Observe como este código ainda não utiliza compreensões. Em vez disso, segue os mesmos paradigmas que já vimos antes.

- Agora podemos aplicar compreensões de dicionário modificando nosso código da seguinte maneira:

      students = ["Hermione", "Harry", "Ron"]

      gryffindors = [{"name": student, "house": "Gryffindor"} for student in students]

      print(gryffindors)

  Observe como todo o código anterior é simplificado em uma única linha, onde a estrutura do dicionário é fornecida para cada `student` em `students`.

- Podemos até simplificar ainda mais da seguinte forma:

      students = ["Hermione", "Harry", "Ron"]

      gryffindors = {student: "Gryffindor" for student in students}

      print(gryffindors)

  Observe como o dicionário será construído com pares de chave-valor.

## `enumerate` <a id="enumerate"></a>

- Podemos desejar fornecer uma classificação para cada estudante. Na janela do editor de texto, escreva o seguinte código:

      students = ["Hermione", "Harry", "Ron"]

      for i in range(len(students)):
          print(i + 1, students[i])

  Observe como cada estudante é enumerado ao executar esse código.

- Utilizando a enumeração, podemos fazer o mesmo:

      students = ["Hermione", "Harry", "Ron"]

      for i, student in enumerate(students):
          print(i + 1, student)

  Observe como a enumeração apresenta o índice e o valor de cada `estudante`.

- Você pode aprender mais na documentação do Python sobre [`enumerate`](https://docs.python.org/3/library/functions.html#enumerate).

## Geradores e Iteradores <a id="generators-and-iterators"></a>

- Em Python, existe uma maneira de proteger contra o esgotamento de recursos do sistema quando os problemas que estão sendo abordados se tornam muito grandes.
- Nos Estados Unidos, é costumeiro "contar ovelhas" mentalmente quando se tem dificuldade para adormecer.
- Na janela do editor de texto, digite `code sleep.py` e o código a seguir:

      n = int(input("Qual é o valor de n? "))
      for i in range(n):
          print("🐑" * i)

  Observe como este programa conta a quantidade de ovelhas desejada.

- Podemos tornar nosso programa mais sofisticado adicionando uma função `main` com o seguinte código:

      def main():
          n = int(input("Qual é o valor de n? "))
          for i in range(n):
              print("🐑" * i)


      if __name__ == "__main__":
          main()

  Observe como uma função `main` é fornecida.

- Temos adquirido o hábito de abstrair partes do nosso código.
- Podemos chamar uma função de ovelha modificando nosso código da seguinte forma:

      def main():
          n = int(input("Qual é o valor de n? "))
          for i in range(n):
              print(sheep(i))


      def sheep(n):
          return "🐑" * n


      if __name__ == "__main__":
          main()

  Observe como a função `main` faz a iteração.

- Podemos fornecer mais habilidades à função `sheep`. Na janela do editor de texto, digite o seguinte código:

      def main():
          n = int(input("Qual é o valor de n? "))
          for s in sheep(n):
              print(s)


      def sheep(n):
          flock = []
          for i in range(n):
              flock.append("🐑" * i)
          return flock


      if __name__ == "__main__":
          main()

  Observe como criamos um rebanho de ovelhas e retornamos o `flock`.

- Ao executar nosso código, você pode experimentar com diferentes números de ovelhas, como `10`, `1000` e `10000`. E se você pedisse `1000000` ovelhas, seu programa poderia travar ou falhar completamente. Porque você tentou gerar uma lista enormemente grande de ovelhas, seu computador pode ter dificuldades para concluir o cálculo.
- O gerador `yield` pode resolver esse problema, retornando um pequeno pedaço dos resultados de cada vez. Na janela do editor de texto, digite o seguinte código:

      def main():
          n = int(input("Qual é o valor de n? "))
          for s in sheep(n):
              print(s)


      def sheep(n):
          for i in range(n):
              yield "🐑" * i


      if __name__ == "__main__":
          main()

  Observe como o `yield` fornece apenas um valor de cada vez, enquanto o `for` continua funcionando.

- Você pode aprender mais na documentação do Python sobre [geradores](https://docs.python.org/3/howto/functional.html#generators).
- Você pode aprender mais na documentação do Python sobre [iteradores](https://docs.python.org/3/howto/functional.html#iterators).
