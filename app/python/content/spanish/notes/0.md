## Creando Código con Python <a id="creating-code-with-python"></a>

- VS Code es un tipo especial de editor de texto que se llama compilador. En la parte superior, notarás un editor de texto. En la parte inferior, verás una terminal donde puedes ejecutar comandos.
- En la terminal, puedes ejecutar `code hello.py` para comenzar a codificar.
- En el editor de texto de arriba, puedes escribir `print("hello, world")`. Este es un programa canónico famoso que casi todos los programadores escriben durante su proceso de aprendizaje.
- En la ventana de la terminal, puedes ejecutar comandos. Para ejecutar este programa, necesitarás mover tu cursor hacia la parte inferior de la pantalla, haciendo clic en la ventana de la terminal. Ahora puedes escribir un segundo comando en la ventana de la terminal. Junto al signo de dólar, escribe `python hello.py` y presiona la tecla Enter en tu teclado.
- Recuerda que las computadoras realmente solo entienden ceros y unos. Por lo tanto, cuando ejecutas `python hello.py`, python interpretará el texto que creaste en `hello.py` y lo traducirá a los ceros y unos que la computadora puede entender.
- El resultado de ejecutar el programa `python hello.py` es `hello, world`.
- ¡Felicidades! Acabas de crear tu primer programa.

## Funciones <a id="funciones"></a>

- Las funciones son verbos o acciones que la computadora o lenguaje informático ya sabrá cómo realizar.
- En tu programa `hello.py`, la función `print` sabe cómo imprimir en la ventana del terminal.
- La función `print` toma argumentos. En este caso, `"hello, world"` son los argumentos que toma la función `print`.

## Errores <a id="bugs"></a>

- Los errores son una parte natural de la programación. ¡Estos son fallos, problemas para que resuelvas! ¡No te desanimes! Esto es parte del proceso para convertirte en un gran programador.
- Imagina en nuestro programa `hello.py` que accidentalmente escribimos `print("hello, world"` notarás que nos falta el `)` final requerido por el compilador. ¡Si hago este error a propósito, el compilador mostrará un error en la ventana del terminal!
- A menudo, los mensajes de error te informarán de tus errores y te darán pistas sobre cómo corregirlos. Sin embargo, habrá muchas veces en las que el compilador no será tan amigable.

## Mejorando tu Primer Programa en Python <a id="mejorando-tu-primer-programa-en-python"></a>

- Podemos personalizar tu primer programa en Python.
- En nuestro editor de texto en `hello.py` podemos agregar otra función. `input` es una función que toma un argumento como indicación. Podemos editar nuestro código de la siguiente manera

      input("¿Cuál es tu nombre? ")
      print("¡Hola, mundo!")

- Sin embargo, solo esta edición no permitirá que tu programa muestre lo que ingresa el usuario. Para eso, necesitaremos presentarte a las variables.

### Variables <a id="variables"></a>

- Una variable es simplemente un contenedor para un valor dentro de tu propio programa.
- En tu programa, puedes introducir tu propia variable editándolo de la siguiente manera

      nombre = input("¿Cuál es tu nombre? ")
      print("¡Hola, mundo!")

  Observa que este signo igual `=` en el medio de `nombre = input("¿Cuál es tu nombre? ")` tiene un papel especial en la programación. Este signo igual asigna literalmente lo que está a la derecha a lo que está a la izquierda. Por lo tanto, el valor devuelto por `input("¿Cuál es tu nombre? ")` se asigna a `nombre`.

- Si editas tu código de la siguiente manera, notarás un error

      nombre = input("¿Cuál es tu nombre? ")
      print("¡Hola, nombre")

- El programa devolverá `¡Hola, nombre` en la ventana del terminal, independientemente de lo que el usuario escriba.
- Al continuar editando nuestro código, podrías escribir

      nombre = input("¿Cuál es tu nombre? ")
      print("¡Hola,")
      print(nombre)

- El resultado en la ventana del terminal sería

      ¿Cuál es tu nombre? David
      ¡Hola
      David

- ¡Estamos más cerca del resultado que podríamos intentar!
- Puedes aprender más en la documentación de Python sobre [tipos de datos](https://docs.python.org/3/library/datatypes.html).

### Comentarios <a id="comentarios"></a>

- Los comentarios son una forma para que los programadores rastreen lo que están haciendo en sus programas e incluso informen a otros sobre sus intenciones para un bloque de código. En resumen, ¡son notas para ti y para otros que verán tu código!
- Puedes agregar comentarios a tu programa para poder ver qué es lo que está haciendo. Podrías editar tu código de la siguiente manera:

      # Pedir al usuario su nombre
      nombre = input("¿Cuál es tu nombre? ")
      print("¡Hola,")
      print(nombre)

- Los comentarios también pueden servir como una lista de tareas pendientes para ti.

### Pseudocódigo <a id="pseudocodigo"></a>

- El pseudocódigo es un tipo importante de comentario que se convierte en un tipo especial de lista de tareas pendientes, especialmente cuando no entiendes cómo completar una tarea de codificación. Por ejemplo, en tu código, podrías editarlo para que diga:

      # Pedir al usuario su nombre
      nombre = input("¿Cuál es tu nombre? ")

      # Imprimir hola
      print("¡Hola, ")

      # Imprimir el nombre ingresado
      print(nombre)

## Mejorando Aún Más Tu Primer Programa en Python <a id="mejorando-aun-mas-tu-primer-programa-en-python"></a>

- Podemos editar nuestro código de la siguiente manera:

      # Pedir al usuario su nombre
      name = input("¿Cuál es tu nombre? ")

      # Imprimir hola y el nombre ingresado
      print("¡Hola, " + name)

- Resulta que algunas funciones aceptan muchos argumentos.
- Podemos usar una coma `,` para pasar múltiples argumentos al editar nuestro código de la siguiente manera:

      # Pedir al usuario su nombre
      name = input("¿Cuál es tu nombre? ")

      # Imprimir hola y el nombre ingresado
      print("¡Hola,", name)

  El resultado en la terminal, si escribiéramos “David” sería `¡Hola, David`. Éxito.

## Cadenas y Parámetros <a id="cadenas-y-parametros"></a>

- Una cadena, conocida como `str` en Python, es una secuencia de texto.
- Retrocediendo un poco en nuestro código a lo siguiente, hubo un efecto visual de que el resultado apareciera en varias líneas:

      # Preguntar al usuario su nombre
      name = input("¿Cuál es tu nombre? ")
      print("¡Hola,")
      print(name)

- Las funciones toman argumentos que influyen en su comportamiento. Si observamos la documentación de [`print`](https://docs.python.org/3/library/functions.html#print) notarás que podemos aprender mucho sobre los argumentos que toma la función print.
- Al observar esta documentación, aprenderás que la función print incluye automáticamente un fragmento de código `end='\n'`. Esto `\\n` indica que la función print creará automáticamente un salto de línea al ejecutarse. La función print toma un argumento llamado `end` y el valor por defecto es crear una nueva línea.
- Sin embargo, técnicamente podemos proporcionar un argumento para `end` de manera que una nueva línea no se cree.
- Podemos modificar nuestro código de la siguiente manera:

      # Preguntar al usuario su nombre
      name = input("¿Cuál es tu nombre? ")
      print("¡Hola,", end="")
      print(name)

  Al proporcionar `end=""` estamos sobrescribiendo el valor por defecto de `end` de manera que nunca cree una nueva línea después de esta primera instrucción de impresión. Al proporcionar el nombre como "David", la salida en la ventana del terminal será `¡Hola, David`.

- Los parámetros, por lo tanto, son argumentos que puede tomar una función.
- Puedes aprender más en la documentación de Python sobre [`print`](https://docs.python.org/3/library/functions.html#print).

### Un pequeño problema con las comillas <a id="un-pequeno-problema-con-las-comillas"></a>

- Observa cómo agregar comillas como parte de tu cadena puede ser un desafío.
- `print("¡Hola,"amigo"")` no funcionará y el compilador mostrará un error.  
- Generalmente, hay dos enfoques para solucionar esto. Primero, podrías simplemente cambiar las comillas por comillas simples.
- Otro enfoque, más comúnmente usado, sería codificar como `print("¡Hola, \"amigo\"")`. Las barras invertidas le indican al compilador que el carácter siguiente debe ser considerado como una comilla en la cadena y evitar un error del compilador.

## Formateo de Cadenas <a id="formatting-strings"></a>

- Probablemente la forma más elegante de usar cadenas sería de la siguiente manera:
      
      # Preguntar al usuario por su nombre
      name = input("¿Cuál es tu nombre? ")
      print(f"hola, {name}")

  Observa la `f` en `print(f"hola, {name}")`. Esta `f` es un indicador especial para que Python trate a esta cadena de una manera especial, diferente a los enfoques anteriores que hemos ilustrado en esta clase. Espera usar este estilo de cadenas con bastante frecuencia en este curso.

## Más sobre Cadenas <a id="más-sobre-cadenas"></a>

- Nunca debes esperar que tu usuario coopere según lo esperado. Por lo tanto, deberás asegurarte de corregir o verificar la entrada de tu usuario.
- Resulta que en las cadenas está integrada la capacidad de eliminar espacios en blanco de una cadena.
- Utilizando el método `strip` en `name` como `name = name.strip()`, se eliminarán todos los espacios en blanco a la izquierda y derecha de la entrada del usuario. Puedes modificar tu código de la siguiente manera:

      # Preguntar al usuario por su nombre
      name = input("¿Cuál es tu nombre? ")

      # Eliminar espacios en blanco de la cadena
      name = name.strip()

      # Imprimir el resultado
      print(f"hola, {name}")

  Al ejecutar nuevamente este programa, sin importar cuántos espacios escribas antes o después del nombre, se eliminarán todos los espacios en blanco.

- Utilizando el método `title`, se escribirá en mayúscula la primera letra del nombre del usuario:

      # Preguntar al usuario por su nombre
      name = input("¿Cuál es tu nombre? ")

      # Eliminar espacios en blanco de la cadena
      name = name.strip()

      # Capitalizar la primera letra de cada palabra
      name = name.title()

      # Imprimir el resultado
      print(f"hola, {name}")

- En este punto, es posible que estés muy cansado de escribir `python` repetidamente en la ventana del terminal. Puedes usar la flecha hacia arriba de tu teclado para recordar los comandos de terminal más recientes que hayas realizado.
- Observa que puedes modificar tu código para ser más eficiente:

      # Preguntar al usuario por su nombre
      name = input("¿Cuál es tu nombre? ")

      # Eliminar espacios en blanco de la cadena y capitalizar la primera letra de cada palabra
      name = name.strip().title()

      # Imprimir el resultado
      print(f"hola, {name}")

  Esto crea el mismo resultado que tu código anterior.

- ¡Incluso podríamos ir más lejos!

      # Preguntar al usuario por su nombre, eliminar espacios en blanco de la cadena y capitalizar la primera letra de cada palabra
      name = input("¿Cuál es tu nombre? ").strip().title()

      # Imprimir el resultado
      print(f"hola, {name}")

- Puedes aprender más sobre cadenas en la documentación de Python sobre [`str`](https://docs.python.org/3/library/stdtypes.html#str)

## Enteros o int <a id="integers-or-int"></a>

- En Python, un entero se denomina como `int`.
- En el mundo de las matemáticas, estamos familiarizados con los operadores +, -, \*, / y %. Ese último operador `%` o operador módulo puede que no te resulte muy familiar.
- No es necesario utilizar la ventana del editor de texto en tu compilador para ejecutar código de Python. Abajo en tu terminal, puedes ejecutar `python` solo. Se presentará `>>>` en la ventana de la terminal. Entonces puedes ejecutar código en vivo e interactivo. Podrías escribir `1+1`, y se calculará esa operación. Este modo no se usará comúnmente durante este curso.
- Volviendo a abrir VS Code, podemos escribir `code calculate.py` en la terminal. Esto creará un nuevo archivo en el cual crearemos nuestra propia calculadora.
- Primero, podemos declarar algunas variables.

      x = 1
      y = 2

      z = x + y

      print(z)

  Naturalmente, al ejecutar `python calculate.py`, obtenemos el resultado en la ventana de la terminal de `3`. Podemos hacer esto más interactivo usando la función `input`.

      x = input("¿Cuánto es x? ")
      y = input("¿Cuánto es y? ")

      z = x + y

      print(z)

- Al ejecutar este programa, descubrimos que la salida es incorrecta como `12`. ¿Por qué podría ser esto?
- Anteriormente, hemos visto cómo el signo `+` concatena dos cadenas. Como la entrada desde tu teclado en tu computadora llega al compilador como texto, se trata como una cadena. Por lo tanto, debemos convertir esta entrada de una cadena a un entero. Podemos hacerlo de la siguiente manera:

      x = input("¿Cuánto es x? ")
      y = input("¿Cuánto es y? ")

      z = int(x) + int(y)

      print(z)

  El resultado ahora es correcto. El uso de `int(x)` se llama "casting", donde un valor se cambia temporalmente de un tipo de variable (en este caso, una cadena) a otro (aquí, un entero).

- Podemos mejorar nuestro programa aún más de la siguiente manera:

      x = int(input("¿Cuánto es x? "))
      y = int(input("¿Cuánto es y? "))

      print(x + y)

  Esto ilustra que puedes ejecutar funciones sobre funciones. La función interna se ejecuta primero, y luego la externa. Primero se ejecuta la función `input`. Luego, la función `int`.

- Puedes aprender más en la documentación de Python sobre [`int`](https://docs.python.org/3/library/functions.html?highlight=float#int).

## Conceptos básicos de los números de punto flotante <a id="float-basics"></a>

- Un valor de punto flotante es un número real que tiene un punto decimal, como `0.52`.
- Puedes modificar tu código para admitir números de punto flotante de la siguiente manera:

      x = float(input("¿Cuál es x? "))
      y = float(input("¿Cuál es y? "))

      print(x + y)

  Este cambio permite al usuario ingresar `1.2` y `3.4` para obtener un total de `4.6`.

- Sin embargo, imaginemos que deseas redondear el total al entero más cercano. Al consultar la documentación de Python para la función `round`, verás que los argumentos disponibles son `round(number[n, ndigits])`. Los corchetes cuadrados indican que algo opcional puede ser especificado por el programador. Por lo tanto, podrías usar `round(n)` para redondear un dígito a su entero más cercano. Alternativamente, podrías codificar de la siguiente manera:

      # Obtener la entrada del usuario
      x = float(input("¿Cuál es x? "))
      y = float(input("¿Cuál es y? "))

      # Crear un resultado redondeado
      z = round(x + y)

      # Imprimir el resultado
      print(z)

  La salida se redondeará al entero más cercano.

- ¿Qué sucede si queremos formatear la salida de números largos? Por ejemplo, en lugar de ver `1000`, quizás desees ver `1,000`. Podrías modificar tu código de la siguiente manera:

      # Obtener la entrada del usuario
      x = float(input("¿Cuál es x? "))
      y = float(input("¿Cuál es y? "))

      # Crear un resultado redondeado
      z = round(x + y)

      # Imprimir el resultado formateado
      print(f"{z:,}")

  Aunque algo críptico, esa línea `print(f"{z:,}")` crea un escenario donde el `z` impreso incluirá comas, por lo que el resultado podría parecerse a `1,000` o `2,500`.

## Más sobre los Floats <a id="más-sobre-los-floats"></a>

- ¿Cómo podemos redondear los valores de punto flotante? Primero, modifica tu código de la siguiente manera:

      # Obtener la entrada del usuario
      x = float(input("¿Cuál es x? "))
      y = float(input("¿Cuál es y? "))

      # Calcular el resultado
      z = x / y

      # Imprimir el resultado
      print(z)

  Al ingresar `2` como x y `3` como y, el resultado z es `0.6666666666`, aparentemente continuando hacia el infinito como podríamos esperar.

- Imaginemos que queremos redondear esto hacia abajo. Podríamos modificar nuestro código de la siguiente manera:

      # Obtener la entrada del usuario
      x = float(input("¿Cuál es x? "))
      y = float(input("¿Cuál es y? "))

      # Calcular el resultado y redondear
      z = round(x / y, 2)

      # Imprimir el resultado
      print(z)

  Como podríamos esperar, esto redondeará el resultado a los dos decimales más cercanos.

- También podríamos usar `fstring` para formatear la salida de la siguiente manera:

      # Obtener la entrada del usuario
      x = float(input("¿Cuál es x? "))
      y = float(input("¿Cuál es y? "))

      # Calcular el resultado
      z = x / y

      # Imprimir el resultado
      print(f"{z:.2f}")

  Este código críptico de `fstring` muestra lo mismo que nuestra estrategia de redondeo anterior.

- Puedes aprender más en la documentación de Python sobre [`float`](https://docs.python.org/3/library/functions.html?highlight=float#float).

## Definición <a id="def"></a>

- ¿No sería genial crear nuestras propias funciones?
- Traigamos de vuelta nuestro código final de `hello.py` escribiendo `code hello.py` en la ventana de la terminal. Tu código inicial debería lucir de la siguiente manera:

      # Pedir al usuario su nombre, eliminar los espacios en blanco de la cadena y capitalizar la primera letra de cada palabra
      nombre = input("¿Cuál es tu nombre? ").strip().title()

      # Imprimir la salida
      print(f"¡Hola, {nombre}!")

  ¡Podemos mejorar nuestro código para crear nuestra propia función especial que nos diga "hola"!

- Al borrar todo nuestro código en el editor de texto, comencemos desde cero:

      nombre = input("¿Cuál es tu nombre? ")
      hello()
      print(nombre)

  Al intentar ejecutar este código, tu compilador lanzará un error. Después de todo, no hay una función definida para `hello`.

- Podemos crear nuestra propia función llamada `hello` de la siguiente manera:

      def hello():
          print("¡Hola!")

      nombre = input("¿Cuál es tu nombre? ")
      hello()
      print(nombre)

  Observa que todo lo que está bajo `def hello()` está sangrado. Python es un lenguaje sangrado. Utiliza la sangría para entender qué forma parte de la función anterior. Por lo tanto, todo en la función `hello` debe estar sangrado. Cuando algo no está sangrado, lo trata como si no estuviera dentro de la función `hello`. Al ejecutar `python hello.py` en la ventana de la terminal, verás que la salida no es exactamente como quisieras.

- Podemos mejorar aún más nuestro código:

      # Crear nuestra propia función
      def hello(para):
          print("¡Hola,", para,"!")

      # Salida usando nuestra propia función
      nombre = input("¿Cuál es tu nombre? ")
      hello(nombre)

  Aquí, en las primeras líneas, estás creando tu función `hello`. Sin embargo, esta vez le estás diciendo al compilador que esta función toma un solo parámetro: una variable llamada `para`. Por lo tanto, cuando llamas a `hello(nombre)`, la computadora pasa `nombre` a la función `hello` como `para`. Así es como pasamos valores a las funciones. Muy útil. Al ejecutar `python hello.py` en la ventana de la terminal, verás que la salida se acerca mucho a la ideal presentada anteriormente en esta lección.

- Podemos cambiar nuestro código para agregar un valor predeterminado a `hello`:

      # Crear nuestra propia función
      def hello(para="mundo"):
          print("¡Hola,", para, "!")

      # Salida usando nuestra propia función
      nombre = input("¿Cuál es tu nombre? ")
      hello(nombre)

      # Salida sin pasar los argumentos esperados
      hello()

  Prueba tu código por ti mismo. Observa cómo el primer `hello` se comportará como esperas, y el segundo `hello`, al que no se le pasa un valor, por defecto imprimirá `¡Hola, mundo!`.

- No es necesario tener nuestra función al principio de nuestro programa. Podemos moverla a un lugar inferior, pero debemos indicarle al compilador que tenemos una función `main` y una función `hello` separada.

      def main():

          # Salida usando nuestra propia función
          nombre = input("¿Cuál es tu nombre? ")
          hello(nombre)

          # Salida sin pasar los argumentos esperados
          hello()


      # Crear nuestra propia función
      def hello(para="mundo"):
          print("¡Hola,", para, "!")

  Sin embargo, esto creará un error de algún tipo. ¡Si ejecutas `python hello.py`, no sucede nada! La razón de esto es que nada en este código realmente llama a la función `main` y da vida a nuestro programa.

- La siguiente modificación muy pequeña llamará a la función `main` y restaurará nuestro programa a su funcionamiento normal:

      def main():

          # Salida usando nuestra propia función
          nombre = input("¿Cuál es tu nombre? ")
          hello(nombre)

          # Salida sin pasar los argumentos esperados
          hello()


      # Crear nuestra propia función
      def hello(para="mundo"):
          print("¡Hola,", para, "!")

      main()

## Devolución de valores <a id="returning-values"></a>

- Puedes imaginar muchos escenarios en los que no solo deseas que una función realice una acción, sino también que devuelva un valor a la función principal. Por ejemplo, en lugar de simplemente imprimir el cálculo de `x + y`, es posible que desees que una función devuelva el valor de este cálculo a otra parte de tu programa. A este proceso de "devolver" un valor lo llamamos un valor de `retorno`.
- Volviendo a nuestro código `calculator.py` escribiendo `code calculator.py`. Borra todo el código allí. Reorganiza el código de la siguiente manera:

      def main():
          x = int(input("¿Cuál es el valor de x? "))
          print("El cuadrado de x es", square(x))


      def square(n):
          return n * n


      main()

  Efectivamente, `x` se pasa a `square`. Luego, el cálculo de `x * x` se devuelve a la función principal.
