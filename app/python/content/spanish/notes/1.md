## Condiciones <a id="condicionales"></a>

-   Las condiciones te permiten, como programador, hacer que tu programa tome decisiones: como si tu programa tuviera la opción de tomar el camino de la mano izquierda o de la mano derecha basándose en ciertas condiciones.
-   Dentro de Python se encuentran incorporados un conjunto de "operadores" que se utilizan para formular preguntas matemáticas.
-   Los símbolos `>` y `<` probablemente te resulten bastante familiares.
-   `>=` denota "mayor que o igual a".
-   `<=` denota "menor que o igual a".
-   `==` denota "igual, ¡aunque debes notar el doble signo igual! Un solo signo igual asignaría un valor. Los signos iguales dobles se utilizan para comparar variables.
-   `!=` denota "no igual a".
-   Las sentencias condicionales comparan un término de la mano izquierda con un término de la mano derecha.

## Declaraciones if <a id="if-statements"></a>

-   En la ventana de tu terminal, escribe `code compare.py`. Esto creará un nuevo archivo llamado "compare."
-   En la ventana del editor de texto, comienza con lo siguiente:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        if x < y:
            print("x es menor que y")

    Observa cómo tu programa toma la entrada del usuario para x e y, convirtiéndolos en enteros y guardándolos en sus respectivas variables x e y. Luego, la declaración `if` compara x e y. Si se cumple la condición de `x < y`, se ejecuta la declaración `print`.

-   Las declaraciones `if` utilizan valores `bool` o booleanos (verdadero o falso) para decidir si deben ejecutarse. Si la declaración `x > y` es verdadera, el compilador la registrará como `verdadero` y ejecutará el código.

## Flujo de control, elif y else <a id="control-flow-elif-and-else"></a>

-   Revise aún más su código de la siguiente manera:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        Si x < y:
            imprimir("x es menor que y")
        Si x > y:
            imprimir("x es mayor que y")
        Si x == y:
            imprimir("x es igual a y")

    Note cómo se están proporcionando una serie de declaraciones `if`. Primero, se evalúa la primera afirmación `if`. Luego, se ejecuta la evaluación de la segunda afirmación `if`. Por último, se ejecuta la evaluación de la última afirmación `if`. Este flujo de decisiones se llama "flujo de control".

-   Nuestro código puede representarse de la siguiente manera: <pre class="mermaid">
diagrama de flujo TD
A([inicio]) --> B{x < y}
B -- Verdadero --> C["#quot;x es menor que y#quot;"]
C --> D{x > y}
D -- Verdadero --> E["#quot;x es mayor que y#quot;"]
E --> F{x == y}
F -- Verdadero --> G["#quot;x es igual a y#quot;"]
G --> H([fin])
B -- Falso --> D
D -- Falso --> F
F -- Falso --> H
</pre>

-   Este programa puede mejorar al no hacer tres preguntas consecutivas. ¡Después de todo, no todas las tres preguntas pueden tener un resultado de `verdadero`! Revise su programa de la siguiente manera:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        Si x < y:
            imprimir("x es menor que y")
        elif x > y:
            imprimir("x es mayor que y")
        elif x == y:
            imprimir("x es igual a y")

    Note cómo el uso de `elif` permite al programa tomar menos decisiones. Primero, se evalúa la afirmación `if`. Si se encuentra que esta afirmación es verdadera, no se ejecutarán las declaraciones `elif` en absoluto. Sin embargo, si se evalúa la afirmación `if` y se encuentra que es falsa, se evaluará la primera afirmación `elif`. Si esto es verdadero, no se ejecutará la evaluación final.

-   Nuestro código puede representarse de la siguiente manera: <pre class="mermaid">
diagrama de flujo TD
A([inicio]) --> B{x < y}
B -- Verdadero --> C["#quot;x es menor que y#quot;"]
B -- Falso --> D{x > y}
D -- Verdadero --> F["#quot;x es mayor que y#quot;"]
D -- Falso --> F{x == y}
F -- Verdadero --> G["#quot;x es igual a y#quot;"]
G --> H([fin])
F -- Falso --> H
C --> H
E --> H
</pre>

-   Aunque su computadora puede no notar una diferencia en velocidad entre nuestro primer programa y este programa revisado, considere cómo un servidor en línea que ejecute miles de millones o billones de estos cálculos diariamente podría verse afectado definitivamente por una decisión de codificación tan pequeña.

-   Hay una mejora final que podemos hacer a nuestro programa. Note cómo lógicamente `elif x == y` no es una evaluación necesaria para ejecutar. Después de todo, si lógicamente x no es menor que y Y x no es mayor que y, x DEBE ser igual a y. Por lo tanto, no tenemos que ejecutar `elif x == y`. Podemos crear una salida predeterminada, "catch-all," usando una afirmación `else`. Podemos revisar de la siguiente manera:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        Si x < y:
            imprimir("x es menor que y")
        elif x > y:
            imprimir("x es mayor que y")
        más:
            imprimir("x es igual a y")

    Note cómo la complejidad relativa de este programa ha disminuido a través de nuestra revisión.

-   Nuestro código puede representarse de la siguiente manera: <pre class="mermaid">
diagrama de flujo TD
A([inicio]) --> B{x < y}
B -- Verdadero --> C["#quot;x es menor que y#quot;"]
B -- Falso --> D{x > y}
D -- Verdadero --> E["#quot;x es mayor que y#quot;"]
D -- Falso --> F["#quot;x es igual a y#quot;"]
F --> G([fin])
C --> G
E --> G
</pre>

## o <a id="or"></a>

-   `or` permite que tu programa decida entre una o más alternativas. Por ejemplo, podríamos editar nuestro programa de la siguiente manera:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        if x < y or x > y:
            print("x no es igual a y")
        else:
            print("x es igual a y")

    Observa que el resultado de nuestro programa es el mismo, pero la complejidad se reduce. La eficiencia de nuestro código aumenta.

-   En este punto, nuestro código es bastante bueno. Sin embargo, ¿podría mejorarse aún más el diseño? Podríamos editar nuestro código de la siguiente manera:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        if x != y:
            print("x no es igual a y")
        else:
            print("x es igual a y")

    Observa cómo eliminamos completamente el `or` y simplemente preguntamos, "¿x no es igual a y?" Hacemos una y solo una pregunta. ¡Muy eficiente!

-   Con fines ilustrativos, también podríamos cambiar nuestro código de la siguiente manera:

        x = int(input("¿Cuál es el valor de x? "))
        y = int(input("¿Cuál es el valor de y? "))

        if x == y:
            print("x es igual a y")
        else:
            print("x no es igual a y")

    Observa que el operador `==` evalúa si lo que está a la izquierda y a la derecha son iguales entre sí. El uso de dos signos iguales es muy importante. Si usas solo un signo igual, es probable que el compilador lance un error.

-   Nuestro código puede ser ilustrado de la siguiente manera: <pre class="mermaid">
flowchart TD
A([inicio]) --> B{x == y}
B -- Verdadero --> C["#quot;x es igual a y#quot;"]
B -- Falso --> E["#quot;x no es igual a y#quot;"]
C --> F([fin])
E --> F
</pre>

          score = int(input("Puntaje: "))

    Observa cómo Python te permite encadenar los operadores y condiciones de una manera bastante inusual en otros lenguajes de programación.

-   Aun así, podemos mejorar aún más nuestro programa:

        score = int(input("Puntaje: "))

## Módulo <a id="modulo"></a>

-   En matemáticas, la paridad se refiere a si un número es par o impar.
-   El operador de módulo `%` en programación permite ver si dos números se dividen de forma exacta o si dividen y tienen un residuo.
-   Por ejemplo, 4 % 2 daría como resultado cero, porque se divide de forma exacta. Sin embargo, 3 % 2 no se divide de forma exacta y daría como resultado un número distinto de cero.
-   En la ventana de terminal, crea un nuevo programa escribiendo `code parity.py`. En la ventana del editor de texto, escribe tu código de la siguiente manera:

        x = int(input("¿Cuál es x? "))

        if x % 2 == 0:
            print("Par")
        else:
            print("Impar")

    Observa cómo nuestros usuarios pueden ingresar cualquier número mayor o igual a 1 para ver si es par o impar.

## Creando nuestra propia función de paridad <a id="creando-nuestra-propia-funcion-de-paridad"></a>

-   Como se discutió en la Lección 0, ¡encontrarás útil crear tu propia función!
-   Podemos crear nuestra propia función para verificar si un número es par o impar. Ajusta tu código de la siguiente manera:

        def main():
            x = int(input("¿Cuál es x? "))
            if es_par(x):
                print("Par")
            else:
                print("Impar")


        def es_par(n):
            if n % 2 == 0:
                return True
            else:
                return False


        main()

    Nota que nuestra instrucción `if` `es_par(x)` funciona aunque no haya operador allí. Esto se debe a que nuestra función devuelve un `bool` (o booleano), verdadero o falso, de regreso a la función principal. La instrucción `if` simplemente evalúa si `es_par` de `x` es verdadero o falso.

## Pythonico <a id="pythonic"></a>

-   En el mundo de la programación, existen tipos de programación que son llamados "Pythonicos" por naturaleza. Es decir, hay formas de programar que a veces solo se ven en la programación en Python. Considera la siguiente revisión a nuestro programa:

        def main():
            x = int(input("¿Cuál es el valor de x? "))
            if is_even(x):
                print("Par")
            else:
                print("Impar")


        def is_even(n):
            return True if n % 2 == 0 else False


        main()

    Observa que esta instrucción de retorno en nuestro código es casi como una oración en inglés. Esta es una forma única de codificación solo vista en Python.

-   Podemos revisar aún más nuestro código y hacerlo cada vez más legible:

        def main():
            x = int(input("¿Cuál es el valor de x? "))
            if is_even(x):
                print("Par")
            else:
                print("Impar")


        def is_even(n):
            return n % 2 == 0


        main()

    Observa que el programa evaluará lo que está sucediendo dentro de `n % 2 == 0` como verdadero o falso y simplemente devolverá eso a la función principal.

## `match` <a id="match"></a>

-   Similar al uso de declaraciones `if`, `elif`, y `else`, las declaraciones `match` pueden ser utilizadas para ejecutar condicionalmente código que coincida con ciertos valores.
-   Considera el siguiente programa:

          name = input("¿Cuál es tu nombre? ")

          if name == "Harry":
              print("Gryffindor")
          elif name == "Hermione":
              print("Gryffindor")
          elif name == "Ron":
              print("Gryffindor")
          elif name == "Draco":
              print("Slytherin")
          else:
              print("¿Quién?")

    Observa que las primeras tres declaraciones condicionales imprimen la misma respuesta.

-   Podemos mejorar este código ligeramente utilizando la palabra clave `or`:

          name = input("¿Cuál es tu nombre? ")

          if name == "Harry" or name == "Hermione" or name == "Ron":
              print("Gryffindor")
          elif name == "Draco":
              print("Slytherin")
          else:
              print("¿Quién?")

    Nota que el número de declaraciones `elif` ha disminuido, mejorando la legibilidad de nuestro código.

-   Alternativamente, podemos utilizar declaraciones `match` para asignar nombres a casas. Considera el siguiente código:

          name = input("¿Cuál es tu nombre? ")

          match name:
              case "Harry":
                  print("Gryffindor")
              case "Hermione":
                  print("Gryffindor")
              case "Ron":
                  print("Gryffindor")
              case "Draco":
                  print("Slytherin")
              case _:
                  print("¿Quién?")

    Observa el uso del símbolo `_` en el último caso. Esto coincidirá con cualquier entrada, resultando en un comportamiento similar a una declaración `else`.

-   Una declaración `match` compara el valor que sigue a la palabra clave `match` con cada uno de los valores que siguen a las palabras clave `case`. En caso de encontrar una coincidencia, se ejecuta la sección de código con sangría respectiva, y el programa detiene la coincidencia.
-   Podemos mejorar el código:

          name = input("¿Cuál es tu nombre? ")

          match name:
              case "Harry" | "Hermione" | "Ron":
                  print("Gryffindor")
              case "Draco":
                  print("Slytherin")
              case _:
                  print("¿Quién?")

    Nota el uso de la barra vertical única `|`. Al igual que la palabra clave `or`, esto nos permite verificar múltiples valores en la misma declaración `case`.
