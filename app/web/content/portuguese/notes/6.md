## Interfaces do Usuário <a id="interfaces-do-usuario"></a>

Uma Interface do Usuário é como os visitantes de uma página da web interagem com essa página. Nosso objetivo como desenvolvedores web é tornar essas interações o mais agradáveis possível para o usuário, e existem muitos métodos que podemos usar para fazer isso.

## Aplicações de Página Única <a id="single-page-applications"></a>

Anteriormente, se quiséssemos um site com várias páginas, conseguiríamos isso usando rotas diferentes em nossa aplicação Django. Agora, temos a capacidade de carregar apenas uma única página e depois usar JavaScript para manipular o DOM. Uma grande vantagem de fazer isso é que só precisamos modificar a parte da página que está realmente mudando. Por exemplo, se tivermos uma Barra de Navegação que não muda com base na sua página atual, não queremos ter que redesenhar essa Barra de Navegação toda vez que mudamos para uma nova parte da página.

Vamos ver um exemplo de como poderíamos simular a troca de páginas em JavaScript:

```html
<!DOCTYPE html>
<html lang="pt">
    <head>
        <title>Página Única</title>
        <style>
            div {
                display: none;
            }
        </style>
        <script src="singlepage.js"></script>
    </head>
    <body>
        <button data-page="page1">Página 1</button>
        <button data-page="page2">Página 2</button>
        <button data-page="page3">Página 3</button>
        <div id="page1">
            <h1>Esta é a página 1</h1>
        </div>
        <div id="page2">
            <h1>Esta é a página 2</h1>
        </div>
        <div id="page3">
            <h1>Esta é a página 3</h1>
        </div>
    </body>
</html>
```

Observe no HTML acima que temos três botões e três divs. No momento, as divs contêm apenas um pequeno texto, mas poderíamos imaginar cada div contendo o conteúdo de uma página em nosso site. Agora, vamos adicionar algum JavaScript que nos permite usar os botões para alternar entre as páginas.

```js
// Mostra uma página e oculta as outras duas
function showPage(page) {
    // Oculta todas as divs:
    document.querySelectorAll("div").forEach((div) => {
        div.style.display = "none";
    });

    // Mostra a div fornecida no argumento
    document.querySelector(`#${page}`).style.display = "block";
}

// Espera a página carregar:
document.addEventListener("DOMContentLoaded", function () {
    // Seleciona todos os botões
    document.querySelectorAll("button").forEach((button) => {
        // Quando um botão é clicado, muda para aquela página
        button.onclick = function () {
            showPage(this.dataset.page);
        };
    });
});
```

Em muitos casos, será ineficiente carregar o conteúdo inteiro de cada página quando visitamos um site pela primeira vez, então precisaremos usar um servidor para acessar novos dados. Por exemplo, ao visitar um site de notícias, levaria muito tempo para o site carregar se tivesse que carregar cada artigo disponível quando você visitasse a página pela primeira vez. Podemos evitar esse problema usando uma estratégia semelhante à que usamos ao carregar taxas de câmbio na aula anterior. Desta vez, vamos ver como usar Django para enviar e receber informações de nossa aplicação de página única. Para mostrar como isso funciona, vamos dar uma olhada em uma aplicação Django simples. Ela tem dois padrões de URL em `urls.py`:

```python
urlpatterns = [
    path("", views.index, name="index"),
    path("sections/<int:num>", views.section, name="section")
]
```

E duas rotas correspondentes em `views.py`. Observe que a rota `section` recebe um número inteiro e retorna uma string de texto com base nesse inteiro como uma Resposta HTTP.

```python
from django.http import Http404, HttpResponse
from django.shortcuts import render

# Crie suas views aqui.
def index(request):
    return render(request, "singlepage/index.html")

# Os textos são muito mais longos na realidade, mas foram
# encurtados aqui para economizar espaço
texts = ["Texto 1", "Texto 2", "Texto 3"]

def section(request, num):
    if 1 <= num <= 3:
        return HttpResponse(texts[num - 1])
    else:
        raise Http404("Nenhuma seção encontrada")
```

Agora, dentro do nosso arquivo `index.html`, vamos aproveitar o AJAX, que aprendemos na aula passada, para fazer uma solicitação ao servidor para obter o texto de uma seção específica e exibi-lo na tela:

```html
<!DOCTYPE html>
<html lang="pt">
    <head>
        <title>Página Única</title>
        <style></style>
        <script>
            // Mostra a seção fornecida
            function showSection(section) {
                // Encontra o texto da seção no servidor
                fetch(`/sections/${section}`)
                    .then((response) => response.text())
                    .then((text) => {
                        // Registra o texto e exibe na página
                        console.log(text);
                        document.querySelector("#content").innerHTML = text;
                    });
            }

            document.addEventListener("DOMContentLoaded", function () {
                // Adiciona funcionalidade aos botões
                document.querySelectorAll("button").forEach((button) => {
                    button.onclick = function () {
                        showSection(this.dataset.section);
                    };
                });
            });
        </script>
    </head>
    <body>
        <h1>Olá!</h1>
        <button data-section="1">Seção 1</button>
        <button data-section="2">Seção 2</button>
        <button data-section="3">Seção 3</button>
        <div id="content"></div>
    </body>
</html>
```

Agora, criamos um site onde podemos carregar novos dados de um servidor sem recarregar toda a página HTML!

Uma desvantagem do nosso site, no entanto, é que a URL agora é menos informativa. Você notará no vídeo acima que a URL permanece a mesma mesmo quando alternamos entre seções. Podemos resolver esse problema usando a [API de Histórico do JavaScript](https://developer.mozilla.org/pt-BR/docs/Web/API/History_API). Esta API nos permite adicionar informações ao histórico do navegador e atualizar a URL manualmente. Vamos ver como podemos usar essa API. Imagine que temos um projeto Django idêntico ao anterior, mas desta vez queremos alterar nosso script para usar a API de Histórico:

```js
// Quando a seta de voltar é clicada, mostra a seção anterior
window.onpopstate = function (event) {
    console.log(event.state.section);
    showSection(event.state.section);
};

function showSection(section) {
    fetch(`/sections/${section}`)
        .then((response) => response.text())
        .then((text) => {
            console.log(text);
            document.querySelector("#content").innerHTML = text;
        });
}

document.addEventListener("DOMContentLoaded", function () {
    document.querySelectorAll("button").forEach((button) => {
        button.onclick = function () {
            const section = this.dataset.section;

            // Adiciona o estado atual ao histórico
            history.pushState({ section: section }, "", `section${section}`);
            showSection(section);
        };
    });
});
```

Na função `showSection` acima, utilizamos a função `history.pushState`. Esta função adiciona um novo elemento ao histórico de navegação com base em três argumentos:

1. Qualquer dado associado ao estado.
2. Um parâmetro de título ignorado pela maioria dos navegadores da web.
3. O que deve ser exibido na URL.

A outra alteração que fazemos no JavaScript acima é configurar o parâmetro `onpopstate`, que especifica o que devemos fazer quando o usuário clica na seta de voltar. Neste caso, queremos mostrar a seção anterior quando o botão é pressionado. Agora, o site parece um pouco mais amigável ao usuário:

## Scroll <a id="scroll"></a>

Para atualizar e acessar o histórico do navegador, utilizamos um importante objeto JavaScript conhecido como [window](https://www.w3schools.com/js/js_window.asp). Existem outras propriedades do objeto window que podemos utilizar para deixar nossos sites mais atraentes:

-   `window.innerWidth`: largura da janela em pixels
-   `window.innerHeight`: Altura da janela em pixels

![medidas internas](https://cs50.harvard.edu/web/2020/notes/6/images/innerMeasures.png)

Enquanto a janela representa o que o usuário pode ver atualmente, o [document](https://www.w3schools.com/js/js_htmldom_document.asp) refere-se à página da web inteira, que muitas vezes é muito maior do que a janela, obrigando o usuário a rolar para cima e para baixo para ver o conteúdo da página. Para trabalhar com o scrolling, temos acesso a outras variáveis:

-   `window.scrollY`: Quantos pixels rolamos a partir do topo da página
-   `document.body.offsetHeight`: a altura em pixels de todo o documento.

![Medidas de rolagem](https://cs50.harvard.edu/web/2020/notes/6/images/scroll.png)

Podemos usar essas medidas para determinar se o usuário rolou até o final de uma página usando a comparação `window.scrollY + window.innerHeight >= document.body.offsetHeight`. A página seguinte, por exemplo, mudará a cor do fundo para verde quando chegarmos ao final da página:

    <!DOCTYPE html>
    <html lang="pt">
        <head>
            <title>Scroll</title>
            <script>

                // Event listener para o scrolling
                window.onscroll = () => {

                    // Verificar se estamos no final
                    if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {

                        // Mudar cor para verde
                        document.querySelector('body').style.background = 'green';
                    } else {

                        // Mudar cor para branco
                        document.querySelector('body').style.background = 'white';
                    }

                };

            </script>
        </head>
        <body>
            <p>1</p>
            <p>2</p>
            <!-- Mais parágrafos omitidos para economizar espaço -->
            <p>99</p>
            <p>100</p>
        </body>
    </html>

![rolar-verde-branco](https://cs50.harvard.edu/web/2020/notes/6/images/scrollgreen.gif)

### Rolagem Infinita <a id="infinite-scroll"></a>

Mudar a cor de fundo no final da página provavelmente não é muito útil, mas podemos querer detectar que estamos no final da página se quisermos implementar **rolagem infinita**. Por exemplo, em um site de redes sociais, você não quer carregar todas as postagens de uma vez, pode querer carregar as primeiras dez e, em seguida, quando o usuário chegar ao final, carregar as próximas dez. Vamos dar uma olhada em um aplicativo Django que poderia fazer isso. Este aplicativo tem dois caminhos em `urls.py`:

    urlpatterns = [
        path("", views.index, name="index"),
        path("posts", views.posts, name="posts")
    ]

E duas visualizações correspondentes em `views.py`:

    import time

    from django.http import JsonResponse
    from django.shortcuts import render

    # Crie suas visualizações aqui.
    def index(request):
        return render(request, "posts/index.html")

    def posts(request):

        # Obter pontos de início e fim
        start = int(request.GET.get("start") or 0)
        end = int(request.GET.get("end") or (start + 9))

        # Gerar lista de postagens
        data = []
        for i in range(start, end + 1):
            data.append(f"Post #{i}")

        # Atraso artificial para velocidade de resposta
        time.sleep(1)

        # Retornar lista de postagens
        return JsonResponse({
            "posts": data
        })

Observe que a visualização `posts` requer dois argumentos: um ponto de `início` e um ponto de `fim`. Nesta visualização, criamos nossa própria **API**, que podemos testar visitando a URL `localhost:8000/posts?start=10&end=15`, que retorna o seguinte JSON:

    {
        "posts": [
            "Post #10",
            "Post #11",
            "Post #12",
            "Post #13",
            "Post #14",
            "Post #15"
        ]
    }

Agora, no modelo `index.html` que o site carrega, começamos apenas com uma `div` vazia no corpo e alguma estilização. Observe que carregamos nossos arquivos estáticos no início e, em seguida, referenciamos um arquivo JavaScript dentro da nossa pasta `static`.

    {% load static %}
    <!DOCTYPE html>
    <html>
        <head>
            <title>Minha Página Web</title>
            <style>
                .post {
                    background-color: #77dd11;
                    padding: 20px;
                    margin: 10px;
                }

                body {
                    padding-bottom: 50px;
                }
            </style>
            <script scr="{% static 'posts/script.js' %}"></script>
        </head>
        <body>
            <div id="posts">
            </div>
        </body>
    </html>

Agora, com JavaScript, esperaremos até que um usuário role até o final da página e, em seguida, carregaremos mais postagens usando nossa API:

    // Comece com a primeira postagem
    let counter = 1;

    // Carregar postagens de 20 em 20
    const quantidade = 20;

    // Quando o DOM carregar, renderize as primeiras 20 postagens
    document.addEventListener('DOMContentLoaded', load);

    // Se rolou até o final, carregue as próximas 20 postagens
    window.onscroll = () => {
        if (window.innerHeight + window.scrollY >= document.body.offsetHeight) {
            load();
        }
    };

    // Carregar próximo conjunto de postagens
    function load() {

        // Definir números das postagens de início e fim e atualizar o contador
        const start = counter;
        const end = start + quantidade - 1;
        counter = end + 1;

        // Obter novas postagens e adicionar postagens
        fetch(`/posts?start=${start}&end=${end}`)
        .then(response => response.json())
        .then(data => {
            data.posts.forEach(add_post);
        })
    };

    // Adicione uma nova postagem com o conteúdo fornecido ao DOM
    function add_post(contents) {

        // Crie uma nova postagem
        const post = document.createElement('div');
        post.className = 'post';
        post.innerHTML = contents;

        // Adicionar postagem ao DOM
        document.querySelector('#posts').append(post);
    };

Agora, criamos um site com rolagem infinita!

## Animação <a id="animation"></a>

Uma outra forma de tornar nossos sites um pouco mais interessantes é adicionando alguma animação a eles. Descobre-se que, além de fornecer estilos, o CSS nos permite animar elementos HTML de forma fácil.

Para criar uma animação em CSS, utilizamos o formato abaixo, onde os detalhes da animação podem incluir estilos de início e fim (`from` e `to`) ou estilos em diferentes estágios na duração (de `0%` a `100%`). Por exemplo:

```
@keyframes animation_name {
    from {
        /* Alguns estilos para o início */
    }

    to {
        /* Alguns estilos para o fim */
    }
}
```

ou:

```
@keyframes animation_name {
    0% {
        /* Alguns estilos para o início */
    }

    75% {
        /* Alguns estilos depois de 3/4 da animação */
    }

    100% {
        /* Alguns estilos para o fim */
    }
}
```

Em seguida, para aplicar uma animação a um elemento, incluímos o `animation-name`, o `animation-duration` (em segundos) e o `animation-fill-mode` (normalmente `forwards`). Por exemplo, aqui está uma página onde um título cresce quando entramos na página pela primeira vez:

```
<!DOCTYPE html>
<html lang="pt">
    <head>
        <title>Animar</title>
        <style>
            @keyframes grow {
                from {
                    font-size: 20px;
                }
                to {
                    font-size: 100px;
                }
            }

            h1 {
                animation-name: grow;
                animation-duration: 2s;
                animation-fill-mode: forwards;
            }
        </style>
    </head>
    <body>
        <h1>Bem-vindo!</h1>
    </body>
</html>
```

![Título crescendo](https://cs50.harvard.edu/web/2020/notes/6/images/animate0.gif)

Podemos fazer mais do que apenas manipular o tamanho: o exemplo abaixo mostra como podemos alterar a posição de um cabeçalho apenas alterando algumas linhas:

```
<!DOCTYPE html>
<html lang="pt">
    <head>
        <title>Animar</title>
        <style>
            @keyframes move {
                from {
                    left: 0%;
                }
                to {
                    left: 50%;
                }
            }

            h1 {
                position: relative;
                animation-name: move;
                animation-duration: 2s;
                animation-fill-mode: forwards;
            }
        </style>
    </head>
    <body>
        <h1>Bem-vindo!</h1>
    </body>
</html>
```

![Cabeçalho movendo](https://cs50.harvard.edu/web/2020/notes/6/images/animate1.gif)

Agora, vamos olhar para definir algumas propriedades CSS intermediárias também. Podemos especificar o estilo em qualquer porcentagem durante uma animação. No exemplo abaixo, moveremos o título da esquerda para a direita e, em seguida, de volta para a esquerda, alterando apenas a animação de acima:

```
@keyframes move {
    0% {
        left: 0%;
    }
    50% {
        left: 50%;
    }
    100% {
        left: 0%;
    }
}
```

![para trás e para frente](https://cs50.harvard.edu/web/2020/notes/6/images/animate2.gif)

Se quisermos repetir uma animação várias vezes, podemos alterar o `animation-iteration-count` para um número maior que um (ou até mesmo `infinite` para uma animação infinita). Existem muitas [propriedades de animação](https://www.w3schools.com/cssref/css3_pr_animation.asp) que podemos definir para mudar diferentes aspectos da nossa animação.

Além do CSS, podemos usar JavaScript para controlar ainda mais nossas animações. Vamos usar nosso exemplo de cabeçalho em movimento (com repetição infinita) para mostrar como podemos criar um botão que inicia e pausa a animação. Assumindo que já temos uma animação, botão e cabeçalho, podemos adicionar o seguinte script para começar e pausar a animação:

```
document.addEventListener('DOMContentLoaded', function() {

    // Encontrar cabeçalho
    const h1 = document.querySelector('h1');

    // Pausar animação por padrão
    h1.style.animationPlayState = 'paused';

    // Aguardar o botão ser clicado
    document.querySelector('button').onclick = () => {

        // Se a animação estiver pausada, comece a reproduzi-la
        if (h1.style.animationPlayState == 'paused') {
            h1.style.animationPlayState = 'running';
        }

        // Caso contrário, pause a animação
        else {
            h1.style.animationPlayState = 'paused';
        }
    }

})
```

![reproduzir/pausar animação](https://cs50.harvard.edu/web/2020/notes/6/images/animate4.gif)

Agora, vejamos como podemos aplicar nosso novo conhecimento sobre animações à página de posts que fizemos anteriormente. Especificamente, digamos que queremos a capacidade de ocultar posts assim que terminarmos de lê-los. Vamos imaginar um projeto Django idêntico ao que acabamos de criar, porém com algum HTML e JavaScript ligeiramente diferente. A primeira alteração que faremos será na função `add_post`, adicionando também um botão ao lado direito do post:

```
// Adicionar um novo post com o conteúdo fornecido ao DOM
function add_post(contents) {

    // Criar novo post
    const post = document.createElement('div');
    post.className = 'post';
    post.innerHTML = `${contents} <button class="hide">Esconder</button>`;

    // Adicionar post ao DOM
    document.querySelector('#posts').append(post);
};
```

Agora, vamos trabalhar em ocultar um post quando o botão `esconder` é clicado. Para fazer isso, adicionamos um ouvinte de evento acionado sempre que um usuário clica em qualquer lugar na página. Em seguida, escrevemos uma função que recebe o `evento` como argumento, o que é útil porque podemos usar o atributo `event.target` para acessar o que foi clicado. Podemos também usar a classe `parentElement` para encontrar o pai de um determinado elemento no DOM.

```
// Se o botão esconder for clicado, excluir o post
document.addEventListener('click', event => {

    // Encontrar o que foi clicado
    const elemento = event.target;

    // Verificar se o usuário clicou em um botão esconder
    if (elemento.className === 'hide') {
        elemento.parentElement.remove()
    }

});
```

![ocultar ingênuo](https://cs50.harvard.edu/web/2020/notes/6/images/hide0.gif)

Agora podemos ver que implementamos o botão de ocultar, mas não parece tão agradável quanto poderia. Talvez queiramos fazer com que o post desapareça lentamente e encolha antes de removê-lo. Para fazer isso, primeiro criaremos uma animação CSS. A animação abaixo passará 75% do tempo alterando a `opacidade` de 1 para 0, o que faz com que o post desapareça lentamente. Em seguida, ele passa o restante do tempo movendo todos os seus atributos relacionados à `altura` para 0, encolhendo efetivamente o post até desaparecer.

```
@keyframes hide {
    0% {
        opacity: 1;
        height: 100%;
        line-height: 100%;
        padding: 20px;
        margin-bottom: 10px;
    }
    75% {
        opacity: 0;
        height: 100%;
        line-height: 100%;
        padding: 20px;
        margin-bottom: 10px;
    }
    100% {
        opacity: 0;
        height: 0px;
        line-height: 0px;
        padding: 0px;
        margin-bottom: 0px;
    }
}
```

Em seguida, adicionamos esta animação ao CSS do nosso post. Note que inicialmente definimos o `animation-play-state` como `paused`, significando que o post não será ocultado por padrão.

```
.post {
    background-color: #77dd11;
    padding: 20px;
    margin-bottom: 10px;
    animation-name: hide;
    animation-duration: 2s;
    animation-fill-mode: forwards;
    animation-play-state: paused;
}
```

Por fim, queremos poder iniciar a animação assim que o botão `esconder` for clicado e, em seguida, remover o post. Podemos fazer isso editando nosso JavaScript de cima:

```
// Se o botão esconder for clicado, excluir o post
document.addEventListener('click', event => {

    // Encontrar o que foi clicado
    const elemento = event.target;

    // Verificar se o usuário clicou em um botão esconder
    if (elemento.className === 'hide') {
        elemento.parentElement.style.animationPlayState = 'running';
        elemento.parentElement.addEventListener('animationend', () => {
            elemento.parentElement.remove();
        });
    }

});
```

## React <a id="react"></a>

Neste ponto, você pode imaginar o quanto de código JavaScript seria necessário para um site mais complicado. Podemos mitigar a quantidade de código que realmente precisamos escrever ao usar um framework de JavaScript, assim como usamos o Bootstrap como um framework CSS para reduzir a quantidade de CSS que realmente tínhamos que escrever. Um dos frameworks de JavaScript mais populares é uma biblioteca chamada [React](https://reactjs.org/).

Até agora neste curso, estivemos usando métodos de **programação imperativa**, onde fornecemos ao computador um conjunto de instruções para executar. Por exemplo, para atualizar o contador em uma página HTML, poderíamos ter um código que se parece com isso:

Visualização:

    <h1>0</h1>

Lógica:

    let num = parseInt(document.querySelector("h1").innerHTML);
    num += 1;
    document.querySelector("h1").innerHTML = num;

React nos permite usar a **programação declarativa**, que nos permitirá simplesmente escrever código explicando _o que_ desejamos exibir e não nos preocupar _como_ estamos exibindo isso. Em React, um contador pode ser algo mais assim:

Visualização:

    <h1>{num}</h1>

Lógica:

    num += 1;

O framework React é construído em torno da ideia de componentes, cada um dos quais pode ter um estado subjacente. Um componente seria algo que você pode ver em uma página da web, como um post ou uma barra de navegação, e um estado é um conjunto de variáveis associadas a esse componente. A beleza do React é que quando o estado muda, o React automaticamente altera o DOM em conformidade.

Existem várias maneiras de usar o React, (incluindo o popular comando [create-react-app](https://reactjs.org/docs/create-a-new-react-app.html) publicado pelo Facebook), mas hoje vamos nos concentrar em começar diretamente em um arquivo HTML. Para fazer isso, teremos que importar três Pacotes JavaScript:

-   `React`: Define componentes e seu comportamento
-   `ReactDOM`: Pega componentes do React e os insere no DOM
-   `Babel`: Traduz de [JSX](https://reactjs.org/docs/introducing-jsx.html), a linguagem na qual escreveremos em React, para JavaScript simples que nossos navegadores podem interpretar. O JSX é muito semelhante ao JavaScript, mas com alguns recursos adicionais, incluindo a capacidade de representar HTML dentro do nosso código.

Vamos mergulhar e criar nossa primeira aplicação React!

    <!DOCTYPE html>
    <html lang="pt-BR">
        <head>
            <script src="https://unpkg.com/react@17/umd/react.production.min.js" crossorigin></script>
            <script src="https://unpkg.com/react-dom@17/umd/react-dom.production.min.js" crossorigin></script>
            <script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>
            <title>Olá</title>
        </head>
        <body>
            <div id="app"></div>

            <script type="text/babel">
                function App() {
                    return (
                        <div>
                            Olá!
                        </div>
                    );
                }

                ReactDOM.render(<App />, document.querySelector("#app"));
            </script>
        </body>
    </html>

Como esta é nossa primeira aplicação React, vamos analisar em detalhes o que cada parte desse código está fazendo:

-   Nas três linhas acima do título, importamos as versões mais recentes do React, ReactDOM e Babel.
-   No corpo, incluímos um único `div` com um `id` de `app`. Quase sempre queremos deixar isso vazio e preenchê-lo em nosso código react abaixo.
-   Incluímos uma tag de script onde especificamos `type="text/babel"`. Isso sinaliza ao navegador que o script a seguir precisa ser traduzido usando o Babel.
-   Em seguida, criamos um componente chamado `App`. Componentes no React podem ser representados por funções JavaScript.
-   Nosso componente retorna o que gostaríamos de renderizar no DOM. Neste caso, simplesmente retornamos `<div>Olá!</div>`.
-   A última linha do nosso script usa a função `ReactDOM.render`, que recebe dois argumentos:
    1. Um componente a ser renderizado
    2. Um elemento no DOM dentro do qual o componente deve ser renderizado

Agora que entendemos o que o código está fazendo, vamos dar uma olhada na página resultante:

![welcome hello react](https://cs50.harvard.edu/web/2020/notes/6/images/react0.png)

Um recurso útil do React é a capacidade de renderizar componentes dentro de outros componentes. Para demonstrar isso, vamos criar outro componente chamado `Hello`:

    function Hello(props) {
        return (
            <h1>Olá</h1>
        );
    }

E agora, vamos renderizar três componentes `Hello` dentro do nosso componente `App`:

    function App() {
        return (
            <div>
                <Hello />
                <Hello />
                <Hello />
            </div>
        );
    }

Isso nos dá uma página que se parece com:

![Three hellos](https://cs50.harvard.edu/web/2020/notes/6/images/react1.png)

Até agora, os componentes não foram tão interessantes, pois são todos exatamente iguais. Podemos tornar esses componentes mais flexíveis adicionando propriedades adicionais (**props** em termos do React) a eles. Por exemplo, digamos que desejamos saudar três pessoas diferentes. Podemos fornecer os nomes dessas pessoas em um método que se parece com atributos HTML:

    function App() {
        return (
            <div>
                <Hello name="Harry" />
                <Hello name="Ron" />
                <Hello name="Hermione" />
            </div>
        );
    }

Podemos então acessar essas props usando `props.PROP_NAME`. Podemos então inserir isso em nosso JSX usando chaves:

    function Hello(props) {
        return (
            <h1>Olá, {props.name}!</h1>
        );
    }

Agora, nossa página exibe os três nomes!

![Three names](https://cs50.harvard.edu/web/2020/notes/6/images/react2.png)

Agora, vejamos como podemos usar o React para reimplantar a página do contador que criamos quando estávamos trabalhando pela primeira vez com JavaScript. Nossa estrutura geral permanecerá a mesma, mas dentro do nosso componente `App`, usaremos o `useState` do React para adicionar estado ao nosso componente. O argumento para `useState` é o valor inicial do estado, que definiremos como `0`. A função retorna tanto uma variável representando o estado quanto uma função que nos permite atualizar o estado.

    const [count, setCount] = React.useState(0);

Agora, podemos trabalhar no que a função irá renderizar, onde especificaremos um cabeçalho e um botão. Também adicionaremos um ouvinte de evento para quando o botão for clicado, o que o React manipula usando o atributo `onClick`:

    return (
        <div>
            <h1>{count}</h1>
            <button onClick={updateCount}>Contar</button>
        </div>
    );

Finalmente, vamos definir a função `updateCount`. Para fazer isso, usaremos a função `setCount`, que pode aceitar como argumento um novo valor para o estado.

    function updateCount() {
        setCount(count + 1);
    }

Agora temos um site de contador funcional!

### Adição <a id="addition"></a>

Agora que temos uma ideia do framework React, vamos trabalhar usando o que aprendemos para construir um site semelhante a um jogo onde os usuários resolverão problemas de adição. Começaremos criando um novo arquivo com a mesma configuração de nossas outras páginas React. Para começar a construir esta aplicação, vamos pensar no que pode ser importante manter controlado no estado. Devemos incluir qualquer coisa que achamos que pode mudar enquanto um usuário estiver em nossa página. Nosso estado poderia incluir:

-   `num1`: O primeiro número a ser adicionado
-   `num2`: O segundo número a ser adicionado
-   `response`: O que o usuário digitou
-   `score`: Quantas perguntas o usuário respondeu corretamente.

Agora, nosso estado pode ser um objeto JavaScript que inclui todas essas informações:

    const [state, setState] = React.useState({
        num1: 1,
        num2: 1,
        response: "",
        score: 0
    });

Agora, usando os valores no estado, podemos renderizar uma interface de usuário básica.

    return (
        <div>
            <div>{state.num1} + {state.num2}</div>
            <input value={state.response} />
            <div>Pontuação: {state.score}</div>
        </div>
    );

Agora, o layout básico do site fica assim:

![Addition layout](https://cs50.harvard.edu/web/2020/notes/6/images/add0.png)

Neste ponto, o usuário não pode digitar nada na caixa de input porque seu valor é fixo como `state.response`, que atualmente é uma string vazia. Para corrigir isso, vamos adicionar um atributo `onChange` ao elemento de input e definir igual a uma função chamada `updateResponse`

    onChange={updateResponse}

Agora, teremos que definir a função `updateResponse`, que recebe o evento que acionou a função e define `response` para o valor atual do input. Esta função permite ao usuário digitar e armazena o que foi digitado no `state`.

    function updateResponse(event) {
        setState({
            ...state,
            response: event.target.value
        });
    }

Agora, vamos adicionar a capacidade para um usuário enviar um problema. Vamos primeiro adicionar outro ouvinte de evento e vinculá-lo a uma função que escreveremos em seguida:

    onKeyPress={inputKeyPress}

Agora, vamos definir a função `inputKeyPress`. Nesta função, primeiro verificaremos se a tecla `Enter` foi pressionada e depois verificamos se a resposta está correta. Quando o usuário está correto, queremos aumentar a pontuação em 1, escolher números aleatórios para o próximo problema e limpar a resposta. Se a resposta estiver incorreta, queremos diminuir a pontuação em 1 e limpar a resposta.

    function inputKeyPress(event) {
        if (event.key === "Enter") {
            const answer = parseInt(state.response);
            if (answer === state.num1 + state.num2) {
                // O usuário acertou a pergunta
                setState({
                    ...state,
                    score: state.score + 1,
                    response: "",
                    num1: Math.ceil(Math.random() * 10),
                    num2: Math.ceil(Math.random() * 10)
                });
            } else {
                // O usuário errou a pergunta
                setState({
                    ...state,
                    score: state.score - 1,
                    response: ""
                })
            }
        }
    }

Para finalizar a aplicação, vamos adicionar algum estilo à página. Vamos centralizar tudo no aplicativo e depois aumentar o problema adicionando um `id` de `problem` ao div que contém o problema, e então adicionando o seguinte CSS a uma tag de estilo:

    #app {
        text-align: center;
        font-family: sans-serif;
    }

    #problem {
        font-size: 72px;
    }

Por fim, vamos adicionar a capacidade de ganhar o jogo após obter 10 pontos. Para fazer isso, adicionaremos uma condição à função `render`, retornando algo completamente diferente uma vez que alcançarmos 10 pontos:

    if (state.score === 10) {
        return (
            <div id="vencedor">Você ganhou!</div>
        );
    }

Para tornar a vitória mais emocionante, vamos adicionar algum estilo à div alternativa também:

    #vencedor {
        font-size: 72px;
        color: verde;
    }

Agora, vamos dar uma olhada em nossa aplicação!
