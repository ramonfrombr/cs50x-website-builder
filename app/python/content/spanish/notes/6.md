## Entrada/Salida de Archivos <a id="file-io"></a>

- Hasta ahora, todo lo que hemos programado ha almacenado información en la memoria. Es decir, una vez que el programa termina, toda la información recopilada del usuario o generada por el programa se pierde.
- La entrada/salida de archivos es la capacidad de un programa para tomar un archivo como entrada o crear un archivo como salida.
- Para comenzar, en la ventana de la terminal escriba `code names.py` y codifique de la siguiente manera:

      name = input("¿Cuál es tu nombre?")
      print(f"Hola, {name}")

  Observe que al ejecutar este código se obtiene la salida deseada. El usuario puede ingresar un nombre. La salida es la esperada.

- Sin embargo, ¿qué pasaría si quisiéramos permitir múltiples nombres como entrada? ¿Cómo podríamos lograrlo? Recuerde que una `lista` es una estructura de datos que nos permite almacenar múltiples valores en una sola variable. Codifique de la siguiente manera:

      names = []

      for _ in range(3):
          name = input("¿Cuál es tu nombre?")
          names.append(name)

  Observe que al usuario se le solicitará la entrada tres veces. El método `append` se utiliza para agregar el `name` a nuestra lista `names`.

- Este código podría simplificarse de la siguiente manera:

      names = []

      for _ in range(3):
          names.append(input("¿Cuál es tu nombre?"))

  Observe que esto tiene el mismo resultado que el bloque anterior de código.

- Ahora, permitamos la capacidad de imprimir la lista de nombres como una lista ordenada. Codifique de la siguiente manera:

    names = []

    for \_ in range(3):
    names.append(input("¿Cuál es tu nombre?"))

    for name in sorted(names):
    print(f"Hola, {name}")

  Observe que una vez que se ejecuta este programa, toda la información se pierde. La entrada/salida de archivos permite que su programa almacene esta información de modo que pueda ser utilizada posteriormente.

- Puede obtener más información en la documentación de Python sobre [sorted](https://docs.python.org/3/library/functions.html#sorted).

## `open` <a id="open"></a>

- `open` es una funcionalidad incorporada en Python que te permite abrir un archivo y utilizarlo en tu programa. La función `open` te permite abrir un archivo para leer de él o escribir en él.
- Para mostrarte cómo habilitar la E/S de archivos en tu programa, retrocedamos un poco y codifiquemos de la siguiente manera:

      name = input("¿Cuál es tu nombre? ")

      file = open("nombres.txt", "w")
      file.write(name)
      file.close()

  Observa que la función `open` abre un archivo llamado `nombres.txt` con permisos de escritura, como se señaliza con la `w`. El código anterior asigna ese archivo abierto a una variable llamada `file`. La línea `file.write(name)` escribe el nombre en el archivo de texto. La línea siguiente cierra el archivo.

- Probando tu código escribiendo `python nombres.py`, puedes ingresar un nombre y se guardará en el archivo de texto. Sin embargo, si ejecutas tu programa múltiples veces usando nombres distintos, notarás que este programa reescribirá completamente el archivo `nombres.txt` cada vez.
- Idealmente, queremos poder añadir cada uno de nuestros nombres al archivo. Elimina el archivo de texto existente escribiendo `rm nombres.txt` en la ventana de terminal. Luego, modifica tu código de la siguiente manera:

      name = input("¿Cuál es tu nombre? ")

      file = open("nombres.txt", "a")
      file.write(name)
      file.close()

  Observa que el único cambio en nuestro código es que la `w` ha sido cambiada por `a` para “añadir”. Al ejecutar nuevamente este programa múltiples veces, notarás que los nombres se agregarán al archivo. ¡Sin embargo, notarás un nuevo problema!

- Examinando tu archivo de texto después de ejecutar tu programa varias veces, notarás que los nombres se están concatenando. Los nombres se están añadiendo sin ningún espacio entre cada uno de ellos. Puedes solucionar este problema. Nuevamente, elimina el archivo de texto existente escribiendo `rm nombres.txt` en la ventana de terminal. Luego, modifica tu código de la siguiente manera:

      name = input("¿Cuál es tu nombre? ")

      file = open("nombres.txt", "a")
      file.write(f"{name}\n")
      file.close()

  Observa que la línea con `file.write` ha sido modificada para agregar un salto de línea al final de cada nombre.

- Este código está funcionando muy bien. Sin embargo, hay formas de mejorar este programa. Resulta que es bastante fácil olvidar cerrar el archivo.
- Puedes aprender más en la documentación de Python sobre [open](https://docs.python.org/3/library/functions.html#open).

## `with` <a id="with"></a>

- La palabra clave `with` te permite automatizar el cierre de un archivo.
- Modifica tu código de la siguiente manera:

      name = input("¿Cuál es tu nombre? ")

      with open("names.txt", "a") as file:
          file.write(f"{name}\n")

  Observa que la línea debajo de `with` está sangrada.

- Hasta este punto, hemos estado escribiendo exclusivamente en un archivo. ¿Qué pasa si queremos leer desde un archivo? Para habilitar esta funcionalidad, modifica tu código de la siguiente manera:

      with open("names.txt", "r") as file:
          lines = file.readlines()

      for line in lines:
          print("¡Hola,", line)

  Observa que `readlines` tiene la habilidad especial de leer todas las líneas de un archivo y almacenarlas en una variable llamada lines. Al ejecutar tu programa, notarás que la salida es un poco desordenada. Parece haber múltiples saltos de línea donde solo debería haber uno.

- Hay muchas formas de solucionar este problema. Sin embargo, aquí hay una forma sencilla de corregir este error en nuestro código:

      with open("names.txt", "r") as file:
          lines = file.readlines()

      for line in lines:
          print("¡Hola,", line.rstrip())

  Observa que `rstrip` tiene el efecto de eliminar el salto de línea sobrante al final de cada línea.

- Aún así, este código podría simplificarse aún más:

      with open("names.txt", "r") as file:
          for line in file:
              print("¡Hola,", line.rstrip())

  Observa que al ejecutar este código, es correcto. Sin embargo, observa que no estamos ordenando los nombres.

- Este código podría mejorarse aún más para permitir la ordenación de los nombres:

      names = []

      with open("names.txt") as file:
          for line in file:
              names.append(line.rstrip())

      for name in sorted(names):
          print(f"¡Hola, {name}")

  Observa que `names` es una lista vacía donde podemos recopilar los nombres. Cada nombre se agrega a la lista `names` en memoria. Luego, se imprime cada nombre en la lista ordenada en memoria. Al ejecutar tu código, verás que los nombres ahora están correctamente ordenados.

## CSV <a id="csv"></a>

- CSV significa “valores separados por comas”.
- En la ventana de tu terminal, escribe `code students.csv`. Asegúrate de que tu nuevo archivo CSV se vea así:

      Hermoine, Gryffindor
      Harry, Gryffindor
      Ron, Gryffindor
      Draco, Slytherin

- Creemos un nuevo programa escribiendo `code students.py` y código de la siguiente manera:

      with open("students.csv") as file:
          for line in file:
              row = line.rstrip().split(",")
              print(f"{row[0]} está en {row[1]}")

  Observa que `rstrip` elimina el final de cada línea en nuestro archivo CSV. `split` le indica al compilador dónde encontrar el final de cada uno de nuestros valores en el archivo CSV. `row[0]` es el primer elemento en cada línea de nuestro archivo CSV. `row[1]` es el segundo elemento en cada línea de nuestro archivo CSV.

- El código anterior es efectivo para dividir cada línea o “registro” de nuestro archivo CSV. Sin embargo, puede resultar un poco críptico si no estás familiarizado con este tipo de sintaxis. Python tiene la capacidad integrada que podría simplificar aún más este código. Modifica tu código de la siguiente manera:

      with open("students.csv") as file:
          for line in file:
              name, house = line.rstrip().split(",")
              print(f"{name} está en {house}")

  Observa que la función `split` en realidad devuelve dos valores: el antes de la coma y el después de la coma. Por lo tanto, podemos aprovechar esa funcionalidad para asignar dos variables a la vez en lugar de una.

- ¿Imagina si nuevamente quisiéramos proporcionar esta lista como salida ordenada? Puedes modificar tu código de la siguiente manera:

      students = []

      with open("students.csv") as file:
          for line in file:
              name, house = line.rstrip().split(",")
              students.append(f"{name} está en {house}")

      for student in sorted(students):
          print(student)

  Observa que creamos una `lista` llamada `students`. Agregamos cada cadena a esta lista. Luego, mostramos una versión ordenada de nuestra lista.

- Recuerda que Python permite `diccionarios` donde una clave puede estar asociada con un valor. Este código podría mejorarse aún más de la siguiente manera:

      students = []

      with open("students.csv") as file:
          for line in file:
              name, house = line.rstrip().split(",")
              student = {}
              student["name"] = name
              student["house"] = house
              students.append(student)

      for student in students:
          print(f"{student['name']} está en {student['house']}")

  Observa que creamos un diccionario vacío llamado `student`. Añadimos los valores de cada estudiante, incluyendo su nombre y casa, al diccionario `student`. Luego, añadimos ese estudiante a la `lista` llamada `students`.

- Podemos mejorar nuestro código de la siguiente manera:

      students = []

      with open("students.csv") as file:
          for line in file:
              name, house = line.rstrip().split(",")
              student = {"name": name, "house": house}
              students.append(student)

      for student in students:
          print(f"{student['name']} está en {student['house']}")

  Observa que esto produce el resultado deseado, sin la clasificación de estudiantes.

- Lamentablemente, no podemos clasificar a los estudiantes como antes porque cada estudiante es ahora un diccionario dentro de una lista. Sería útil si Python pudiera ordenar la lista de `students` de diccionarios de `student` que ordena esta lista de diccionarios por el nombre del estudiante.
- Para implementar esto en nuestro código, realiza los siguientes cambios:

      students = []

      with open("students.csv") as file:
          for line in file:
              name, house = line.rstrip().split(",")
              students.append({"name": name, "house": house})


      def get_name(student):
          return student["name"]


      for student in sorted(students, key=get_name):
          print(f"{student['name']} está en {student['house']}")

  Observa que `sorted` necesita saber cómo obtener la clave de cada estudiante. Python permite un parámetro llamado `key` donde podemos definir en qué "clave" se ordenará la lista de estudiantes. Por lo tanto, la función `get_name` simplemente devuelve la clave de `student["name"]`. Al ejecutar este programa, ahora verás que la lista está ordenada por nombre.

- Aún así, nuestro código puede mejorarse aún más. Resulta que si solo necesitas usar una función como `get_name` una vez, puedes simplificar tu código de la siguiente manera. Modifica tu código de la siguiente manera:

      students = []

      with open("students.csv") as file:
          for line in file:
              name, house = line.rstrip().split(",")
              students.append({"name": name, "house": house})

      for student in sorted(students, key=lambda student: student["name"]):
          print(f"{student['name']} está en {student['house']}")

  Observa cómo usamos una función `lambda`, una función anónima, que dice “Oye Python, aquí tienes una función que no tiene nombre: Dado un `student`, accede a su `name` y devuélvelo a la `clave`.

- Desafortunadamente, nuestro código es un poco frágil. Supongamos que cambiamos nuestro archivo CSV de tal manera que indicamos dónde creció cada estudiante. ¿Cuál sería el impacto de esto en nuestro programa? Primero, modifica tu archivo `students.csv` de la siguiente manera:

  Harry,"Number Four, Privet Drive"
  Ron,The Burrow
  Draco,Malfoy Manor

Observa cómo ahora al ejecutar nuestro programa se producirán varios errores.

- Ahora que estamos tratando con hogares en lugar de casas, modifica tu código de la siguiente manera:

      students = []

      with open("students.csv") as file:
          for line in file:
              name, home = line.rstrip().split(",")
              students.append({"name": name, "home": home})

      for student in sorted(students, key=lambda student: student["name"]):
          print(f"{student['name']} está en {student['home']}")

  Observa que ejecutar nuestro programa todavía no funciona correctamente. ¿Puedes adivinar por qué?

- El error `ValueError: too many values to unpack` producido por el compilador es resultado del hecho de que anteriormente creamos este programa esperando que el archivo CSV sea `split` utilizando una `,` (coma). Podríamos dedicar más tiempo a abordar esto, ¡pero de hecho alguien más ya ha desarrollado una forma de “analizar” (es decir, leer) archivos CSV!
- La biblioteca `csv` integrada de Python viene con un objeto llamado un `reader`. Como sugiere el nombre, podemos usar un `reader` para leer nuestro archivo CSV a pesar de la coma adicional en “Number Four, Privet Drive”. Un `reader` funciona en un ciclo `for`, donde en cada iteración el `reader` nos da otra fila de nuestro archivo CSV. Esta fila en sí misma es una lista, donde cada valor en la lista corresponde a un elemento en esa fila. `row[0]`, por ejemplo, es el primer elemento de la fila dada, mientras que `row[1]` es el segundo elemento.

      import csv

      students = []

      with open("students.csv") as file:
          reader = csv.reader(file)
          for row in reader:
              students.append({"name": row[0], "home": row[1]})

      for student in sorted(students, key=lambda student: student["name"]):
          print(f"{student['name']} es de {student['home']}")

  Observa que ahora nuestro programa funciona como se esperaba.

- Hasta este punto, hemos confiado en que nuestro programa decida específicamente qué partes de nuestro archivo CSV son los nombres y qué partes son los hogares. Sin embargo, es un mejor diseño incorporar esto directamente en nuestro archivo CSV editándolo de la siguiente manera:

      name,home
      Harry,"Number Four, Privet Drive"
      Ron,The Burrow
      Draco,Malfoy Manor

  Observa cómo estamos diciendo explícitamente en nuestro archivo CSV que cualquier cosa que lo lea debería esperar que haya un valor de nombre y un valor de hogar en cada línea.

- Podemos modificar nuestro código para utilizar una parte de la biblioteca `csv` llamada `DictReader` para tratar nuestro archivo CSV con aún más flexibilidad:

      import csv

      students = []

      with open("students.csv") as file:
          reader = csv.DictReader(file)
          for row in reader:
              students.append({"name": row["name"], "home": row["home"]})

      for student in sorted(students, key=lambda student: student["name"]):
          print(f"{student['name']} está en {student['home']}")

  Observa que hemos reemplazado `reader` con `DictReader`, que devuelve un diccionario a la vez. Además, observa que el compilador accederá directamente al diccionario `row`, obteniendo el `name` y `home` de cada estudiante. Esto es un ejemplo de programación defensiva. Si la persona que diseña el archivo CSV ha ingresado la información de encabezado correcta en la primera línea, podemos acceder a esa información utilizando nuestro programa.

- Hasta este punto, hemos estado leyendo archivos CSV. ¿Qué pasa si queremos escribir en un archivo CSV?
- Para empezar, limpiemos un poco nuestros archivos. Primero, elimina el archivo `students.csv` escribiendo `rm students.csv` en la ventana de la terminal. Este comando solo funcionará si estás en la misma carpeta que tu archivo `students.csv`.
- Luego, en `students.py`, modifica tu código de la siguiente manera:

      import csv

      name = input("¿Cuál es tu nombre? ")
      home = input("¿Dónde está tu hogar? ")

      with open("students.csv", "a") as file:
          writer = csv.DictWriter(file, fieldnames=["name", "home"])
          writer.writerow({"name": name, "home": home})

  Observa cómo estamos aprovechando la funcionalidad integrada de `DictWriter`, que toma dos parámetros: el `archivo` al que se está escribiendo y los `fieldnames` a escribir. Además, observa cómo la función `writerow` toma un diccionario como parámetro. Literalmente, le estamos diciendo al compilador que escriba una fila con dos campos llamados `name` y `home`.

- Ten en cuenta que hay muchos tipos de archivos desde los que puedes leer y a los que puedes escribir.
- Puedes aprender más en la documentación de Python sobre [CSV](https://docs.python.org/3/library/csv.html).

## Archivos binarios y `PIL` <a id="binary-files-and-pil"></a>

- Un tipo más de archivo del que discutiremos hoy es un archivo binario. Un archivo binario es simplemente una colección de unos y ceros. Este tipo de archivo puede almacenar cualquier cosa, incluidos datos de música e imágenes.
- Hay una popular biblioteca de Python llamada `PIL` que funciona bien con archivos de imagen.
- Los GIF animados son un tipo popular de archivo de imagen que contiene muchas imágenes que se reproducen en secuencia una y otra vez, creando una animación o efecto de video simple.

- En la ventana de la terminal, escribe `code costumes.py` y codifica lo siguiente:

      import sys

      from PIL import Image

      images = []

      for arg in sys.argv[1:]:
          image = Image.open(arg)
          images.append(image)

      images[0].save(
          "costumes.gif", save_all=True, append_images=[images[1]], duration=200, loop=0
      )

  Observa que importamos la funcionalidad `Image` de `PIL`. Observa que el primer bucle `for` simplemente recorre las imágenes proporcionadas como argumentos de línea de comandos y las almacena en la lista llamada `images`. El `1:` comienza a rebanar `argv` en su segundo elemento. Las últimas líneas de código guardan la primera imagen y también añaden una segunda imagen, creando un gif animado. Escribe `python costumes.py costume1.gif costume2.gif` en la terminal. Ahora, escribe `code costumes.gif` en la ventana de la terminal, y podrás ver un GIF animado.
