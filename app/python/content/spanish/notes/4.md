## Bibliotecas <a id="libraries"></a>

- En general, las bibliotecas son fragmentos de código escritos por ti u otras personas que puedes utilizar en tu programa.
- Python te permite compartir funciones o características con otras personas como "módulos".
- Si copias y pegas código de un proyecto antiguo, lo más probable es que puedas crear un módulo o biblioteca que podrías incorporar en tu nuevo proyecto.

## Aleatorio <a id="random"></a>

- `random` es una biblioteca que viene con Python y que puedes importar en tu propio proyecto.
- Es más fácil para un programador pararse sobre los hombros de programadores anteriores.
- Entonces, ¿cómo cargas un módulo en tu propio programa? Puedes usar la palabra `import` en tu programa.
- Dentro del módulo `random`, hay una función incorporada llamada `random.choice(seq)`. `random` es el módulo que estás importando. Dentro de ese módulo, está la función `choice`. Esa función toma como argumento una secuencia `seq` que es una lista.
- En la ventana de tu terminal escribe `code generate.py`. En tu editor de texto, escribe el siguiente código:

      import random

      coin = random.choice(["cara", "cruz"])
      print(coin)

  Nota que la lista dentro de `choice` tiene corchetes cuadrados, comillas y una coma. Dado que has pasado dos elementos, Python hace el cálculo y otorga una probabilidad del 50% para cara y cruz. Al ejecutar tu código, notarás que este código, de hecho, funciona correctamente.

- Podemos mejorar nuestro código. `from` nos permite ser muy específicos sobre lo que queremos importar. Antes, nuestra línea de código `import` está trayendo todo el contenido de las funciones de `random`. Sin embargo, ¿qué pasa si solo queremos cargar una pequeña parte de un módulo? Modifica tu código de la siguiente manera:

      from random import choice

      coin = choice(["cara", "cruz"])
      print(coin)

  Observa que ahora podemos importar solo la función `choice` de `random`. A partir de ese momento, ya no necesitamos escribir `random.choice`. Ahora solo podemos escribir `choice` únicamente. `choice` se carga explícitamente en nuestro programa. ¡Esto ahorra recursos del sistema y potencialmente puede hacer que nuestro código se ejecute más rápido!

- Continuando, considera la función `random.randint(a, b)`. Esta función generará un número aleatorio entre `a` y `b`. Modifica tu código de la siguiente manera:

      import random

      numero = random.randint(1, 10)
      print(numero)

  Nota que nuestro código generará aleatoriamente un número entre `1` y `10`.

- Podemos introducir en nuestra tarjeta `random.shuffle(x)` donde se barajará una lista en un orden aleatorio.

      import random

      cartas = ["jota", "reina", "rey"]
      random.shuffle(cartas)
      for carta in cartas:
          print(carta)

  Nota que `random.shuffle` barajará las cartas en su lugar. A diferencia de otras funciones, no devolverá un valor. En su lugar, tomará la lista `cartas` y las barajará dentro de esa lista. Ejecuta tu código varias veces para ver el funcionamiento del código.

- Ahora tenemos estas tres formas anteriores de generar información aleatoria.
- Puedes aprender más en la documentación de Python sobre [`random`](https://docs.python.org/3/library/random.html).

## Argumentos de línea de comandos <a id="argumentos-de-linea-de-comandos"></a>

- Hasta ahora, hemos estado proporcionando todos los valores dentro del programa que creamos. ¿Qué sucedería si quisiéramos poder recibir entradas desde la línea de comandos? Por ejemplo, en lugar de escribir `python average.py` en la terminal, ¿qué pasaría si quisiéramos escribir `python average.py 100 90` y obtener el promedio entre `100` y `90`?
- `sys` es un módulo que nos permite recibir argumentos en la línea de comandos.
- `argv` es una función dentro del módulo `sys` que nos permite conocer lo que el usuario escribió en la línea de comandos. Observa cómo se utiliza `sys.argv` en el código a continuación. En la ventana de la terminal, escribe `código nombre.py`. En el editor de texto, codifica lo siguiente:

      import sys

      print("hola, mi nombre es", sys.argv[1])

  Observa que el programa va a revisar lo que el usuario escribió en la línea de comandos. Actualmente, si escribes `python name.py David` en la ventana de la terminal, verás `hola, mi nombre es David`. Observa que `sys.argv[1]` es donde se almacena `David`. ¿Por qué sucede esto? Bueno, en lecciones anteriores, recordarás que las listas comienzan en el elemento `0`. ¿Qué crees que se almacena actualmente en `sys.argv[0]`? Si adivinaste `name.py`, ¡estarías en lo correcto!

- Hay un pequeño problema con nuestro programa tal como está. ¿Qué sucede si el usuario no escribe el nombre en la línea de comandos? Pruébalo tú mismo. Escribe `python name.py` en la ventana de la terminal. El compilador presentará un error `índice de lista fuera de rango`. La razón de esto es que no hay nada en `sys.argv[1]` ¡porque no se escribió nada! Así es como podemos proteger nuestro programa de este tipo de error:

      import sys

      try:
          print("hola, mi nombre es", sys.argv[1])
      except IndexError:
          print("Demasiados pocos argumentos")

  Observa que ahora el usuario recibirá una pista útil sobre cómo hacer funcionar el programa si olvida escribir un nombre. Sin embargo, ¿podríamos ser más defensivos para garantizar que el usuario introduzca los valores correctos?

- Nuestro programa puede mejorarse de la siguiente manera:

      import sys

      if len(sys.argv) < 2:
          print("Demasiados pocos argumentos")
      elif len(sys.argv) > 2:
          print("Demasiados argumentos")
      else:
          print("hola, mi nombre es", sys.argv[1])

  Observa cómo si pruebas tu código, verás cómo se manejan estas excepciones, brindando al usuario consejos más refinados. Incluso si el usuario escribe demasiados o muy pocos argumentos, se le proporcionan instrucciones claras sobre cómo solucionar el problema.

- Actualmente, nuestro código es lógicamente correcto. Sin embargo, hay algo muy útil en mantener nuestra verificación de errores separada del resto de nuestro código. ¿Cómo podríamos separar nuestro manejo de errores? Modifica tu código de la siguiente manera:

      import sys

      if len(sys.argv) < 2:
          sys.exit("Demasiados pocos argumentos")
      elif len(sys.argv) > 2:
          sys.exit("Demasiados argumentos")

      print("hola, mi nombre es", sys.argv[1])

  Observa cómo estamos utilizando una función incorporada de `sys` llamada `exit` que nos permite salir del programa si se introduce un error por parte del usuario. Ahora podemos estar seguros de que el programa nunca ejecutará la última línea de código y generará un error. Por lo tanto, `sys.argv` proporciona una forma mediante la cual los usuarios pueden introducir información desde la línea de comandos. `sys.exit` ofrece un medio por el cual el programa puede salir si surge un error.

- Puedes aprender más en la documentación de Python sobre [`sys`](https://docs.python.org/3/library/sys.html).

## `slice` <a id="slice"></a>

- `slice` es un comando que nos permite tomar una `lista` y le indica al compilador dónde queremos que éste considere el inicio de la `lista` y el final de la `lista`. Por ejemplo, modifica tu código de la siguiente manera:
  
      import sys

      if len(sys.argv) < 2:
          sys.exit("Demasiados pocos argumentos")

      for arg in sys.argv:
          print("hola, mi nombre es", arg)

  Observa que si escribes `python nombre.py David Carter Rongxin` en la ventana de la terminal, el compilador dará como salida no solo los nombres, sino también `hola, mi nombre es nombre.py`. ¿Cómo podemos hacer para que el compilador ignore el primer elemento de la lista donde está almacenado actualmente `nombre.py`?

- ¡`slice` se puede utilizar en nuestro código para comenzar la lista en un lugar diferente! Modifica tu código de la siguiente manera:
  
      import sys

      if len(sys.argv) < 2:
          sys.exit("Demasiados pocos argumentos")

      for arg in sys.argv[1:]:
          print("hola, mi nombre es", arg)

  Observa que en lugar de comenzar la lista en `0`, utilizamos corchetes cuadrados para indicar al compilador que comience en `1` y vaya hasta el final utilizando el argumento `1:`. Al ejecutar este código, notarás que podemos mejorar nuestro código utilizando una sintaxis relativamente sencilla.

## Paquetes <a id="paquetes"></a>

- Una de las razones por las que Python es tan popular es que hay numerosas bibliotecas de terceros poderosas que agregan funcionalidades. Llamamos a estas bibliotecas de terceros, implementadas como una carpeta, "paquetes".
- PyPI es un repositorio o directorio de todos los paquetes de terceros disponibles actualmente.
- `cowsay` es un paquete conocido que permite que una vaca hable con el usuario.
- Python tiene un gestor de paquetes llamado `pip` que te permite instalar paquetes rápidamente en tu sistema.
- En la ventana de la terminal, puedes instalar el paquete `cowsay` escribiendo `pip install cowsay`. Después de un poco de salida, ahora puedes utilizar este paquete en tu código.
- En la ventana de tu terminal escribe `code say.py`. En el editor de texto, escribe lo siguiente:

      import cowsay
      import sys

      if len(sys.argv) == 2:
          cowsay.cow("hola, " + sys.argv[1])

  Observa que el programa primero verifica si el usuario introdujo al menos dos argumentos en la línea de comandos. Luego, la vaca debería hablar al usuario. Escribe `python say.py David` y verás a una vaca diciendo "hola" a David.

- Modifica aún más tu código:

      import cowsay
      import sys

      if len(sys.argv) == 2:
          cowsay.trex("hola, " + sys.argv[1])

  Observa que ahora un t-rex está diciendo "hola".

- Ahora puedes ver cómo instalar paquetes de terceros.
- Puedes aprender más en la entrada de PyPI para [`cowsay`](https://pypi.org/project/cowsay/)
- Puedes encontrar otros paquetes de terceros en [PyPI](https://pypi.org/)

## APIs <a id="apis"></a>

- Las APIs o "interfaces de programación de aplicaciones" te permiten conectar con el código de otros.
- `requests` es un paquete que permite que tu programa se comporte como lo haría un navegador web.
- En tu terminal, escribe `pip install requests`. Luego, escribe `code itunes.py`.
- Resulta que Apple iTunes tiene su propia API a la que puedes acceder en tus programas. En tu navegador de internet, puedes visitar [https://itunes.apple.com/search?entity=song&limit=1&term=weezer](https://itunes.apple.com/search?entity=song&limit=1&term=weezer) y se descargará un archivo de texto. David construyó esta URL leyendo la documentación de la API de Apple. Observa cómo esta consulta está buscando una `canción`, con un `límite` de un resultado, relacionado con el `término` llamado `weezer`. Al mirar este archivo de texto descargado, podrías encontrar que el formato es similar al que hemos programado previamente en Python.
- El formato en el archivo de texto descargado se llama JSON, un formato de texto utilizado para intercambiar datos basados en texto entre aplicaciones. Literalmente, Apple está proporcionando un archivo JSON que podríamos interpretar en nuestro propio programa en Python.
- En la ventana de la terminal, escribe `code itunes.py`. Código a continuación:

      import requests
      import sys

      if len(sys.argv) != 2:
          sys.exit()

      response = requests.get("https://itunes.apple.com/search?entity=song&limit=1&term=" + sys.argv[1])
      print(response.json())

  Observa cómo el valor devuelto de `requests.get` se almacenará en `response`. David, al haber leído la documentación de Apple sobre esta API, sabe que lo que se devuelve es un archivo JSON. Al ejecutar `python itunes.py weezer`, verás el archivo JSON devuelto por Apple. Sin embargo, la respuesta JSON es convertida por Python en un diccionario. ¡Al observar la salida, puede resultar bastante abrumadora!

- Resulta que Python tiene una biblioteca JSON incorporada que puede ayudarnos a interpretar los datos recibidos. Modifica tu código de la siguiente manera:

      import json
      import requests
      import sys

      if len(sys.argv) != 2:
          sys.exit()

      response = requests.get("https://itunes.apple.com/search?entity=song&limit=1&term=" + sys.argv[1])
      print(json.dumps(response.json(), indent=2))

  Observa que `json.dumps` está implementado de tal manera que utiliza `indent` para hacer que la salida sea más legible. Al ejecutar `python itunes.py weezer`, verás el mismo archivo JSON. Sin embargo, esta vez es mucho más legible. Observa ahora que verás un diccionario llamado `results` dentro de la salida. Dentro de ese diccionario llamado `results` hay numerosas claves presentes. Observa el valor de `trackName` en la salida. ¿Qué nombre de pista ves en tus resultados?

- ¿Cómo podríamos simplemente mostrar el nombre de esa pista? Modifica tu código de la siguiente manera:

      import json
      import requests
      import sys

      if len(sys.argv) != 2:
          sys.exit()

      response = requests.get("https://itunes.apple.com/search?entity=song&limit=50&term=" + sys.argv[1])

      o = response.json()
      for result in o["results"]:
          print(result["trackName"])

  Observa cómo estamos tomando el resultado de `response.json()` y almacenándolo en `o` (en minúsculas). Luego, estamos iterando a través de los `results` en `o` e imprimiendo cada `trackName`. También observa cómo hemos aumentado el número límite de resultados a `50`. Ejecuta tu programa. Observa los resultados.

- Puedes aprender más sobre `requests` a través de la [documentación de la biblioteca](https://docs.python-requests.org/).
- Puedes aprender más sobre JSON en la documentación de Python sobre [JSON](https://docs.python.org/3/library/json.html).

## Creación de tus Propias Bibliotecas <a id="making-your-own-libraries"></a>

- ¡Tienes la capacidad como programador de Python para crear tu propia biblioteca!
- ¡Imagina situaciones en las que puedas querer reutilizar fragmentos de código una y otra vez e incluso compartirlos con otros!
- Hasta ahora hemos escrito mucho código solo para decir "hola" en este curso. ¡Creemos un paquete que nos permita decir "hola" y "adiós"! En tu ventana de terminal, escribe `code sayings.py`. En el editor de texto, escribe el código de la siguiente manera:

      def hello(name):
          print(f"hola, {name}")


      def goodbye(name):
          print(f"adiós, {name}")

  Observa que este código en sí mismo no hace nada para el usuario. Sin embargo, si un programador importara este paquete en su propio programa, las capacidades creadas por las funciones anteriores podrían implementarse en su código.

- Veamos cómo podríamos implementar código utilizando este paquete que creamos. En la ventana de terminal, escribe `code say.py`. En este nuevo archivo en tu editor de texto, escribe lo siguiente:

      import sys

      from saying import goodbye

      if len(sys.argv) == 2:
          goodbye(sys.argv[1])

  Observa que este código importa las capacidades de `goodbye` del paquete `sayings`. Si el usuario ingresó al menos dos argumentos en la línea de comandos, dirá "adiós" junto con la cadena ingresada en la línea de comandos.
