## Condições <a id="condições"></a>

- As condições permitem que você, o programador, permita que seu programa tome decisões: como se seu programa tivesse a escolha entre pegar a estrada da esquerda ou da direita com base em determinadas condições.
- Dentro do Python, existem um conjunto de "operadores" que são usados para fazer perguntas matemáticas.
- Os símbolos `>` e `<` provavelmente são bem conhecidos por você.
- `>=` representa "maior que ou igual a".
- `<=` representa "menor que ou igual a".
- `==` representa "igual, mas repare no sinal de igual duplo! Um único sinal de igual atribuiria um valor. Sinais de igual duplos são usados para comparar variáveis.
- `!=` representa "diferente de".
- As declarações condicionais comparam um termo à esquerda com um termo à direita.

## Declarações if <a id="if-statements"></a>

- Na janela do seu terminal, digite `code compare.py`. Isso irá criar um novo arquivo chamado "compare".
- Na janela do editor de texto, comece com o seguinte:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x < y:
          print("x é menor que y")

  Observe como o seu programa recebe a entrada do usuário para x e y, convertendo-as para inteiros e salvando-as nas respectivas variáveis x e y. Em seguida, a declaração `if` compara x e y. Se a condição `x < y` for atendida, a declaração `print` será executada.

- As declarações `if` usam valores `bool` ou booleanos (verdadeiro ou falso) para decidir se devem ou não ser executadas. Se a declaração `x > y` for verdadeira, o compilador a registrará como `true` e executará o código.

## Fluxo de Controle, elif e else <a id="control-flow-elif-and-else"></a>

- Revise ainda mais o seu código da seguinte forma:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x < y:
          print("x é menor que y")
      if x > y:
          print("x é maior que y")
      if x == y:
          print("x é igual a y")

  Observe como você está fornecendo uma série de declarações `if`. Primeiro, a primeira declaração `if` é avaliada. Em seguida, a segunda declaração `if` executa sua avaliação. Por fim, a última declaração `if` executa sua avaliação. Esse fluxo de decisões é chamado de "fluxo de controle".

- Nosso código pode ser representado da seguinte forma: <pre class="mermaid">
flowchart TD
A([início]) --> B{x < y}
B -- Verdadeiro --> C["#quot;x é menor que y#quot;"]
C --> D{x > y}
D -- Verdadeiro --> E["#quot;x é maior que y#quot;"]
E --> F{x == y}
F -- Verdadeiro --> G["#quot;x é igual a y#quot;"]
G --> H([fim])
B -- Falso --> D
D -- Falso --> F
F -- Falso --> H
</pre>

- Este programa pode ser melhorado ao não fazer três perguntas consecutivas. Afinal de contas, nem todas as três perguntas podem ter um resultado "verdadeiro"! Revise seu programa da seguinte forma:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x < y:
          print("x é menor que y")
      elif x > y:
          print("x é maior que y")
      elif x == y:
          print("x é igual a y")

  Observe como o uso do `elif` permite que o programa faça menos decisões. Primeiro, a declaração `if` é avaliada. Se essa declaração for verdadeira, todas as declarações `elif` não serão executadas. No entanto, se a declaração `if` for avaliada e for falsa, a primeira declaração `elif` será avaliada. Se isso for verdadeiro, ele não executará a avaliação final.

- Nosso código pode ser representado da seguinte forma: <pre class="mermaid">
flowchart TD
A([início]) --> B{x < y}
B -- Verdadeiro --> C["#quot;x é menor que y#quot;"]
B -- Falso --> D{x > y}
D -- Verdadeiro --> E["#quot;x é maior que y#quot;"]
D -- Falso --> F{x == y}
F -- Verdadeiro --> G["#quot;x é igual a y#quot;"]
G --> H([fim])
F -- Falso --> H
C --> H
E --> H
</pre>

- Embora seu computador possa não notar uma diferença em termos de velocidade entre nosso primeiro programa e este programa revisado, considere como um servidor online que executa bilhões ou trilhões desses tipos de cálculos todos os dias pode definitivamente ser afetado por uma decisão de codificação tão pequena.
- Há uma última melhoria que podemos fazer em nosso programa. Observe como logicamente `elif x == y` não é uma avaliação necessária para ser executada. Afinal de contas, se logicamente x não é menor que y E x não é maior que y, x DEVE ser igual a y. Portanto, não precisamos executar o `elif x == y`. Podemos criar um resultado "por padrão", usando uma instrução `else`. Podemos revisar da seguinte forma:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x < y:
          print("x é menor que y")
      elif x > y:
          print("x é maior que y")
      else:
          print("x é igual a y")

  Observe como a complexidade relativa deste programa diminuiu por meio da nossa revisão.

- Nosso código pode ser representado da seguinte forma: <pre class="mermaid">
flowchart TD
A([início]) --> B{x < y}
B -- Verdadeiro --> C["#quot;x é menor que y#quot;"]
B -- Falso --> D{x > y}
D -- Verdadeiro --> E["#quot;x é maior que y#quot;"]
D -- Falso --> F["#quot;x é igual a y#quot;"]
F --> G([fim])
C --> G
E --> G
</pre>

## ou <a id="or"></a>

- `ou` permite que seu programa escolha entre uma ou mais alternativas. Por exemplo, poderíamos editar ainda mais nosso programa da seguinte forma:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x < y or x > y:
          print("x não é igual a y")
      else:
          print("x é igual a y")

  Note que o resultado do nosso programa é o mesmo, mas a complexidade é reduzida. A eficiência do nosso código é aumentada.

- Neste ponto, nosso código está muito bom. No entanto, será que o design poderia ser melhorado ainda mais? Poderíamos editar nosso código dessa forma:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x != y:
          print("x não é igual a y")
      else:
          print("x é igual a y")

  Note como removemos o `ou` completamente e simplesmente perguntamos: "x não é igual a y?" Fazemos apenas uma pergunta. Muito eficiente!

- Para fins de ilustração, também poderíamos alterar nosso código da seguinte forma:

      x = int(input("Qual é o valor de x? "))
      y = int(input("Qual é o valor de y? "))

      if x == y:
          print("x é igual a y")
      else:
          print("x não é igual a y")

  Note que o operador `==` avalia se o que está à esquerda e à direita são iguais. O uso de duas sinais de igual é muito importante. Se você usar apenas um sinal de igual, é provável que o compilador gere um erro.

- Nosso código pode ser ilustrado da seguinte forma: <pre class="mermaid">
flowchart TD
A([início]) --> B{x == y}
B -- Verdadeiro --> C["#quot;x é igual a y#quot;"]
B -- Falso --> E["#quot;x não é igual a y#quot;"]
C --> F([fim])
E --> F
</pre>

## e <a id="e"></a>

- Similar ao `or`, `e` pode ser usado em declarações condicionais.
- Execute no terminal o comando `python grade.py`. Inicie o seu novo programa da seguinte forma:

      score = int(input("Nota: "))

      if score >= 90 and score <= 100:
          print("Nota: A")
      elif score >=80 and score < 90:
          print("Nota: B")
      elif score >=70 and score < 80:
          print("Nota: C")
      elif score >=60 and score < 70:
          print("Nota: D")
      else:
          print("Nota: F")

  Observe que ao executar `python grade.py`, você poderá inserir uma nota e obter uma classificação. No entanto, observe como há potencial para erros.

- Normalmente, não queremos confiar que os usuários inserirão as informações corretas. Podemos melhorar nosso código da seguinte forma:

        score = int(input("Nota: "))

        if 90 <= score <= 100:
            print("Nota: A")
        elif 80 <= score < 90:
            print("Nota: B")
        elif 70 <= score < 80:
            print("Nota: C")
        elif 60 <= score < 70:
            print("Nota: D")
        else:
            print("Nota: F")

  Observe como o Python permite encadear os operadores e condições de uma maneira bastante incomum em outras linguagens de programação.

- Ainda assim, podemos melhorar ainda mais nosso programa:

      score = int(input("Nota: "))

      if score >= 90:
          print("Nota: A")
      elif score >= 80:
          print("Nota: B")
      elif score >= 70:
          print("Nota: C")
      elif score >= 60:
          print("Nota: D")
      else:
          print("Nota: F")

  Observe como o programa é aprimorado ao fazer menos perguntas. Isso torna nosso programa mais fácil de ler e muito mais sustentável no futuro.

## Módulo <a id="modulo"></a>

- Em matemática, paridade se refere a se um número é par ou ímpar.
- O operador módulo `%` na programação permite verificar se dois números dividem-se igualmente ou se dividem e possuem um resto.
- Por exemplo, 4 % 2 resultaria em zero, porque divide-se igualmente. No entanto, 3 % 2 não divide-se igualmente e resultaria em um número diferente de zero!
- Na janela terminal, crie um novo programa digitando `code parity.py`. Na janela do editor de texto, digite o código conforme abaixo:

      x = int(input("Qual é o valor de x? "))

      if x % 2 == 0:
          print("Par")
      else:
          print("Ímpar")

  Observe como nossos usuários podem digitar qualquer número maior ou igual a 1 para verificar se ele é par ou ímpar.

## Criando nossa própria função de paridade <a id="criando-nossa-própria-função-de-paridade"></a>

- Como discutido na Aula 0, será útil criar uma função própria!
- Podemos criar nossa própria função para verificar se um número é par ou ímpar. Ajuste seu código da seguinte forma:

      def main():
          x = int(input("Qual é o valor de x? "))
          if is_even(x):
              print("Par")
          else:
              print("Ímpar")


      def is_even(n):
          if n % 2 == 0:
              return True
          else:
              return False


      main()

  Observe que nossa declaração `if` `is_even(x)` funciona mesmo não havendo operador ali. Isso ocorre porque nossa função retorna um valor `bool` (ou booleano), verdadeiro (true) ou falso (false), de volta para a função principal. A declaração `if` simplesmente avalia se `is_even` de x é verdadeiro ou falso.

## Pythonic <a id="pythonic"></a>

- No mundo da programação, existem tipos de programação chamados de "Pythonic". Isso significa que há maneiras de programar que são exclusivas da programação em Python. Considere a seguinte revisão do nosso programa:

      def main():
          x = int(input("Qual é o valor de x? "))
          if is_even(x):
              print("Par")
          else:
              print("Ímpar")


      def is_even(n):
          return True if n % 2 == 0 else False


      main()

  Observe que esta instrução de retorno em nosso código é quase como uma frase em inglês. Essa é uma forma única de programar vista apenas em Python.

- Podemos revisar ainda mais nosso código e torná-lo cada vez mais legível:

      def main():
          x = int(input("Qual é o valor de x? "))
          if is_even(x):
              print("Par")
          else:
              print("Ímpar")


      def is_even(n):
          return n % 2 == 0


      main()

  Observe que o programa avaliará o que está acontecendo dentro de `n % 2 == 0` como verdadeiro ou falso e simplesmente retornará isso para a função principal.

## `match` <a id="match"></a>

- Semelhante às declarações `if`, `elif` e `else`, as declarações `match` podem ser usadas para executar condicionalmente o código que corresponde a determinados valores.
- Considere o seguinte programa:

        name = input("Qual é o seu nome? ")

        if name == "Harry":
            print("Grifinória")
        elif name == "Hermione":
            print("Grifinória")
        elif name == "Ron":
            print("Grifinória")
        elif name == "Draco":
            print("Sonserina")
        else:
            print("Quem?")

  Observe que as primeiras três declarações condicionais imprimem a mesma resposta.

- Podemos melhorar um pouco esse código usando a palavra-chave `or`:

        name = input("Qual é o seu nome? ")

        if name == "Harry" or name == "Hermione" or name == "Ron":
            print("Grifinória")
        elif name == "Draco":
            print("Sonserina")
        else:
            print("Quem?")

  Observe que o número de declarações `elif` diminuiu, melhorando a legibilidade do nosso código.

- Alternativamente, podemos usar declarações `match` para mapear nomes a casas. Considere o seguinte código:

        name = input("Qual é o seu nome? ")

        match name:
            case "Harry":
                print("Grifinória")
            case "Hermione":
                print("Grifinória")
            case "Ron":
                print("Grifinória")
            case "Draco":
                print("Sonserina")
            case _:
                print("Quem?")

  Observe o uso do símbolo `_` no último caso. Isso corresponderá a qualquer entrada, resultando em um comportamento semelhante a uma declaração `else`.

- Uma declaração `match` compara o valor seguinte à palavra-chave `match` com cada um dos valores seguintes às palavras-chave `case`. No caso de uma correspondência ser encontrada, a seção de código indentada correspondente é executada e o programa para de procurar correspondências.
- Podemos melhorar o código:

        name = input("Qual é o seu nome? ")

        match name:
            case "Harry" | "Hermione" | "Ron":
                print("Grifinória")
            case "Draco":
                print("Sonserina")
            case _:
                print("Quem?")

  Observe o uso da barra vertical `|`. Assim como a palavra-chave `or`, isso nos permite verificar vários valores na mesma declaração `case`.
