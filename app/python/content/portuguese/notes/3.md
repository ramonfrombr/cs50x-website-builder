## Exceções <a id="exceptions"></a>

- As exceções são coisas que dão errado dentro do nosso código.
- No nosso editor de texto, digite `code hello.py` para criar um novo arquivo. Digite o seguinte (com os erros intencionais incluídos):

      print("hello, world)

  Observe que intencionalmente deixamos de fora uma marcação de citação.

- Ao executar `python hello.py` na janela do nosso terminal, um erro é exibido. O compilador informa que é um "erro de sintaxe". Erros de sintaxe são aqueles que exigem que você verifique duas vezes se digitou corretamente o seu código.
- Você pode aprender mais na documentação do Python sobre [Erros e Exceções](https://docs.python.org/3/tutorial/errors.html).

## Erros de Tempo de Execução <a id="erros-de-tempo-de-execução"></a>

- Erros de tempo de execução referem-se àqueles criados por comportamentos inesperados dentro do seu código. Por exemplo, talvez você pretendesse que o usuário inserisse um número, mas ele inseriu um caractere em vez disso. Seu programa pode lançar um erro devido a essa entrada inesperada do usuário.
- Na janela do terminal, execute `code number.py`. O código segue como mostrado em seu editor de texto:

      x = int(input("Qual é o valor de x? "))
      print(f"x é {x}")

  Observe que ao incluir o `f`, estamos dizendo ao Python para interpolar o que está entre chaves como o valor de `x`. Além disso, ao testar seu código, você pode imaginar como alguém poderia facilmente digitar uma string ou um caractere em vez de um número. Ainda assim, um usuário pode não digitar nada - simplesmente pressionando a tecla Enter.

- Como programadores, devemos ser defensivos para garantir que nossos usuários estejam inserindo o que esperávamos. Podemos considerar "casos limite" como `-1`, `0` ou `gato`.
- Se executarmos esse programa e digitarmos "gato", veremos de repente `ValueError: literal inválido para int() com base 10: 'gato'`. Essencialmente, o interpretador do Python não gosta de termos passado "gato" para a função `print`.
- Uma estratégia eficaz para corrigir esse erro potencial seria criar um "tratamento de erro" para garantir que o usuário se comporte como pretendemos.
- Você pode aprender mais na documentação do Python sobre [Erros e Exceções](https://docs.python.org/3/tutorial/errors.html).

## `try` <a id="try"></a>

- Em Python, `try` e `except` são formas de testar a entrada do usuário antes de algo dar errado. Modifique o seu código da seguinte forma:

      try:
          x = int(input("Qual é o valor de x?"))
          print(f"x é {x}")
      except ValueError:
          print("x não é um número inteiro")

  Observe como, ao executar este código, a entrada de `50` será aceita. No entanto, digitar `cat` produzirá um erro visível ao usuário, informando o motivo pelo qual a entrada não foi aceita.

- Ainda não é a melhor forma de implementar esse código. Observe que estamos tentando executar duas linhas de código. Para uma boa prática, devemos apenas `try` a menor quantidade de linhas possível que poderiam falhar. Ajuste o seu código da seguinte forma:

      try:
          x = int(input("Qual é o valor de x?"))
      except ValueError:
          print("x não é um número inteiro")

      print(f"x é {x}")

  Observe que, embora isso alcance nosso objetivo de tentar o menor número possível de linhas, agora enfrentamos um novo erro! Encontramos um `NameError`, onde `x não está definido`. Olhe para esse código e considere: por que em alguns casos x não está definido?

- De fato, se examinarmos a ordem das operações em `x = int(input("Qual é o valor de x?"))`, trabalhando da direita para a esquerda, poderíamos pegar um caractere incorretamente inserido e tentar atribuí-lo como um número inteiro. Se isso falhar, a atribuição do valor de `x` nunca ocorre. Portanto, não há x para ser impresso na última linha do nosso código final.

## `else` <a id="else"></a>

- Descobre-se que existe outra maneira de implementar `try` que poderia capturar erros desse tipo.
- Ajuste o seu código da seguinte forma:

      try:
          x = int(input("Qual é o valor de x?"))
      except ValueError:
          print("x não é um número inteiro")
      else:
          print(f"x é {x}")

  Observe que, se nenhuma exceção ocorrer, o bloco de código dentro de `else` será executado. Ao executar `python number.py` e fornecer `50`, você notará que o resultado será impresso. Tentando novamente, desta vez fornecendo `cat`, você notará que o programa agora captura o erro.

- Considerando melhorar nosso código, observe que estamos sendo um pouco rudes com o nosso usuário. Se nosso usuário não cooperar, atualmente simplesmente encerramos o programa. Considere como podemos usar um loop para solicitar ao usuário o valor de `x` e continuar pedindo se eles não fornecerem o valor corretamente! Melhore o seu código da seguinte forma:

      while True:
          try:
              x = int(input("Qual é o valor de x?"))
          except ValueError:
              print("x não é um número inteiro")
          else:
              break

      print(f"x é {x}")

  Observe que `while True` criará um loop infinito. Se o usuário fornecer o valor correto, podemos sair do loop e imprimir o resultado. Agora, um usuário que inserir algo incorretamente será solicitado novamente para fornecer o valor corretamente.

## Criando uma Função para Obter um Número Inteiro <a id="creating-a-function-to-get-an-integer"></a>

- Certamente, muitas vezes gostaríamos de obter um número inteiro do usuário. Modifique seu código da seguinte maneira:

      def main():
          x = get_int()
          print(f"x é {x}")


      def get_int():
          while True:
              try:
                  x = int(input("Qual é o valor de x?"))
              except ValueError:
                  print("x não é um número inteiro")
              else:
                  break
          return x


      main()

  Observe que estamos manifestando várias propriedades importantes. Primeiro, abstraímos a capacidade de obter um número inteiro. Agora, todo o programa se reduz às primeiras três linhas do programa.

- Ainda assim, podemos aprimorar esse programa. Considere o que mais você poderia fazer para melhorá-lo. Modifique seu código da seguinte maneira:

      def main():
          x = get_int()
          print(f"x é {x}")


      def get_int():
          while True:
              try:
                  x = int(input("Qual é o valor de x?"))
              except ValueError:
                  print("x não é um número inteiro")
              else:
                  return x


      main()

  Observe que o `return` não somente interromperá um loop, mas também retornará um valor.

- Algumas pessoas podem argumentar que você poderia fazer o seguinte:

      def main():
          x = get_int()
          print(f"x é {x}")


      def get_int():
          while True:
              try:
                  return int(input("Qual é o valor de x?"))
              except ValueError:
                  print("x não é um número inteiro")


      main()

  Observe que isso faz a mesma coisa que a iteração anterior do nosso código, apenas com menos linhas.

## `pass` <a id="pass"></a>

- Podemos fazer com que nosso código não avise o usuário, mas simplesmente pergunte novamente a pergunta de solicitação, modificando nosso código da seguinte forma:

      def main():
          x = get_int()
          print(f"x é {x}")


      def get_int():
          while True:
              try:
                  return int(input("Qual é o valor de x?"))
              except ValueError:
                  pass


      main()

  Observe que nosso código ainda funcionará, mas não informará repetidamente o usuário sobre o erro. Em alguns casos, você desejará deixar muito claro para o usuário qual erro está sendo produzido. Em outros momentos, você pode decidir simplesmente pedir novamente a entrada.

- Uma última melhoria que poderia melhorar a implementação desta função `get_int`. No momento, observe que estamos atualmente dependendo do sistema de honra de que o `x` está presente tanto nas funções `main` quanto `get_int`. Provavelmente queremos passar uma solicitação que o usuário vê ao pedir a entrada. Modifique seu código da seguinte forma:

      def main():
          x = get_int("Qual é o valor de x? ")
          print(f"x é {x}")


      def get_int(prompt):
          while True:
              try:
                  return int(input(prompt))
              except ValueError:
                  pass


      main()

- Você pode aprender mais na documentação do Python sobre [`pass`](https://docs.python.org/3/tutorial/controlflow.html#pass-statements).
